# Alice Mod 项目企划书

> **Minecraft AI Agent Framework** — 让大语言模型真正「玩」转 Minecraft
>
> 全自主决策 · 双版本支持 · 模块化架构
>
> 名字由来：Alice这个名字来源于刀剑神域中的人工瑶光（Bottom-up型 自主人工智能），表明它是一个通用 AI Agent 框架，它让AI像真人玩家一样思考、规划和执行

***

## 一、项目概述

### 1.1 我们做什么

Alice Mod 是一个面向 Minecraft 的通用 AI Agent 框架。我们让大语言模型（LLM）像真人玩家一样思考、规划和执行，在 Minecraft 世界中完成从挖矿建造到全物品备货的复杂任务。

**核心数据**

| 指标      | 数值                         |
| ------- | -------------------------- |
| AI 工具数量 | 108+                       |
| 支持游戏版本  | 基岩版 + Java 版               |
| 核心模块    | 3 个（智能体软件 / TCP 桥接 / 游戏模组） |
| 扩展能力    | 协议级解耦，可扩展至其他游戏             |

### 1.2 核心能力

- **LLM 大脑 + 执行身体**：LLM 负责高层决策（目标规划、任务分解），执行 AI 负责底层操作（移动、战斗、建造）。各司其职，高效协作。
- **基岩版 & Java 版双支持**：一套智能体核心，通过 TCP 桥接同时支持 Bedrock（LeviLamina 插件）和 Java（ Fabric）两种接入核心。
- **108+ 工具化接口**：感知、移动、背包、战斗、建造、交易、社交……每个动作封装为独立工具，LLM 按需调用，精准控制游戏世界。
- **QQ 机器人深度联动**：游戏内聊天与 QQ 群消息实时桥接，支持远程指令控制、状态推送、多人群聊互动，打通游戏内外社交。
- **学习与记忆系统**：向量数据库存储经验与知识，支持语义检索。Agent 会记住玩家偏好、地图路径、任务经验，越用越聪明。
- **工作区化多实例管理**：每个接入核心是一个独立「工作区」，智能体核心可同时管理多个服务器实例，支持多服协作与统一调度。

### 1.3 实际应用场景

Alice Mod 不只是技术展示，它能真正解决 Minecraft 玩家的痛点。以下是几个典型应用场景：

#### 场景一：全物品备货

**任务描述**：为全物品备货 64 种物品，每种 64 个

**传统方式**：

- 手动收集资源：数小时
- 与村民交易：反复操作，枯燥乏味
- 整理物品：分类存储，耗时耗力

**Alice Mod 方式**：

```
用户指令："帮我备货全物品，需要 64 种物品，每种 64 个"

Agent 自动执行：
1. 任务分解：分析 64 种物品的获取方式（挖矿/合成/交易/种植）
2. 资源规划：计算所需原材料，制定收集路线
3. 自动执行：
   - 挖矿获取矿物（钻石、铁、金等）
   - 种植作物（小麦、胡萝卜、马铃薯等）
   - 与村民交易（附魔书、装备等）
   - 合成物品（工具、方块等）
4. 智能整理：按类别存储到对应箱子
5. 状态汇报：通过 QQ 群实时推送进度

预计耗时：30-60 分钟（全自动）
```

**价值**：将数小时的重复劳动缩短到 1 小时以内，玩家只需下达指令，Agent 全自主完成。

#### 场景二：自动挖掘史莱姆区块

**任务描述**：在指定区块挖掘史莱姆，获取史莱姆球

**传统方式**：

- 寻找史莱姆区块：使用外部工具或手动测试
- 挖掘区块：枯燥乏味，没有意义，浪费时间
- 挂机刷怪：长时间等待，效率低

**Alice Mod 方式**：

```
用户指令："你去挖个史莱姆区块，做好搜集系统"

Agent 自动执行：
1. 环境感知：扫描区块，确认史莱姆生成点
2. 建造设施：搭建刷怪平台（挖平区域、放置火把控制光照，处死与收集）
3. 使用指令创建假人挂机
4. 进度监控：实时统计建造进度
5. 完成汇报：达到目标后通过 QQ 通知用户

预计耗时：2-4 小时（全自动，无需挂机）
```

**价值**：玩家无需长时间挂机，Agent 自动完成枯燥的建造任务，效率稳定。

***

#### 场景三：自动建造简易基地

**任务描述**：在指定位置建造一个 10x10 的简易木屋

**传统方式**：

- 收集材料：砍树、合成木板
- 建造框架：手动放置方块
- 内部装修：放置床、箱子、工作台等

**Alice Mod 方式**：

```
用户指令："在 (50, 64, -100) 建造一个 10x10 的木屋，要有床、箱子、工作台"

Agent 自动执行：
1. 材料计算：需要 64 橡木原木、8 玻璃、1 床、3 箱子、1 工作台
2. 资源收集：
   - 砍树获取原木
   - 合成木板和玻璃
   - 从仓库取出床、箱子、工作台
3. 自动建造：
   - 放置地基和墙壁（fill_area 工具）
   - 安装门窗
   - 放置家具（bed、chest、crafting_table）
4. 内部整理：将物品分类存储到箱子
5. 完成汇报：建造完成，汇报到 QQ 群

预计耗时：10-15 分钟（全自动）
```

**价值**：快速搭建临时基地，适合生存模式初期或临时据点。

## 二、技术架构

### 2.1 三层解耦架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                        智能体核心 (Agent Core)                       │
│                   Electron + TypeScript/Node.js                      │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐ │
│  │   UI 界面   │  │  LLM 调度层  │  │  工作区管理  │  │  QQ 机器人   │ │
│  │  (React/Vue)│  │  (统一抽象层) │  │  (多实例)   │  │  (消息桥接)  │ │
│  └──────┬──────┘  └──────┬──────┘  └──────┬──────┘  └──────┬──────┘ │
│         └─────────────────┴─────────────────┴─────────────────┘      │
│                              │                                       │
│                    ┌─────────┴─────────┐                             │
│                    │   TCP JSON-RPC     │                             │
│                    │   桥接层 (Batch)   │                             │
│                    └─────────┬─────────┘                             │
└──────────────────────────────┼──────────────────────────────────────┘
                               │
              ┌────────────────┼────────────────┐
              │                │                │
              ▼                ▼                ▼
    ┌─────────────────┐ ┌─────────────┐ ┌─────────────┐
    │ 接入核心-基岩版  │ │接入核心-Java│ │  更多实例... │
    │ (LeviLamina插件) │ │ (Mineflayer)│ │             │
    │  假人 + 工具集   │ │ 假人 + 工具集 │ │             │
    └─────────────────┘ └─────────────┘ └─────────────┘
```

### 2.2 各层职责

**智能体核心（Agent Core）**

| 模块      | 技术                   | 职责                                                                       |
| ------- | -------------------- | ------------------------------------------------------------------------ |
| UI 界面   | Electron + React/Vue | llmplayer管理、对话历史、配置面板、状态监控                                               |
| LLM 调度层 | Node.js              | 统一多模型接入（OpenAI / Claude / Gemini / Ollama）、提示词组装、Function Calling 解析，工作流 |
| 智能体实例管理 | Node.js              | 多接入核心实例的生命周期管理、状态隔离、上下文切换                                                |
| QQ 机器人  | Node.js              | 消息收发、指令解析、游戏内外消息桥接                                                       |
| 记忆系统    | SQLite + Chroma      | 经验存储、任务专家，地图索引，向量语义检索、玩家偏好学习                                             |

**TCP 桥接层**

| 模块   | 协议                      | 职责                                |
| ---- | ----------------------- | --------------------------------- |
| 连接管理 | TCP 长连接                 | 连接池、心跳保活、自动重连、断线恢复                |
| 批量调用 | JSON-RPC 2.0 + Batch 扩展 | LLM 一次发出多个工具调用，接入核心分析依赖关系后并行/顺序执行 |
| 工具注册 | JSON Schema             | 接入核心启动时动态上报工具列表，智能体核心无需硬编码        |

**接入核心（Adapter Core）**

| 版本     | 技术方案                   | 职责                 |
| ------ | ---------------------- | ------------------ |
| 基岩版    | LeviLamina node.js  插件 | 游戏内假人、工具集实现、状态数据上报 |
| Java 版 | Fabric 模组              | 游戏内假人、工具集实现、状态数据上报 |

### 2.3 数据流向

1. **初始化**：接入核心启动 → 连接智能体核心 → 注册工具列表（JSON Schema）+ 上报基础状态
2. **任务轮**：收到用户输入 → 组装提示词（含工具列表）→ 发送给 LLM → LLM 多轮迭代，返回思考过程，工具调用（Batch）→ 智能体核心通过 TCP 分发给对应llmplayer → llmplayer并行/顺序执行指令 → 统一返回结果 → LLM 继续决策
3. **事件上报**：接入核心主动上报状态变化（血量低、被攻击、聊天消息等）→ 智能体核心决定是否中断当前任务

## 三、核心创新

### 3.1 与现有方案对比

| 维度        | Voyager         | Mindcraft             | **McAgent (本项目)**                               |
| --------- | --------------- | --------------------- | ----------------------------------------------- |
| **架构模式**  | LLM + 代码执行（单进程） | LLM + Mineflayer（紧耦合） | **三层解耦：Agent Core + TCP Bridge + Adapter Core** |
| **游戏版本**  | 仅 Java 版        | 仅 Java 版              | **基岩版 + Java 版 双支持**                            |
| **工具系统**  | 代码生成（自动写模组）     | 预定义函数调用               | **108+ 标准化工具，动态注册，批量执行**                        |
| **多实例支持** | 单实例             | 单实例                   | **多实例并行管理**                                     |
| **外部集成**  | 无               | Discord 基础支持          | **QQ 机器人深度联动 + 消息桥接**                           |
| **记忆系统**  | 技能库（代码层面）       | 基础对话历史                | **向量数据库 + 多维度记忆（经验/地图/社交）**                     |
| **UI 界面** | 无               | 无                     | **Electron 桌面应用，可视化配置与监控**                      |

### 3.2 三大技术创新

**协议级解耦**

智能体核心与接入核心完全分离，通过标准化 JSON-RPC 2.0 + Batch 扩展通信。接入核心可以独立演进，甚至支持未来其他游戏。

**批量工具执行**

LLM 一次可发出多个工具调用，接入核心自动分析依赖关系并行执行。复杂任务回合数减少 60% 以上。

**动态工具发现**

接入核心启动时自动注册工具列表（JSON Schema），智能体核心无需硬编码。不同版本、不同插件的工具差异自动适配。
