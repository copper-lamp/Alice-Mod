# V1 — 项目初始化执行计划

> **版本**：V1（项目初始化阶段）
> **开始日期**：2026-07-04
> **状态**：进行中
> **目标**：搭建 Monorepo 代码骨架，确保 3 模块 + 1 共享库可构建通过

---

## 1. 总体架构

```
McAgent/
├── packages/
│   ├── shared/           ← 共享类型库（TypeScript）
│   ├── agent-core/       ← Agent Core（Electron + React + TypeScript）
│   ├── adapter-bedrock/  ← BE 接入核心（TypeScript）
│   └── adapter-java/     ← JE 接入核心（Java 21 + Fabric + Gradle）
│
├── package.json          ← Monorepo 根配置
├── pnpm-workspace.yaml   ← pnpm 工作区声明
├── tsconfig.json         ← 全局 TypeScript 配置
├── .eslintrc.cjs         ← ESLint 配置
├── .prettierrc           ← Prettier 配置
└── .gitignore            ← Git 忽略规则
```

## 2. 构建工具链

| 工具 | 用途 | 版本要求 |
|------|------|----------|
| pnpm | 包管理 + Monorepo workspaces | ≥ 8.0 |
| TypeScript | 类型检查 | ≥ 5.4 |
| electron-vite | Electron + React 脚手架 | latest |
| ESLint | 代码检查 | ≥ 8.0 |
| Prettier | 代码格式化 | ≥ 3.0 |
| Vitest | TypeScript 单元测试 | latest |
| Gradle | Java 构建 | ≥ 8.0 |
| JUnit 5 | Java 单元测试 | latest |

## 3. 执行步骤

### Step 1: Monorepo 根配置

创建文件：
- `pnpm-workspace.yaml` — 声明 `packages/*` 工作区
- `package.json` — 根级 scripts（build/test/lint/typecheck）
- `tsconfig.json` — 全局 TS 配置，paths 映射
- `.eslintrc.cjs` — TypeScript + React 规则
- `.prettierrc` — 格式化配置
- `.gitignore` — 忽略规则

### Step 2: packages/shared/ 共享类型库

TypeScript 纯类型库，提供：
- `src/types/` — JSON-RPC 2.0 核心类型（Request, Response, Error）
- `src/schema/` — Tool Schema 定义
- `src/protocol/` — 协议校验工具
- `src/constants/` — 常量与枚举
- `src/index.ts` — 统一导出入口

### Step 3: packages/agent-core/ (Electron + React)

使用 electron-vite 初始化，构建：
- 主进程骨架：`src/main/` 模块目录（llm, workspace, tcp, memory, task, qq, trigger, log）
- 渲染进程骨架：`src/renderer/` React UI 目录（components, hooks, stores）
- 预加载脚本：`src/preload/`
- 测试目录：`__tests__/`

### Step 4: packages/adapter-bedrock/ (TypeScript)

LeviLamina 插件项目，构建：
- `src/` 源码目录（tcp, registry, bot, status, ai, tools 空架子）
- `tools/` 工具分类目录（perception, movement, inventory, entity, survival, block, chat）
- `manifest.json` — LeviLamina 插件清单
- `__tests__/` 测试目录

### Step 5: packages/adapter-java/ (Gradle + Fabric)

Fabric 模组 Gradle 项目，构建：
- Gradle 配置（build.gradle, settings.gradle）
- Java 源码骨架 `io.mcagent.adapter`
- Fabric mod 入口 + mixin 配置
- 测试目录

### Step 6: CI/CD 配置

- `.github/workflows/ci.yml` — TypeScript lint + typecheck + test + Java build
- `.github/workflows/release.yml` — 发布流程（V1 放空壳）

## 4. 依赖关系

```
Step 1: 根配置
  │
Step 2: shared（被 agent-core 和 adapter-bedrock 依赖）
  │
Step 3: agent-core + Step 4: adapter-bedrock（可并行，都依赖 shared）
  │
Step 5: adapter-java（独立，不依赖 TypeScript 包）
  │
Step 6: CI/CD（所有包就绪后配置）
```

## 5. 验收标准

| 检查项 | 验证方式 |
|--------|----------|
| `pnpm install` 成功 | 所有依赖安装无报错 |
| `pnpm build` 成功 | 所有包构建通过 |
| `pnpm lint` 通过 | ESLint 无报错 |
| `pnpm typecheck` 通过 | TypeScript 类型检查通过 |
| `pnpm test` 通过 | Vitest + JUnit 5 测试通过 |
| `packages/` 目录结构完整 | 4 个包各有正确的目录结构 |
| 各模块可独立构建 | `pnpm --filter <name> build` 成功 |
