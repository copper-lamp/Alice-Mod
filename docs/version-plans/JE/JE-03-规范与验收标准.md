## 第1章 概述

本文档定义 Adapter Core JE（Fabric 模组 + Java 21）的开发规范、各版本验收标准和测试策略。内容具体到每个版本的可测量、可验证的验收条件。

**适用范围**：所有 `io.mcagent.adapter.*` 包下的 Java 代码

**设计原则**：
- 每个验收标准必须是可测量的（数值指标）或可验证的（操作步骤可复现）
- 验收标准分为 P0（阻塞级，不满足不可发布）和 P1（重要级，可协商）
- 测试覆盖率达到指定阈值后方可验收通过

#### 2.2.3 缩进与格式

- 缩进：2 空格（禁止 Tab）
- 行宽：最大 100 字符
- 文件编码：UTF-8 without BOM
- 行尾：LF（Unix），文件末尾保留空行
- 禁止尾随空格

#### 2.2.4 Lombok 限制

仅允许使用以下两个 Lombok 注解：

| 注解 | 用途 | 示例 |
|------|------|------|
| `@Data` | 数据类（POJO/DTO） | 工具参数类、结果类、配置类 |
| `@Builder` | 构建器模式 | 配合 `@Data` 使用 |

**禁止使用**：`@Setter`、`@Getter`、`@AllArgsConstructor`、`@NoArgsConstructor`、`@ToString`、`@EqualsAndHashCode`、`@Slf4j`、`@RequiredArgsConstructor`、`@Value`、`@With`

#### 2.2.5 Optional 约束

- `Optional` 仅用于方法返回值，禁止用作字段类型或方法参数类型
- 集合类字段始终返回空集合而非 `null`（`Collections.emptyList()`）

#### 2.2.6 Stream API 约束

- Stream API 限于单行操作
- 复杂多行逻辑使用传统 for 循环

#### 2.2.7 日志规范

- 使用 SLF4J（`LoggerFactory.getLogger()`）
- 禁止 `System.out.println()` / `System.err.println()`
- 日志级别：`debug`（开发调试）、`info`（运行信息）、`warn`（警告）、`error`（错误）
- 工具执行关键路径必须记录 `info` 日志

```java
private static final Logger LOG = LoggerFactory.getLogger(MoveToTool.class);

public void execute(MoveToParams params) {
    LOG.info("move_to: target=({}, {}, {}), dimension={}", params.x, params.y, params.z, params.dimension);
}
```

### 2.3 @ToolModule/@ToolMethod 注解规范

#### 2.3.1 注解定义

```java
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
public @interface ToolModule {
    String category();        // 工具分类: perception/movement/inventory/entity/survival/block/chat
    String description();     // 模块描述
}

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface ToolMethod {
    String name();            // 工具名称（snake_case，全局唯一）
    String description();     // 工具描述（LLM 理解用途）
    ToolParam[] parameters() default {};
}

@Target(ElementType.PARAMETER)
@Retention(RetentionPolicy.RUNTIME)
public @interface ToolParam {
    String name();            // 参数名（snake_case）
    String type();            // 类型: "number", "string", "boolean", "object"
    String description();     // 参数描述
    boolean required() default true;  // 是否必填
}
```

#### 2.3.2 IToolModule 接口

```java
public interface IToolModule {
    String getCategory();
    String getDescription();
    List<ToolDefinition> getToolDefinitions();
    ToolResult execute(String toolName, Map<String, Object> params, ExecutionContext ctx);
}
```

#### 2.3.3 工具模块实现规范

- 每个工具类必须是单例模式（枚举单例或静态内部类）
- 每个 `@ToolModule` 类中的 `@ToolMethod` 方法参数类型为 `Map<String, Object>` 或具体参数 POJO
- 工具方法必须返回 `ToolResult` 类型（含 `success`/`data`/`duration_ms` 字段）
- 工具执行超时默认 30s，可通过 `@ToolMethod` 的 `timeout` 属性覆盖

```java
@ToolModule(category = "movement", description = "移动类工具")
public enum MovementTools {
    INSTANCE;

    @ToolMethod(
        name = "move_to",
        description = "移动到指定坐标位置",
        parameters = {
            @ToolParam(name = "x", type = "number", description = "目标 X 坐标"),
            @ToolParam(name = "y", type = "number", description = "目标 Y 坐标"),
            @ToolParam(name = "z", type = "number", description = "目标 Z 坐标"),
            @ToolParam(name = "dimension", type = "string", description = "维度", required = false)
        }
    )
    public ToolResult moveTo(Map<String, Object> params) {
        long start = System.currentTimeMillis();
        // ... 执行逻辑
        return ToolResult.success()
            .data("position", position)
            .durationMs(System.currentTimeMillis() - start)
            .build();
    }
}
```

### 2.4 通信协议规范

#### 2.4.1 JSON-RPC 2.0 over TCP

| 规范项 | 要求 |
|--------|------|
| 传输层 | TCP 长连接 |
| 应用层 | JSON-RPC 2.0 |
| 序列化 | JSON（UTF-8），Gson |
| 消息边界 | `\n` 换行符分隔 |
| 目标端口 | Agent Core 监听 27541，JE 主动连接 |

#### 2.4.2 Gson 序列化配置

```java
public class JsonUtil {
    private static final Gson GSON = new GsonBuilder()
        .setPrettyPrinting()
        .disableHtmlEscaping()
        .create();

    public static String toJson(Object obj) {
        return GSON.toJson(obj);
    }

    public static <T> T fromJson(String json, Class<T> clazz) {
        return GSON.fromJson(json, clazz);
    }

    public static <T> T fromJson(String json, Type type) {
        return GSON.fromJson(json, type);
    }
}
```

序列化性能要求：单次 JSON-RPC 消息序列化/反序列化 < 5ms（目标 < 2ms）。

#### 2.4.3 粘包处理

接收方按 `\n` 字节分割输入流：

```java
public class MessageFrameCodec {
    public List<String> decode(byte[] data) {
        // 按 \n (0x0A) 分割字节流
        // 返回完整消息帧列表
    }

    public byte[] encode(String message) {
        // 消息末尾追加 \n
        return (message + "\n").getBytes(StandardCharsets.UTF_8);
    }
}
```

| 参数 | 值 |

| 超时阈值 | 30 秒无 pong 视为断开 |

| 重连间隔 | 1s, 2s, 4s, 8s, 16s（指数退避） |
| 总重连超时 | 60 秒 |

### 2.5 寻路算法规范

#### 2.5.1 A* 3D 算法参数

| 参数 | 值 |
|------|-----|
| 搜索空间 | 三维网格，节点 = (x, y, z) 整数坐标 |
| 邻域 | 26 方向（3x3x3 减去自身中心） |
| 启发函数 | 三维欧几里得距离 `sqrt(dx² + dy² + dz²)` |
| 移动代价 | 水平 1.0，跳跃 2.0，下落 0.5，游泳 3.0 |
| 最大节点数 | 5000（超限视为不可达） |
| 最大搜索时间 | 50ms（超时返回失败） |

#### 2.5.2 方块可通过性判断

| 方块类型 | 可通过 |
|----------|:------:|
| 空气/草/花等 | ✅ 是 |
| 非完整方块（台阶/栅栏等） | ✅ 是（标记为"可跳跃通过"） |
| 完整方块（石头/泥土等） | ❌ 否 |
| 液体（水/熔岩） | ✅ 是（标记为"游泳" ，熔岩标记"危险"） |
| 岩浆块/仙人掌等伤害方块 | ❌ 否（除非无其他路径） |

```sql
CREATE TABLE IF NOT EXISTS memory_meta (
    id TEXT PRIMARY KEY,

    access_count INTEGER NOT NULL DEFAULT 0,
    created_at INTEGER NOT NULL DEFAULT (unixepoch()),
```

**logs 表**：

```sql
CREATE TABLE IF NOT EXISTS logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    level TEXT NOT NULL,
    source TEXT NOT NULL,
    message TEXT NOT NULL,
    data_json TEXT,

CREATE INDEX IF NOT EXISTS idx_logs_level ON logs(level);
```

#### 2.6.2 SQL 规范

- SQL 关键字大写（`SELECT`、`INSERT`、`CREATE TABLE` 等）
- 表名单数（`config`、`memory_meta`、`logs`）
- 字段名 snake_case（`created_at`、`content_json`）
- JSON 存储字段后缀 `_json`
- 时间戳使用 `INTEGER` 存储 unix 时间戳

```java
    }

    @Test
    void shouldMoveToCoordinateSuccessfully() {
        var params = Map.of(
            "x", 100.0, "y", 64.0, "z", 200.0,
            "dimension", "overworld"
        );
        var result = tool.execute(params);
        assertTrue(result.isSuccess());

        );

        assertFalse(result.isSuccess());

    }
}
```

## 第6章 与 BE 验收差异

### 6.1 JE 特有验收项

| 验收项 | JE 要求 | BE 对应 | 原因 |
|--------|---------|---------|------|
| GC 暂停间隔 < 200ms | ✅ 必须 | N/A | Java JVM 特有，BE 无此问题 |
| Gson 序列化 < 5ms | ✅ 必须 | N/A | BE 使用原生 JSON，无序列化开销 |
| Fabric 兼容性检查 | ✅ 必须 | N/A | BE 使用 LeviLamina，平台不同 |
| JDK 21 编译通过 | ✅ 必须 | N/A | BE 使用 Node.js |
| Spotless 格式检查 | ✅ 必须 | N/A | BE 使用 ESLint + Prettier |
| @Environment 注解正确 | ✅ 必须 | N/A | Fabric 特有的客户端/服务端区分 |
| Fabric mod.json 正确 | ✅ 必须 | N/A | BE 使用 manifest.json |

### 6.2 与 BE 相同的验收项

以下验收项 BE 和 JE 完全一致，只是实现方式不同：

| 验收项 | 验证方式 | JE 实现 | BE 实现 |
|--------|----------|---------|---------|
| 26 个工具全部注册 | Agent Core 工具列表 | `@ToolMethod` 注解扫描 | TypeScript 装饰器 |
| 状态上报 < 2s | Agent Core 接收频率 | Java Timer + Socket | Node.js setInterval |
| 寻路可达 50 格 | 假人实际移动 | A* on Minecraft Java API | A* on BDS API |
| TCP 握手认证 | 连接日志 | Java Socket + Gson | Node.js net + JSON |
| SQLite 持久化 | 数据库文件 | SQLite JDBC | better-sqlite3 |
| Fabric/LeviLamina 指令 | 游戏内命令 | Fabric Command API | LeviLamina Command API |

### 6.3 验收流程差异

JE 的验收需要额外关注以下方面：

1. **Fabric 版本兼容性**：每次验收前确认 Fabric Loader 和 Fabric API 版本与 mod.json 中的声明一致
2. **Java 版本检查**：验收环境中必须使用 JDK 21，使用 `java -version` 确认
3. **Minecraft 客户端环境**：JE 验收需要启动 Minecraft 客户端（含 GUI），无法在无头环境运行
4. **假人依赖客户端**：JE 假人需要客户端进入世界后才能创建，验收时需先手动进入游戏
5. **Gson 序列化预热**：首次工具调用可能因 Gson 类加载而较慢，验收时应先预热（执行一次任意工具）
