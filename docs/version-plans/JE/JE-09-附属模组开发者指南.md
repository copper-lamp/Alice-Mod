# Alice Mod JE 附属模组开发者指南

> 版本：v1.0
> 日期：2026-07-14
> 前置：[JE-08-工具拓展接口设计文档.md](JE-08-工具拓展接口设计文档.md)

---

## 概述

本文档面向希望为 Alice Mod 开发自定义工具的 **Fabric 模组开发者**。通过实现 `AliceToolPlugin` 接口，你创建的附属模组（Addon Mod）可以向 Alice Mod 注册自定义工具，这些工具与模组内建工具享有**完全对等的运行时能力**。

---

## 第1章 快速开始

### 1.1 创建项目

创建一个标准的 Fabric 模组项目，推荐目录结构：

```
my-custom-tools/
├── src/main/
│   ├── java/com/example/
│   │   ├── MyToolPlugin.java      # 插件入口
│   │   ├── MyCustomTool.java      # 工具实现
│   │   └── MyAddonInitializer.java # 模组主入口（可选）
│   └── resources/
│       └── fabric.mod.json
├── build.gradle
└── settings.gradle
```

### 1.2 配置依赖

在 `build.gradle` 中添加 Alice Mod API 依赖：

```groovy
repositories {
    mavenCentral()
    // 如果 Alice Mod 发布在特定 Maven 仓库
    maven { url "https://maven.alice-mod.io/releases" }
}

dependencies {
    // Fabric 标准依赖
    modImplementation "net.fabricmc:fabric-loader:0.16.0+"
    modImplementation "net.fabricmc.fabric-api:fabric-api:0.105.0+1.21.4"

    // Alice Mod API（编译期依赖）
    modCompileOnly "io.alice:alice-mod-api:1.0.0"

    // Alice Mod 本体（运行时依赖）
    modLocalRuntime "io.alice:alice-mod:1.0.0"
}
```

### 1.3 配置 fabric.mod.json

```json
{
  "schemaVersion": 1,
  "id": "my-custom-tools",
  "version": "1.0.0",
  "name": "My Custom Tools",
  "description": "Alice Mod 的自定义工具附属模组",
  "authors": ["你的名字"],
  "entrypoints": {
    "alice-mod:plugin": ["com.example.MyToolPlugin"],
    "main": ["com.example.MyAddonInitializer"]
  },
  "depends": {
    "alice-mod": ">=1.0.0",
    "fabricloader": ">=0.16.0",
    "minecraft": ">=1.21",
    "java": ">=21"
  }
}
```

> **关键点**：`alice-mod:plugin` 入口点是 Alice Mod 发现你的工具插件的唯一方式。

### 1.4 实现插件入口

```java
package com.example;

import io.alice.mod.adapter.api.AliceToolPlugin;
import io.alice.mod.adapter.api.AliceTool;
import io.alice.mod.adapter.api.ToolRegistrar;
import io.alice.mod.adapter.api.ToolResult;
import io.alice.mod.adapter.api.service.AliceServiceAccess;

import java.util.Map;

public class MyToolPlugin implements AliceToolPlugin {

    @Override
    public void registerTools(ToolRegistrar registrar, AliceServiceAccess services) {
        // 注册一个简单的问候工具
        registrar.register(new AliceTool() {
            @Override
            public String name() { return "my_custom_greet"; }

            @Override
            public String description() { return "向指定玩家打招呼"; }

            @Override
            public Map<String, Object> parameterSchema() {
                return Map.of(
                    "type", "object",
                    "properties", Map.of(
                        "target", Map.of(
                            "type", "string",
                            "description", "目标玩家名称"
                        )
                    ),
                    "required", new String[]{"target"}
                );
            }

            @Override
            public ToolResult invoke(Map<String, Object> args) {
                String target = (String) args.get("target");
                return ToolResult.ok("Hello, " + target + "!");
            }
        });
    }
}
```

## 第2章 使用服务接口

### 2.1 保存服务引用

在插件初始化时保存 `AliceServiceAccess`，在工具执行时使用：

```java
public class MyToolPlugin implements AliceToolPlugin {
    private AliceServiceAccess services;

    @Override
    public void registerTools(ToolRegistrar registrar, AliceServiceAccess services) {
        this.services = services;
        registrar.register(new BotQueryTool(services));
        registrar.register(new WorldQueryTool(services));
    }
}
```

### 2.2 操作假人

```java
public class BotQueryTool implements AliceTool {
    private final BotService botService;

    public BotQueryTool(AliceServiceAccess services) {
        this.botService = services.botService();
    }

    @Override
    public String name() { return "my_bot_health"; }

    @Override
    public String description() { return "查询指定假人的血量"; }

    @Override
    public Map<String, Object> parameterSchema() {
        return Map.of(
            "type", "object",
            "properties", Map.of(
                "bot_name", Map.of("type", "string", "description", "假人名称")
            ),
            "required", new String[]{"bot_name"}
        );
    }

    @Override
    public ToolResult invoke(Map<String, Object> args) {
        String botName = (String) args.get("bot_name");

        // 查找假人
        var botOpt = botService.findByName(botName);
        if (botOpt.isEmpty()) {
            return ToolResult.fail("NOT_FOUND", "Bot not found: " + botName);
        }

        var bot = botOpt.get();
        return ToolResult.ok(botName + " health: " + bot.health(), Map.of(
            "health", bot.health(),
            "max_health", bot.maxHealth(),
            "food", bot.foodLevel(),
            "position", Map.of(
                "x", bot.position().x(),
                "y", bot.position().y(),
                "z", bot.position().z()
            )
        ));
    }
}
```

### 2.3 查询游戏世界

```java
public class WorldQueryTool implements AliceTool {
    private final WorldService worldService;

    public WorldQueryTool(AliceServiceAccess services) {
        this.worldService = services.worldService();
    }

    @Override
    public String name() { return "my_block_check"; }

    @Override
    public String description() { return "检查指定位置的方块"; }

    @Override
    public Map<String, Object> parameterSchema() {
        return Map.of(
            "type", "object",
            "properties", Map.of(
                "x", Map.of("type", "number", "description", "X 坐标"),
                "y", Map.of("type", "number", "description", "Y 坐标"),
                "z", Map.of("type", "number", "description", "Z 坐标"),
                "dimension", Map.of("type", "string", "description", "维度")
            ),
            "required", new String[]{"x", "y", "z"}
        );
    }

    @Override
    public ToolResult invoke(Map<String, Object> args) {
        int x = ((Number) args.get("x")).intValue();
        int y = ((Number) args.get("y")).intValue();
        int z = ((Number) args.get("z")).intValue();
        String dimension = (String) args.getOrDefault("dimension", "overworld");

        String blockId = worldService.getBlockId(x, y, z, dimension);
        var properties = worldService.getBlockProperties(x, y, z, dimension);
        var weather = worldService.getWeather(dimension);
        var time = worldService.getGameTime(dimension);

        return ToolResult.ok("Block at " + x + "," + y + "," + z, Map.of(
            "block_id", blockId,
            "properties", properties,
            "weather", Map.of(
                "raining", weather.raining(),
                "thundering", weather.thundering()
            ),
            "game_time", time
        ));
    }
}
```

### 2.4 订阅事件

```java
public class EventToolPlugin implements AliceToolPlugin {
    @Override
    public void registerTools(ToolRegistrar registrar, AliceServiceAccess services) {
        // 订阅假人事件
        services.eventService().subscribe(BotEvent.SPAWNED, event -> {
            System.out.println("Bot spawned: " + ((BotEvent) event).botName());
        });

        // 订阅所有事件
        services.eventService().subscribeAll(event -> {
            System.out.println("Event: " + event.type());
        });
    }
}
```

### 2.5 发送通知到 Agent Core

```java
public class NotifyTool implements AliceTool {
```

## 第3章 最佳实践

### 3.1 工具命名

为避免与其他附属模组冲突，建议：

```java
@Override
public String name() {
    return "my_mod_tool_name";  // <你的模组ID>_<工具名>
}
```

### 3.2 错误处理

所有工具都应该稳定返回 `ToolResult`，而非抛出异常：

```java
@Override
public ToolResult invoke(Map<String, Object> args) {
    long start = System.currentTimeMillis();
    try {
        // 你的逻辑
        return ToolResult.ok("成功", data, start);
    } catch (IllegalArgumentException e) {
        return ToolResult.fail("INVALID_PARAMS", e.getMessage(), start);
    } catch (Exception e) {
        return ToolResult.fail("INTERNAL_ERROR", e.getMessage(), start);
    }
}
```

### 3.3 服务可用性检查

服务接口要求在 `invoke()` 中使用时有 Minecraft Server 上下文，建议检查 Server 状态：

```java
@Override
public ToolResult invoke(Map<String, Object> args) {
    var botOpt = botService.findByName("someBot");
    if (botOpt.isEmpty()) {
        // BotService 内部会检查 Server 可用性
        return ToolResult.fail("NOT_FOUND", "Bot not available");
    }
    // ...
}
```

### 3.4 使用逃生舱访问原生 Minecraft API

对于高阶需求，可以通过 `getNativePlayer()` 直接操作 Minecraft 原生类型：

```java
botOpt.ifPresent(bot -> {
    // 获取原生 ServerPlayer 实例
    ServerPlayer nativePlayer = bot.getNativePlayer();
    // 操作原生 API（注意版本兼容性）
    nativePlayer.setHealth(20.0f);
});
```

## 第4章 完整示例

### 示例：批量假人管理工具

一个综合使用 BotService、EventService 和 TcpService 的完整示例。

```java
package com.example;

import io.alice.mod.adapter.api.*;
import io.alice.mod.adapter.api.event.BotEvent;
import io.alice.mod.adapter.api.service.*;
import io.alice.mod.adapter.api.types.Vec3;

import java.util.List;
import java.util.Map;
import java.util.UUID;

/**
 * 批量假人管理工具。
 * 提供创建多个假人、批量操作、事件监控等功能。
 */
public class BatchBotPlugin implements AliceToolPlugin {

    private AliceServiceAccess services;

    @Override
    public void registerTools(ToolRegistrar registrar, AliceServiceAccess services) {
        this.services = services;
        registrar.register(new BatchSpawnTool(services));
        registrar.register(new MonitorTool(services));
    }
}

class BatchSpawnTool implements AliceTool {
    private final BotService botService;

    BatchSpawnTool(AliceServiceAccess services) {
        this.botService = services.botService();
    }

    @Override
    public String name() { return "batch_spawn"; }

    @Override
    public String description() { return "批量创建假人"; }

    @Override
    public Map<String, Object> parameterSchema() {
        return Map.of(
            "type", "object",
            "properties", Map.of(
                "names", Map.of(
                    "type", "string",
                    "description", "假人名称列表，逗号分隔（如 'bot1,bot2,bot3'）"
                ),
                "x", Map.of("type", "number", "description", "X 坐标"),
                "y", Map.of("type", "number", "description", "Y 坐标"),
                "z", Map.of("type", "number", "description", "Z 坐标")
            ),
            "required", new String[]{"names", "x", "y", "z"}
        );
    }

    @Override
    public ToolResult invoke(Map<String, Object> args) {
        long start = System.currentTimeMillis();
        try {
            String[] names = ((String) args.get("names")).split(",");
            double x = ((Number) args.get("x")).doubleValue();
            double y = ((Number) args.get("y")).doubleValue();
            double z = ((Number) args.get("z")).doubleValue();

            List<String> succeeded = new java.util.ArrayList<>();
            List<String> failed = new java.util.ArrayList<>();

            for (String name : names) {
                String trimmed = name.trim();
                try {
                    botService.spawn(trimmed, Vec3.of(x, y, z), "overworld");
                    succeeded.add(trimmed);
                } catch (Exception e) {
                    failed.add(trimmed + " (" + e.getMessage() + ")");
                }
            }

            return ToolResult.ok(
                "Spawned " + succeeded.size() + " bots",
                Map.of("succeeded", succeeded, "failed", failed),
                start
            );
        } catch (Exception e) {
            return ToolResult.fail("INTERNAL_ERROR", e.getMessage(), start);
        }
    }
}

class MonitorTool implements AliceTool {
    private final EventService eventService;

    MonitorTool(AliceServiceAccess services) {
        this.eventService = services.eventService();
        // 在工具构造时注册事件监听
        subscribeToEvents();
    }

    private void subscribeToEvents() {
        eventService.subscribe(BotEvent.SPAWNED, event -> {
            BotEvent be = (BotEvent) event;
            System.out.println("[Monitor] Bot spawned: " + be.botName());
        });
        eventService.subscribe(BotEvent.DEATH, event -> {
            BotEvent be = (BotEvent) event;
            System.out.println("[Monitor] Bot died: " + be.botName());
        });
    }

    @Override
    public String name() { return "monitor_status"; }

    @Override
    public String description() { return "查询监控状态"; }

    @Override
    public Map<String, Object> parameterSchema() {
        return Map.of("type", "object",
            "properties", Map.of(),
            "required", new String[]{}
        );
    }

    @Override
    public ToolResult invoke(Map<String, Object> args) {
        return ToolResult.ok("Monitor is active");
    }
}
```

## 第5章 限制与注意事项

### 5.1 注册阶段限制

在 `AliceToolPlugin.registerTools()` 中：

| ✅ 可以 | ❌ 不可以 |
|---------|-----------|
| 创建并注册工具 | 访问游戏世界（Server 未启动） |
| 保存服务引用供后续使用 | 操作假人 |
| 初始化工具内部数据结构 | 发送 TCP 消息（连接未建立） |

### 5.2 运行时限制

- 所有服务接口的方法都是线程安全的
- 每个工具在 `invoke()` 中的执行有超时限制（默认 30s）
- 工具名全局唯一，与内建工具冲突时插件工具会被跳过

### 5.3 版本兼容

- Alice Mod API 遵循语义化版本
- 接口默认方法用于向后兼容
- 废弃的接口会先标注 `@Deprecated` 再移除

## 第6章 常见问题

**Q: 我的工具需要 Fabric Mixin，可以吗？**

A: 可以。你的附属模组是完整的 Fabric 模组，可以使用 Mixin、注册方块/物品等一切 Fabric 能力。Alice Mod 不对此做任何限制。

**Q: 如何调试我的工具？**

A: 在开发环境中使用 `modLocalRuntime` 依赖 Alice Mod，像调试普通 Fabric 模组一样运行 Minecraft 客户端即可。

**Q: 我的工具可以在服务器端使用吗？**

A: Alice Mod 是客户端模组（需要图形界面运行），因此你的附属模组也应在客户端环境下运行。

**Q: 工具性能有保证吗？**

A: 与内建工具一样，你的工具通过服务接口访问底层能力，性能损耗仅多一层接口间接调用，可忽略不计。

```
io.alice.mod.adapter.api

├── annotation/            # 声明式注解（可选）
```
