## 第1章 概述

### 1.1 模块定位

V10 假人管理模块（C3）是 Alice Mod JE 的**核心执行载体**。所有 Agent Core 下发的工具调用最终都需要一个或多个游戏内「假人」（Bot）实体来执行。本模块负责：

- 基于 **Carpet Mod** 的 `EntityPlayerMPFake` 创建/销毁假人实例
- 维护多假人生命周期（生成、死亡、重生、休眠、销毁）
- 假人与 Agent Core 之间的**身份映射**（每个假人对应一个 AI 工作区）
- 向工具执行层暴露统一的假人访问接口（`IBotHandle`）
- 假人状态持久化（存档级 + 文件级双重保障）

### 1.2 核心设计原则

| 原则 | 说明 |
|------|------|
| **ServerPlayer 一等公民** | 假人继承 `ServerPlayer`，所有游戏机制天然兼容 |
| **Carpet 优先** | 基座使用 Carpet 的 `EntityPlayerMPFake`，不自研假人底层 |
| **Agent Core 统一调度** | 所有假人由 Agent Core 通过 `tool_call` 统一管理，模组不自主创建/销毁 |
| **身份隔离** | 每个假人拥有独立 UUID 和名称，互不干扰 |
| **轻量持久化** | 身躯走 `.dat` 存档自动持久化，索引走 SavedData + 文件双保险 |
| **无客户端依赖** | 假人不依赖客户端连接，通过 FakeConnection 空连接模式运行 |

### 1.3 关键术语

| 术语 | 说明 |
|------|------|
| **Bot** | 游戏内假人实体，继承 ServerPlayer |
| **BotManager** | 假人管理器的核心单例，负责 CRUD 和生命周期 |
| **IBotHandle** | 工具执行层获取的假人操作句柄 |
| **FakeConnection** | 假人网络连接的空实现（Discard 模式） |
| **BotRepository** | 假人注册表持久化（SavedData 机制） |
| **Work-Context** | 当前工具调用绑定的假人上下文 |

### 3.5 假人死亡流程

```
AliceBotPlayer.tick()
    │ 检测 health <= 0
    ▼
BotManager.onDeath(bot)
    │ 1. BotEventDispatcher.fireDeath(bot)
    │ 2. 通知 Agent Core (event notification)
    │ 3. 补满血量（保证 .dat 存档完整）
    │ 4. server.execute → 延迟一 tick 销毁
    ▼
CompanionFactory.despawn(server, bot)
    │ 1. server.getPlayerList().remove(player)
    │ 2. 保存 .dat 存档
    ▼
游戏刻检查 → 重生计时器
    │ 延迟 ~30s (600 ticks)
    ▼
BotManager.respawn(bot)
    │ 通过 BotRepository 找到注册信息
    │ 重新执行 spawn 流程
```

#### 4.1.1 `bot_spawn` — 创建假人

```json
{
  "name": "bot_spawn",
  "description": "创建一个游戏内假人。假人是继承 ServerPlayer 的完整玩家实体，可执行所有游戏操作。",
  "category": "management",
  "input_schema": {
    "type": "object",
    "properties": {
      "name": {
        "type": "string",
        "description": "假人名称，仅限字母数字和下划线，最长 16 字符"
      },
      "x": {
        "type": "number",
        "description": "生成位置 X 坐标"
      },
      "y": {
        "type": "number",
        "description": "生成位置 Y 坐标"
      },
      "z": {
        "type": "number",
        "description": "生成位置 Z 坐标"
      },
      "dimension": {
        "type": "string",
        "enum": ["overworld", "nether", "end"],
        "description": "生成维度",
        "default": "overworld"
      }
    },
    "required": ["name", "x", "y", "z"]
  },
  "output_schema": {
    "type": "object",
    "properties": {
      "uuid": { "type": "string", "description": "假人的 UUID" },
      "name": { "type": "string" },
      "position": {
        "type": "object",
        "properties": {
          "x": { "type": "number" },
          "y": { "type": "number" },
          "z": { "type": "number" }
        }
      },
      "dimension": { "type": "string" }
    }
  }
}
```

**成功响应**：
```json
{
  "jsonrpc": "2.0",
  "id": 10,
  "result": {
    "success": true,
    "data": {
      "uuid": "123e4567-e89b-12d3-a456-426614174000",
      "name": "Alice",
      "position": { "x": 0, "y": 64, "z": 0 },
      "dimension": "overworld"
    },
    "duration_ms": 2450
  }
}
```

**错误场景**：

| 错误码 | 原因 | 说明 |
|--------|------|------|
| `NAME_TAKEN` | 名称已被占用 | 同名假人已在线或已注册 |
| `INVALID_NAME` | 名称非法 | 名称含非法字符或超长 |
| `INTERNAL_ERROR` | 内部错误 | 游戏未就绪或 Carpet 异常 |

### 4.3 事件通知 — 假人生命周期事件

| 事件类型 | 触发时机 | 说明 |
|----------|----------|------|
| `bot_spawned` | 假人创建成功 | 在游戏中生成完毕 |
| `bot_despawned` | 假人休眠 | 已保存存档并移除 |
| `bot_dismissed` | 假人销毁 | 已永久删除 |
| `bot_died` | 假人死亡 | 死亡原因 + 预计重生时间 |
| `bot_respawned` | 假人重生 | 死亡后自动重生完成 |
| `bot_awakened` | 假人唤醒 | 从休眠被唤醒 |

**事件格式示例**：

```json
{
  "jsonrpc": "2.0",
  "method": "event",
  "params": {
```

### 6.2 假人创建 — 使用 EntityPlayerMPFake

Carpet Mod 的 `EntityPlayerMPFake` 提供了完整的假人玩家实体，继承自 `ServerPlayer`。我们的 `AliceBotPlayer` 通过继承它来获得假人能力。

```java
// Carpet 的 EntityPlayerMPFake 处理了：
// 1. ✅ constructor: (server, level, profile, ci)
// 2. ✅ doTick(): 假人 tick 驱动（必须手动调用 super.tick() 后的 doTick()）
// 3. ✅ connection: 使用自带的 FakeClientConnection（简化）
//
// AliceBotPlayer 在此基础上附加：
// 1. 死亡检测
// 2. 安全装备切换
```

### 6.3 动作控制 — 使用 EntityPlayerActionPack

Carpet 的 `EntityPlayerActionPack` 提供基础动作控制：

```java
// 获取动作包
EntityPlayerActionPack actions = ((EntityPlayerActionPack) bot);

// 移动
actions.start(EntityPlayerActionPack.Action.forward, 1);
actions.start(EntityPlayerActionPack.Action.jump, 1);

// 停止
actions.stop();
```

**需要注意**：Carpet 的动作包偏向于持续操作（长按 WASD 键），我们的工具系统更偏向于精确原子操作（走到指定坐标、挖一个方块）。因此：
- 移动类工具使用 `move_to`（自研 A* 寻路 + 路径跟随），不依赖 Carpet 动作包
- 持续操作类工具（如 `mine_block`）可以结合动作包实现
- **推荐方式**：直接操作 `ServerPlayer` 的移动和交互方法

### 6.4 FakeConnection 方案

Carpet 的 `EntityPlayerMPFake` 内部使用 `FakeClientConnection`。我们创建自己的 `FakeConnection` 以保证控制权：

```java
public class FakeConnection extends Connection {
    public FakeConnection() {
        super(PacketFlow.SERVERBOUND);
        new EmbeddedChannel(this);
    }
    
    @Override public void send(Packet<?> packet, PacketSendListener listener, boolean flush) {
        // 丢弃所有出站数据包
    }
    
    @Override public boolean isConnected() { return true; }
    
    @Override public void tick() { /* 不驱动 keep-alive */ }
    
    @Override public void disconnect(Component message) { /* 空操作 */ }
    
    @Override public void handleDisconnection() { /* 空操作 */ }
}
```

## 第7章 Agent Core 统一调度策略

### 7.1 工作区-假人映射

```
Agent Core                    Alice Mod JE
┌────────────────┐           ┌──────────────────────────────┐
│ 工作区 A        │           │  Server 实例（一个 Fabric 模组）│
│ └→ LLM 会话    │           │                               │
│ └→ 工具列表    │────TCP───→│  BotManager                   │
│ └→ 假人 Alice  │           │   ├── AliceBotPlayer "Alice"  │
│                │           │   ├── BuilderBotPlyr "Builder"│
│ 工作区 B        │           │   └── MinerBotPlayer "Miner"  │
│ └→ LLM 会话    │           │                               │
│ └→ 工具列表    │────TCP───→│  (同一个模组实例可关联         │
│ └→ 假人 Bob    │           │   多个工作区，每个对应一个假人) │
└────────────────┘           └──────────────────────────────┘
```

### 7.2 工具调用中指定假人

所有工具调用通过 `bot_name` 参数指定目标假人：

```json
{
  "method": "tool_call",
  "params": {
    "tool_name": "move_to",
    "bot_name": "Alice",
    "parameters": { "x": 100, "y": 64, "z": 200 }
  }
}
```

**Agent Core 侧**：工作区管理器记录每个工作区关联的假人名称，LLM 决策时自动在工具参数中注入 `bot_name`。

**模组侧**：工具执行前从参数提取 `bot_name`，通过 `BotManager.resolve(name)` 获取 `IBotHandle`。

### 7.3 多假人场景下的状态上报

状态上报中携带所有在线假人的状态列表，Agent Core 根据工作区-假人映射分发到对应工作区的 LLM 上下文。

### 7.4 假人创建流程（Agent Core 发起）

```
用户 → Agent Core UI 输入 "帮我创建一个叫 Alice 的助手"
    ↓
LLM 决策 → 调用 bot_spawn
    ↓
TCP → 模组执行 spawn → 假人上线
    ↓
Agent Core 创建工作区 A，绑定假人 Alice
    ↓
LLM 输入注入假人 Alice 的状态
    ↓
用户提问 "Alice 去挖点钻石"
    ↓
LLM 决策 → 调用 move_to + mine_block（batch，bot_name: Alice）
    ↓
TCP → 模组执行 → 返回结果
```

### 7.5 权限与隔离

| 场景 | 策略 |
|------|------|
| 名称冲突 | 拒绝创建，返回 `NAME_TAKEN` |
| 同一模组多假人 | UUID 天然隔离，互不干扰 |
| 工具误操作假人 | `bot_name` 参数确保工具操作指定假人 |
| 假人人权 | 假人走完整 ServerPlayer 路径，受游戏规则约束 |
