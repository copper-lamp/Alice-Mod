## 第1章 概述

### 1.1 模块定位

Agent Core 是 McAgent 系统的**智能体核心**，基于 Electron + TypeScript/Node.js + React 构建的桌面应用。它是整个系统的"大脑"，承担三个核心角色：

| 角色 | 说明 |
|------|------|
| **LLM 大脑** | 统一调度大语言模型，组装提示词、解析 Function Calling 结果、管理对话上下文 |
| **调度中心** | 管理工作区生命周期，通过 TCP JSON-RPC 协议与多个 Adapter Core 实例通信 |
| **记忆存储** | 基于 SQLite + Chroma 实现结构化和向量化双模存储 |

Agent Core **不依赖游戏运行**，可独立启动。它通过 TCP 连接与 Adapter Core（运行在游戏服务器内的模组/插件）通信，一个 Agent Core 可同时管理多个 Adapter Core 实例（跨服、跨版本）。

### 1.2 技术栈

| 技术 | 用途 |
|------|------|
| Electron | 桌面应用框架，主进程 + 渲染进程架构 |
| TypeScript | 主进程开发语言（Node.js 20+） |
| React | 渲染进程 UI 框架 |
| Node.js net | TCP 服务端 |
| better-sqlite3 + Chroma | 记忆系统存储 |
| Vitest | 单元测试 |
| electron-builder | 打包发布 |

### 1.3 团队规模

Agent Core 模块由 **2 名开发者**完成全部 15 个版本的迭代开发：

| 开发者 | 负责方向 | 核心模块 |
|--------|----------|----------|
| 开发者 A | UI + LLM + 提示词 | A1 UI 界面、A2 LLM 调度、A3 提示词系统、配置面板 |
| 开发者 B | 工作区 + TCP + 记忆 + 任务 | A4 工作区管理、A5 TCP 服务端、A6 记忆系统、A7 QQ 机器人、A8 事件触发器、A9 任务规划、A10 日志系统 |

## 第7章 V9 详细需求 — 后端调试模块（工具调用记录器 + 日志系统）

> 版本：V9（第 11 周）
> 对应需求：AC-UI-06、AC-LOG-01、AC-LOG-02、AC-LOG-03、AC-LOG-04

### 7.1 模块定位

V9 是 Agent Core **后端调试基础设施**的关键版本。本模块**不接入前端 UI**，纯后端运行，为开发者提供调试 Agent 执行过程的能力。它在 V4 Function Calling Pipeline 的事件系统基础上，引入**工具调用记录器**和**全功能日志系统**，所有数据通过文件、控制台、SQLite 三种渠道输出，供开发者离线或实时分析。

| 职责 | 说明 |
|------|------|
| **工具调用记录器** | 监听 Pipeline 事件，将每次工具调用的参数、结果、耗时、依赖关系持久化到 SQLite，并输出到日志文件和控制台 |
| **分级日志系统** | 基于级别的日志记录（debug/info/warning/error），支持多路输出（文件 + 控制台 + SQLite）和文件轮转 |
| **结构化日志存储** | 所有日志写入 SQLite logs 表，支持按级别/模块/时间范围检索 |
| **调试数据持久化** | 工具调用记录和日志均持久化到磁盘，支持事后回溯分析，不依赖前端渲染 |

**关键设计原则**：
- 纯后端运行，不依赖 Electron 渲染进程
- 工具调用数据通过 Pipeline 事件系统自动收集，零侵入
- 日志和工具调用记录均结构化存储，支持程序化查询
- 所有输出渠道（文件/控制台/SQLite）独立可配置
