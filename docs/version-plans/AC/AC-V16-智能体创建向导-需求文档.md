## 第1章 概述

### 1.1 模块定位

V16 重新设计 AC 的智能体创建流程，将原有的单页长表单改为**多页引导式创建向导**，提升用户体验和配置效率。同时重构后端数据结构，新增人设预设系统、工具分类展示、LLM 模型多角色选择等能力。

### 1.2 核心变更

| 变更项 | 现状（V8） | 目标（V16） |
|--------|-----------|------------|
| 表单布局 | 单页长表单，所有配置项纵向排列 | 多页引导式向导，分步配置 |
| 人设配置 | 简单的提示词片段勾选 | 完整的人设预设系统 + 高级自定义模式 |
| 工具配置 | 分类勾选（无详细信息） | 分类表格展示，中文名称，悬停详情，逐工具启停 |
| LLM 配置 | 单一模型选择 | 三角色模型选择（主智能体/QQ机器人/压缩） |
| 记忆/规则/时间 | 独立配置项 | 移入人设预设或高级选项中 |
| 后端的持久化 | 内存存储，重启丢失 | SQLite 持久化 + 预设存储 |
| 备注字段 | 无 | 新增实例管理显示名称（别名） |

## 第2章 详细需求

### 2.1 多页引导式创建向导（AC-CW-01）

#### 2.1.1 交互流程

```
点击左栏 [+ 新建]
  │
  └──→ 右侧面板切换为创建向导（全页表单）
         │
         ├── 步骤1: 基本信息  ───→ [下一步]
         ├── 步骤2: 人设配置  ───→ [下一步]  ───→ [上一步]
         ├── 步骤3: 工具配置  ───→ [下一步]  ───→ [上一步]
         ├── 步骤4: 机器人绑定 ───→ [下一步]  ───→ [上一步]
         └── 步骤5: LLM 配置  ───→ [确定]    ───→ [上一步]
                                      │
                                      └──→ 创建智能体，添加到实例列表
                                            └──→ 跳转到智能体实例视图
```

#### 2.1.2 向导 UI 布局

#### 2.1.3 步骤指示器规则

- 当前步骤高亮显示，已完成步骤显示 ✅ 图标
- 未完成步骤显示 `○`，当前步骤显示 `●`
- 点击已完成的步骤可跳回（步骤可回退编辑）
- 点击未完成的步骤不可跳转

### 2.2 基本信息页（AC-CW-02）

#### 2.2.1 表单字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| 智能体名称 | 文本输入 | 是 | 智能体的显示名称，用于 LLM 提示词中的身份标识 |
| 备注（可选） | 文本输入 | 否 | 实例管理页中显示的别名，便于用户识别不同实例。如不填则与名称相同 |
| 皮肤/头像 | 文件上传 | 否 | 支持 .png 格式，显示 64×64 预览。上传后转为 base64 存储 |

#### 2.2.2 交互规则

- 名称为空时，"下一步"按钮置灰禁用
- 皮肤上传后显示预览缩略图，支持删除重新上传
- 备注为空时，自动使用名称作为实例显示名

#### 2.3.2 预设数据结构

每个预设包含：
- 预设名称、描述
- 身份设定文本（identity）
- 专业设定标签列表（expertise tags）
- 性格特征列表（personality trait IDs）
- 工作流模板 ID（workflow template id）
- 关联的行为规则（behavior rules）

#### 2.3.3 交互规则

- 默认模式与高级模式通过切换按钮互斥切换
- 从预设模式切换到高级模式时，将预设的内容填充到自定义表单中
- 从高级模式切换到预设模式时，已选预设保留，但自定义内容丢失（需确认提示）
- 高级模式中可"保存为预设"，填写名称后保存到本地预设库
- 预设选择支持搜索过滤

#### 2.4.2 需求细节

- 工具按类别分组展示，每个类别一个可折叠区域
- 每个工具显示：复选框、中文名称、简短描述、分类标签

- 每行工具右侧有独立开关（Checkbox），可单独启用/停用
- 分类头部显示该分类的工具数量
- 默认全部启用
- 顶部显示已启用/总数统计
- 支持全选/取消全选
- 工具数据来源：从后端获取已注册的工具列表（ToolSchema）
- 工具名称需要中文翻译映射（后端或前端提供翻译表）

### 2.7 后端数据重构（AC-CW-08）

#### 2.7.1 新的 AgentConfig 数据结构

```typescript
/** 智能体完整配置（V16 新版） */
export interface AgentConfig {
  id?: string
  name: string
  alias?: string               // 备注/别名（实例管理页显示名）
  skinData?: string            // 皮肤 base64

  // 人设配置
  persona: AgentPersona
  personaPresetId?: string     // 使用的预设 ID（高级模式时可为空）

  // 工具配置
  tools: AgentToolConfig

  // QQ 绑定
  qqBinding: QQBinding

  // LLM 模型配置
  llmConfig: AgentLLMConfig

  // 元数据
  createdAt?: number
  updatedAt?: number
}

/** 人设配置 */
export interface AgentPersona {
  identity: string             // 身份设定文本
  expertise: string[]          // 专业设定标签
  personality: string[]        // 性格特征 ID 列表
  workflowId: string           // 工作流模板 ID
  behaviorRules?: {            // 行为规则（高级模式自定义）
    core: string[]
    strategy: StrategyRule[]
    constraints: ConstraintRule[]
  }
}

/** 人设预设 */
export interface PersonaPreset {
  id: string
  name: string
  description: string
  identity: string
  expertise: string[]
  personality: string[]
  workflowId: string
  behaviorRules: {
    core: string[]
    strategy: StrategyRule[]
    constraints: ConstraintRule[]
  }
  recommendedToolCategories: string[]
  isBuiltin: boolean           // true=内置, false=用户自定义
  createdAt?: number
}

/** 工具配置（V16 新版） */
export interface AgentToolConfig {
  enabledTools: Record<string, boolean>     // toolName → enabled
```

### 2.8 人设预设持久化（AC-CW-09）

- 内置预设（BUILTIN_IDENTITY_TEMPLATES）在代码中定义，不可修改
- 用户自定义预设存储在 SQLite 数据库的 `persona_presets` 表中
- 支持 CRUD 操作：创建、读取、更新、删除自定义预设
- 预设列表加载时合并内置预设和用户自定义预设

### 2.9 实例列表集成（AC-CW-10）

- 创建成功后的智能体自动添加到实例列表（左栏智能体列表）
- 列表中显示：名称（或备注）、在线状态圆点、工具数量
- 点击列表项跳转到智能体实例视图
- 列表项显示备注（如设置了别名）
