## 第1章 概述

本文档定义 Adapter Core BE（基岩版）模块的完整技术规范和各版本验收标准，确保：

1. **技术一致性**：LeviLamina 插件开发、TypeScript 编码、工具接口、通信协议、寻路算法、数据库设计等各方面有统一规范可循
2. **可验证性**：每个版本有具体、可测量、可操作的验收条件，避免模糊描述
3. **质量保障**：通过明确的测试策略和性能基准，确保模块质量达标

**与 JE 的差异**：BE 使用 LeviLamina Node.js 插件 + TypeScript，状态上报和工具执行通过 BDS API 实现。JE 使用 Fabric 模组 + Java 21。本文档仅适用于 BE。

## 第2章 技术规范

### 2.1 LeviLamina 插件开发规范

#### 2.1.1 manifest.json 结构

LeviLamina 插件的 `manifest.json` 必须遵循以下格式：

```json
{
  "format_version": 2,
  "header": {
    "name": "McAgent Adapter BE",
    "description": "McAgent Bedrock Edition Adapter Core",
    "version": [1, 0, 0],
    "min_engine_version": [1, 21, 0]
  },
  "modules": [
    {
      "type": "nodejs",
      "entry": "index.js",
      "language": "javascript"
    }
  ],
  "dependencies": [
    {
      "module_name": "legacy_remote_call",
      "version": [1, 0, 0]
    }
  ]
}
```

| 字段 | 约束 | 说明 |
|------|:----:|------|
| `format_version` | 必须为 2 | LeviLamina 0.12.x 标准 |
| `header.name` | 固定为 `"McAgent Adapter BE"` | 不允许自定义 |
| `header.version` | 语义化版本 `[MAJOR, MINOR, PATCH]` | 与插件包的版本一致 |
| `header.min_engine_version` | `[1, 21, 0]` | BDS 1.21.x 兼容 |
| `modules[0].type` | 必须为 `"nodejs"` | Node.js 类型插件 |
| `modules[0].entry` | 必须为 `"index.js"` | 入口文件名固定 |
| `dependencies` | 至少包含 `legacy_remote_call` | 确保远程调用API可用 |

### 2.2 TypeScript 编码规范

遵循 [standards/01-代码规范与风格指南.md](../standards/01-代码规范与风格指南.md) 第3章的全部规定，以下为 BE 模块的补充规范：

#### 2.2.1 文件命名

| 文件类型 | 命名规则 | 示例 |
|----------|----------|------|
| 工具实现文件 | kebab-case，与工具名一致 | `move-to.ts`, `drop-item.ts` |
| 接口定义文件 | kebab-case + `.types` 后缀 | `tool-module.types.ts` |
| 测试文件 | 与被测文件同名 + `.test` | `pathfinder.test.ts` |

#### 2.2.2 工具文件的导出约定

每个工具文件必须默认导出 `IToolModule` 接口的实现：

```typescript
// tools/movement/move-to.ts
import type { IToolModule, ToolMetadata, ToolResult, ToolContext } from '../../registry/tool-module.types';

export class MoveToTool implements IToolModule {
  metadata(): ToolMetadata {
    return {
      name: 'move_to',
      description: '移动假人到指定目标位置',
      category: 'movement',
      input_schema: { /* ... */ },
      output_schema: { /* ... */ },
      execution: {
        timeout_default_ms: 30000,
        timeout_max_ms: 120000,
        is_movement: true,
        is_async: false,
      },
    };
  }

  async execute(params: Record<string, any>, ctx: ToolContext): Promise<ToolResult> {
    // 工具实现
  }
}

// 必须导出默认实例
export default new MoveToTool();
```

#### 2.2.3 禁止使用的 API

| 禁止使用 | 替代方案 | 原因 |
|----------|----------|------|
| `console.log` / `console.error` | `logger.info` / `logger.error` | 日志分级输出 |
| `require('fs')` | `import { ... } from 'node:fs'` | ESM 模块规范 |
| `any` 类型（除极少数情况） | `unknown` + 类型断言 | 类型安全 |
| `process.exit()` | 通过插件生命周期正常关闭 | 不导致 BDS 崩溃 |
| 全局变量 | 依赖注入或模块级变量 | 避免污染 |

### 2.3 IToolModule 接口规范

#### 2.3.1 接口定义

```typescript
// registry/tool-module.types.ts

export type ToolCategory =
  | 'perception' | 'movement' | 'inventory'
  | 'entity' | 'survival' | 'block' | 'chat';

export interface ToolMetadata {
  name: string;
  description: string;
  category: ToolCategory;
  input_schema: Record<string, any>;
  output_schema?: Record<string, any>;
  execution?: {
    timeout_default_ms?: number;
    timeout_max_ms?: number;
    is_movement?: boolean;
    is_async?: boolean;
  };
}

export interface ToolContext {
  player: PlayerAccess;
  world: WorldAccess;
  bot: BotAccess;
  sendEvent(event: EventNotification): void;
  logger: Logger;
  getElapsedMs(): number;
}

export interface ToolResult {
  success: boolean;
  data?: Record<string, any>;
  error?: string;
  duration_ms: number;
}

export interface IToolModule {
  metadata(): ToolMetadata;
  execute(params: Record<string, any>, ctx: ToolContext): Promise<ToolResult>;
}
```

#### 2.3.2 接口实现约束

| 约束 | 说明 |
|------|------|
| **无状态** | 工具实现不应持有可变状态，所有状态通过 ToolContext 访问 |
| **幂等性** | 同一参数重复执行应产生相同结果（或合理近似） |
| **超时处理** | 超过 timeout_max_ms 必须抛出 TimeoutError |
| **错误返回** | 失败时必须返回 `{ success: false, error: '描述' }`，不抛异常 |
| **耗时记录** | 必须记录 `duration_ms`，从 execute 开始到返回的毫秒数 |
| **资源释放** | 执行完成后必须清理所有占用的游戏资源（如打开的容器界面） |

### 2.4 通信协议规范

遵循 [protocols/01-通信协议规范.md](../protocols/01-通信协议规范.md) 的全部规定，以下为 BE 模块的补充：

#### 2.4.1 TCP 客户端参数

| 参数 | 值 | 说明 |
|------|:---:|------|
| 默认连接地址 | `127.0.0.1:27541` | 可在 config.json 中配置 |
| 连接超时 | 5000ms | 超过此时间未连接成功视为失败 |
| 心跳回复超时 | 10000ms | 收到 ping 后必须在 10s 内回复 pong |
| 重连间隔 | 1s, 2s, 4s, 8s, 16s | 指数退避 |
| 最大重连次数 | 5 | 超过后停止自动重连 |
| 消息缓冲区大小 | 65536 bytes | 粘包处理缓冲区上限 |

#### 2.4.2 握手消息格式

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "hello",
  "params": {
    "instance_id": "<UUID v4>",
    "schema_version": "1.0.0",
    "auth_token": "<auth_token>",
    "game_version": {
      "edition": "bedrock",
      "version": "1.21.x"
    },
    "mod_version": "1.0.0"
  }
}
```

#### 2.4.3 工具调用响应格式

成功响应：
```json
{
  "jsonrpc": "2.0",
  "id": <request_id>,
  "result": {
    "success": true,
    "data": { ... },
    "duration_ms": <number>
  }
}
```

失败响应（不使用 JSON-RPC error，使用 result 中的 success=false）：
```json
{
  "jsonrpc": "2.0",
  "id": <request_id>,
  "result": {
    "success": false,
    "data": null,
    "error": "错误描述",
    "duration_ms": <number>
  }
}
```

超时响应（使用 JSON-RPC error）：
```json
{
  "jsonrpc": "2.0",
  "id": <request_id>,
  "error": {
    "code": -32005,
    "message": "Tool execution timeout",
    "data": {
      "tool_name": "<tool_name>",
      "timeout_ms": <timeout_ms>
    }
  }
}
```

### 2.5 寻路算法规范

#### 2.5.1 算法参数

| 参数 | 值 | 说明 |
|:----:|:---:|------|
| 算法 | A* 3D | 3D 网格寻路 |
| 步长 | 1 格 | 每次移动 1 个方块单位 |
| 最大搜索范围 | 128 格 | 起点到终点的直线距离超过 128 直接返回不可达 |
| 邻居扩展 | 26 方向 | 3x3x3 立方体除自身外全部方向 |
| 启发函数 | 3D 曼哈顿距离（高度权重 x2） | `|dx| + |dy|*2 + |dz|` |
| 最大迭代次数 | 100,000 | 防止死循环 |
| 路径平滑 | 是 | 去除共线冗余中间点 |

#### 2.5.2 通行规则

| 地形类型 | 通行规则 | 额外代价 |
|----------|----------|:--------:|
| 空气（可站立） | 可通行 | 1.0 |
| 高度差 1 格 | 自动跳跃 | 2.0 |
| 高度差 2 格 | 尝试搭方块或绕行 | 5.0 |
| 水源/水流 | 游泳模式通行 | 3.0 |
| 岩浆 | 不可通行 | ∞ |
| 仙人掌/岩浆块 | 不可通行 | ∞ |
| 栅栏/墙 | 不可通行（除非有门） | ∞ |
| 楼梯/台阶 | 可通行 | 1.2 |
| 活板门（打开） | 可通行 | 1.0 |
| 活板门（关闭） | 不可通行 | ∞ |

#### 2.5.3 接口定义

```typescript
interface PathOptions {
  max_range?: number;           // 默认 128
  avoid_danger?: boolean;       // 默认 true（避开岩浆等）
  allow_swim?: boolean;         // 默认 true
  max_jump?: number;            // 默认 2
  step_height?: number;         // 默认 1
  prefer_height?: number;       // 偏好高度（如地面移动优先保持 Y）
}

interface PathResult {
  found: boolean;
  path: Vec3[];                 // 路径节点列表（含起点终点）
  node_count: number;           // 搜索节点数
  duration_ms: number;          // 搜索耗时
  total_distance: number;       // 路径总长度
  reason?: string;              // 失败原因
}
```

### 2.6 数据库表设计规范

#### 2.6.1 命名规范

| 对象 | 规则 | 示例 |
|------|------|------|
| 表名 | 单数 snake_case | `config`, `memory_meta`, `log` |
| 字段名 | 小写 snake_case | `created_at`, `content_json` |
| 主键 | `id`（TEXT UUID 或 INTEGER AUTOINCREMENT） | `id TEXT PRIMARY KEY` |
| 外键 | 关联表名 + `_id` | `memory_id` |
| 索引 | `idx_` + 表名 + `_` + 字段 | `idx_log_level` |
| JSON 字段 | 后缀 `_json` | `content_json`, `data_json` |

#### 2.6.2 必须包含的时间戳

所有业务表必须包含 `created_at` 和 `updated_at` 字段：

```sql
created_at INTEGER NOT NULL DEFAULT (unixepoch()),
updated_at INTEGER NOT NULL DEFAULT (unixepoch())
```

#### 2.6.3 表设计

**config 表**：

```sql
CREATE TABLE config (
  key         TEXT    PRIMARY KEY,
  value       TEXT    NOT NULL,
  value_type  TEXT    NOT NULL DEFAULT 'string',
  updated_at  INTEGER NOT NULL DEFAULT (unixepoch()),
  description TEXT
);
```

预定义键见 [protocols/01 §7.2](../protocols/01-通信协议规范.md)。

**memory_meta 表**：

```sql
CREATE TABLE memory_meta (
  id            TEXT    PRIMARY KEY,
  type          TEXT    NOT NULL,
  tags          TEXT    DEFAULT '[]',
  importance    INTEGER NOT NULL DEFAULT 5,
  access_count  INTEGER NOT NULL DEFAULT 0,
  created_at    INTEGER NOT NULL DEFAULT (unixepoch()),
  updated_at    INTEGER NOT NULL DEFAULT (unixepoch()),
  expires_at    INTEGER,
  content_json  TEXT    NOT NULL,
  embedding_id  TEXT
);

CREATE INDEX idx_memory_meta_type       ON memory_meta(type);
CREATE INDEX idx_memory_meta_importance ON memory_meta(importance);
```

**log 表**：

```sql
CREATE TABLE log (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  level      TEXT    NOT NULL,
  source     TEXT    NOT NULL,
  message    TEXT    NOT NULL,
  data_json  TEXT,
  created_at INTEGER NOT NULL DEFAULT (unixepoch())
);

CREATE INDEX idx_log_level     ON log(level);
CREATE INDEX idx_log_created   ON log(created_at);
```

// 性能基准测试示例（使用 Vitest）
describe('Performance benchmarks', () => {
  // 寻路性能
  it('pathfinding should complete within 50ms for 128 range', async () => {
    const pathfinder = new Pathfinder(mockWorld);
    const start = { x: 0, y: 64, z: 0 };
    const end = { x: 100, y: 64, z: 100 };
    
    const startTime = performance.now();
    const result = await pathfinder.findPath(start, end, { max_range: 128 });
    const elapsed = performance.now() - startTime;
    
    expect(result.found).toBe(true);
    expect(elapsed).toBeLessThan(50);
  });

  // 状态上报性能
  it('status report should complete within 100ms', async () => {
    const reporter = new StatusReporter(mockPlayer, mockWorld);

    const elapsed = performance.now() - startTime;

    expect(report).toHaveProperty('position');
    expect(elapsed).toBeLessThan(100);
  });
});
