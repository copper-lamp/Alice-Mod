# 假人模块实现指引

## 概述

本文档指导假人模块（Bot Module）的实现。该模块负责在 BDS 中创建、管理模拟玩家（SimulatedPlayer），提供完整的状态管理、数据持久化、工具函数支持。

## 模块划分

```
src/
├── bot/
│   ├── BotInstance.ts          # 假人实例（生命周期、状态、操作、同步）
│   └── BotManager.ts           # 假人管理器（CRUD、上线/下线、持久化）
├── utils/
│   └── helpers.ts              # 工具函数（坐标计算、解析、验证）
├── persistence/
│   └── index.ts                # 数据持久化（JSON文件 + NBT序列化）
└── config/
    └── index.ts                # 配置管理
```

## 1. BotInstance — 假人实例

### 职责

封装单个假人的完整生命周期：生成、上线、下线、传送、状态同步、操作循环、数据序列化。

### 核心接口

```typescript
// 状态枚举
enum BotStatus {
  OFFLINE = 0,
  ONLINE = 1,
}

// 操作类型
enum BotOperation {
  ATTACK = 'attack',
  DESTROY = 'destroy',
  INTERACT = 'interact',
  USE_ITEM = 'useitem',
  CLEAR = 'clear',
}

// 坐标结构
interface Position {
  x: number;
  y: number;
  z: number;
  dimid: number;
}

// 方向
interface Direction {
  yaw: number;
  pitch: number;
}

// 背包快照（用于远程查询）
interface InventorySnapshot {
  hand: { name: string; count: number } | null;
  offHand: { name: string; count: number } | null;
  inventory: ({ name: string; count: number } | null)[];
  armor: ({ name: string; count: number } | null)[];
}
```

### 类设计

```typescript
class BotInstance {
  // ---- 只读属性 ----
  readonly name: string;

  // ---- 内部状态 ----
  private _pos: Position;
  private _direction: Direction;
  private _gameMode: number;
  private _status: BotStatus;
  private _operation: BotOperation;
  private _opInterval: number;
  private _opMaxTimes: number;
  private _opLength: number;
  private _syncTarget: string;      // 同步目标的 xuid
  private _opTimer: any;            // setTimeout handle
  private _lastSyncPos: Position | null;

  constructor(name: string, pos: Position, ...);

  // ---- 生命周期 ----
  online(): boolean;        // 调用 mc.spawnSimulatedPlayer() 生成
  offline(): boolean;       // 调用 pl.simulateDisconnect() 下线
  isOnline(): boolean;

  // ---- 位置 & 方向 ----
  getPos(): Position;
  setPos(x, y, z, dimid): void;
  updatePos(): void;        // 从游戏内玩家同步位置
  getDirection(): Direction;
  setDirection(yaw, applyNow?): void;
  applyDirection(): void;   // 调用 pl.simulateSetBodyRotation()

  // ---- 游戏模式 ----
  getGameMode(): number;
  setGameMode(mode, applyNow?): void;
  applyGameMode(): void;

  // ---- 操作循环 ----
  setOperation(op, interval, maxTimes, length?): void;
  clearOperation(): void;
  startOperationLoop(): void;
  stopOperationLoop(): void;

  // ---- 玩家同步 ----
  startSync(targetXuid): void;
  stopSync(): void;
  tickLoop(): void;         // 每 tick 同步移动/视角

  // ---- 数据序列化 ----
  serialize(): BotData;
  static deserialize(name, data): BotInstance;
  serializeInventory(): string;   // NBT → SNBT
  deserializeInventory(snbt): boolean;

  // ---- 查询 ----
  getPlayer(): Player | null;     // mc.getPlayer() 包装
  getInfo(): BotInfo;
}
```

### 关键实现要点

- `online()` 执行顺序：`mc.spawnSimulatedPlayer()` → `teleport()` → `applyDirection()` → `applyGameMode()` → `startOperationLoop()`（如有操作）
- `offline()` 执行顺序：`stopOperationLoop()` → `updatePos()`/`updateDirection()`/`updateGameMode()` → `pl.simulateDisconnect()`
- 操作循环使用 `setTimeout` 链式调用，短操作（attack/interact）即时执行，长操作（useitem）按 `opLength` 持续
- `tickLoop()` 每 tick 由 `mc.listen('onTick', ...)` 驱动，处理同步逻辑

## 2. BotManager — 假人管理器

### 职责

单例静态管理器，负责所有假人的全局管理、上线/下线调度、数据持久化。

### 类设计

```typescript
class BotManager {
  private static instances: Record<string, BotInstance> = {};
  private static tickInstances: Record<string, BotInstance> = {};

  // ---- CRUD ----
  static create(name: string, pos: Position, owner?: string): string | void;
  static remove(name: string): void;
  static get(name: string): BotInstance | null;
  static getAll(): BotInstance[];
  static forEach(callback: (name: string, bot: BotInstance) => void): void;
  static list(): string[];

  // ---- 上线/下线 ----
  static online(name: string): string | void;
  static offline(name: string): string | void;
  static onlineAll(executor?: Player): string[];
  static offlineAll(executor?: Player): string[];

  // ---- 传送 ----
  static teleportToPos(name: string, pos: Position): string | void;
  static teleportToEntity(name: string, entity: Entity): string | void;

  // ---- 行走 ----
  static walkToPos(name: string, pos: Position): WalkResult | string;
  static walkToEntity(name: string, entity: Entity): WalkResult | string;

  // ---- 背包 ----
  static getInventory(name: string): InventorySnapshot | string;
  static giveItem(name: string, source: Player): string | void;
  static dropItem(name: string, slotId?: number): string | void;
  static dropAllItems(name: string): string | void;

  // ---- 同步 ----
  static startSync(name: string, target: Player): string | void;
  static stopSync(name: string): string | void;

  // ---- 事件回调 ----
  static onTick(): void;
  static onPlayerDie(player: Player, source: any): void;

  // ---- 持久化 ----
  static saveData(name: string, updatePos?: boolean): boolean;
  static loadAllData(): boolean;
  static deleteData(name: string): boolean;
  static saveInventory(name: string): boolean;
  static loadInventory(name: string): boolean;
  static deleteInventory(name: string): boolean;

  // ---- 启动 ----
  static init(): void;
}
```

### 关键实现要点

- `online()` 逻辑：获取实例 → 检查状态 → `instance.online()` → 加入 `tickInstances`（如需 tick）→ 保存数据 → 切换列表
- `offline()` 逻辑：获取实例 → 移出 `tickInstances` → 保存背包 → `instance.offline()` → 保存数据 → 切换列表
- `loadAllData()` 在 `onServerStarted` 时调用，遍历 `fpdata/` 目录下的 JSON 文件反序列化
- `onTick()` 事件驱动 `tickInstances` 中所有实例的 `tickLoop()`
- `onPlayerDie()` 自动重生假人，支持死亡频率检测（连续5次/20秒则自动下线）

```typescript
// 保存 — 从 Player NBT 中提取 Inventory/Armor/Offhand 标签
function serializeInventory(player: Player): string {
  const nbt = player.getNbt();
  const compound = new NbtCompound();
  compound.setTag('Inventory', nbt.getTag('Inventory'));
  compound.setTag('Armor', nbt.getTag('Armor'));
  compound.setTag('Offhand', nbt.getTag('Offhand'));
  return compound.toSNBT();
}

// 恢复 — 将 SNBT 写回 Player NBT
function deserializeInventory(player: Player, snbt: string): boolean {
  const compound = NBT.parseSNBT(snbt);
  const nbt = player.getNbt();
  nbt.setTag('Inventory', compound.getTag('Inventory'));
  nbt.setTag('Armor', compound.getTag('Armor'));
  nbt.setTag('Offhand', compound.getTag('Offhand'));
  return player.setNbt(nbt);
}
```

## 7. 入口架构

### 插件格式

采用 LLSE-FakePlayer 的直接执行模式，**不使用** `module.exports` 生命周期钩子：

```typescript
// src/index.ts（伪代码）
initPlugin();  // 文件末尾直接调用
```

### 命令注册

使用 LLSE 新命令 API（**不**使用 `mc.regPlayerCmd`）：

```typescript
const cmd = mc.newCommand('mcagent', '描述', PermType.Any, 0x80);
cmd.setEnum('Action', ['subcommand']);
cmd.mandatory('action', ParamType.Enum, 'Action', 'Action', 1);
cmd.overload(['Action']);
cmd.setCallback((_cmd, origin, output, result) => {
  output.success('消息');
  output.error('错误');
});
cmd.setup();
```

### 注册时机

命令注册必须放在 `onServerStarted` 事件回调中，此时 BDS 命令系统已就绪：

```typescript
mc.listen('onServerStarted', () => {
  registerCommands();
  BotManager.loadAllData();
  BotManager.initialAutoOnline();
});
```

### 重要约束

- 传递给 LLSE API 的坐标参数必须使用 `new FloatPos(x, y, z, dimid)` 构建，不能传 plain object
- `logger` 全局对象在脚本加载时即可使用，无需等待生命周期回调
- `require()` 在顶层直接使用，无需 `import`（避免 TypeScript 生成 `__esModule` 头）
