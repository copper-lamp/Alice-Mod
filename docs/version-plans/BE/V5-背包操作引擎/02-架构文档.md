## 第1章 模块定位

V5 背包操作引擎是 Adapter Core BE 执行 AI 引擎（C5）的第三个子模块，负责把 Agent Core 下发的物品管理意图转换为游戏内可执行操作。本模块遵循“LLM 做高层决策、执行 AI 处理细节”的原则：LLM 只决定“丢什么、拿什么、放什么、装备什么”，引擎自动完成物品查找、槽位选择、容器交互、移动靠近、堆叠校验、耐久与装备位映射等细节。

本模块交付 4 个 `inventory` 分类工具：

| 工具 | 说明 | 移动依赖 |
|------|------|:--------:|
| `drop_item` | 丢弃背包中指定物品，可指定数量，也可向实体附近丢弃 | 否 |
| `take_from_container` | 从容器（箱子、熔炉、桶等）取出物品到背包 | 是 |
| `put_to_container` | 将背包中的物品放入容器 | 是 |
| `equip_item` | 装备或卸下 head / chest / legs / feet / offhand 部位 | 否 |

```typescript
  success: boolean;
  item?: string;
  dropped?: number;
  remaining?: number;

export interface EquipResult {
  success: boolean;
  item?: string;
  slot?: ArmorSlot;
  previousItem?: string;
  error?: string;

  success: boolean;
  item?: string;
  transferred?: number;
  remaining?: number;
```


## 第5章 关键设计决策

### 5.1 查找顺序

`InventoryEngine.findAll()` 按 `hand > offhand > inventory > armor` 的顺序返回匹配槽位，确保 LLM 看到的“第一个”物品与玩家主观认知一致。

### 5.2 名称匹配策略

- 忽略 `minecraft:` 命名空间前缀。
- 忽略大小写。
- 支持带下划线的标准物品 ID（如 `iron_helmet`）。
- 具体实现见 [InventoryEngine.ts 中的 normalizeName / matchName](file:///d:/McAgent/packages/adapter-bedrock/src/ai/inventory/InventoryEngine.ts)。

### 5.3 装备位映射

| slot 参数 | 来源容器 | 索引 |
|-----------|----------|:----:|
| head | `player.getArmor()` | 0 |
| chest | `player.getArmor()` | 1 |
| legs | `player.getArmor()` | 2 |
| feet | `player.getArmor()` | 3 |
| offhand | `player.getOffHand()` | 0（独立容器） |

### 5.4 堆叠校验

[stack-size.ts](file:///d:/McAgent/packages/adapter-bedrock/src/ai/inventory/stack-size.ts) 维护常见物品分类映射，返回 64 / 16 / 1 三类最大堆叠数。`ContainerAPI.transfer()` 在合并与填入空槽时均遵守该上限，避免超堆叠或数量丢失。

### 5.5 容器交互距离

- `ContainerAPI.isWithinReach()` 校验玩家与容器方块中心距离 ≤ 5 格。
- 容器类工具在操作前调用 `aiEngine.moveTo()` 移动到 `computeApproachTarget()` 计算的近点（默认距离 2 格）。

### 5.6 异常与回滚

- `ContainerAPI.close()` 在 take/put 成功后清理引用；异常路径中工具也会调用 `containerAPI?.close()`。
- `InventoryEngine.equip()` 在卸下旧装备失败时直接返回错误，避免裸装备位。
- 所有引擎方法返回带 `error` 字段的结果对象，不抛异常到工具层。

## 第7章 扩展性设计

- 新增 inventory 工具只需在 `tools/inventory/{tool-name}/index.ts` 实现 `IToolModule`，无需修改注册器。
- 新增物品最大堆叠规则只需在 `stack-size.ts` 中补充关键词集合。
- 装备位映射集中在 `InventoryEngine.ARMOR_SLOT_INDEX` 常量，便于适配 LLSE 版本差异。
