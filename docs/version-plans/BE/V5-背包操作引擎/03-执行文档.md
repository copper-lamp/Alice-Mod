## 第3章 关键实现细节

### 3.1 物品查找

```typescript
// InventoryEngine.findAll(itemName)
const normalized = normalizeName(itemName);
const result: ItemSlot[] = [];

// 1. 主手
pushIfMatch('hand', this.player.selectedSlot ?? 0, this.player.getHand());

// 2. 副手
pushIfMatch('offhand', 0, this.player.getOffHand()?.getItem(0));

// 3. 背包
for (let i = 0; i < inventorySize; i++) {
  pushIfMatch('inventory', i, inventory.getItem(i));
}

// 4. 盔甲
for (let i = 0; i < armorSize; i++) {
  pushIfMatch('armor', i, armor.getItem(i));
}
```

### 3.2 丢弃流程

```typescript
// 1. 定位槽位
const slotRef = itemName ? this.find(itemName) : { source: 'hand', slot: selectedSlot, item: hand };

// 2. 生成掉落物
const dropItem = item.clone();
dropItem.count = dropCount;
const entity = mc.spawnItem(dropItem, dropPos);

// 3. 扣除源槽位
container.removeItem(slotRef.slot, dropCount);
this.persist();
```

### 3.3 装备流程

```typescript
// 1. 卸下旧装备并移回背包
const destItem = destContainer.getItem(destIndex);
if (destItem && !destItem.isNull()) {
  const movedBack = this.moveItemToInventory(destItem, destSource, destIndex);
  if (!movedBack) return { success: false, error: '背包已满，无法卸下当前装备' };
}

// 2. 将源物品移动到目标槽位
const clone = itemToEquip.clone();
clone.count = 1;
destContainer.setItem(destIndex, clone);
sourceContainer.removeItem(sourceRef.slot, 1);
this.persist();
```

### 3.4 容器转移流程

```typescript
// take_from_container 工具
const approachTarget = containerAPI.computeApproachTarget(blockPos, 2);
const moveResult = await aiEngine.moveTo(botName, approachTarget, { timeout: 20000 });
if (!moveResult.success) return { success: false, error: ... };

const container = containerAPI.open(movedBlock);
if (!container) return { success: false, error: ... };

const result = containerAPI.take(container, item_name, count);
containerAPI.close();
if (result.success) BotManager.saveInventory(botName);
```

### 3.5 工具注册器二级扫描

```typescript
// ToolRegistry.scanAndRegister()
const toolDirs = this.discoverToolDirs(normalizedToolsDir);
for (const toolDir of toolDirs) {
  const indexPath = `${toolDir}/index.js`;
  if (!File.exists(indexPath)) continue;
  const mod = require(indexPath);
  const toolModule = mod.default || mod;
  const metadata = toolModule.metadata();
  this.register(metadata.name, metadata, toolModule);
}
```

## 第5章 生产部署步骤

### 5.1 构建产物

```bash
cd packages/adapter-bedrock
pnpm build
```

该命令会：
1. 执行 `tsc` 编译 `src/` 到 `dist/`。
2. 执行 `scripts/copy-to-bds.js`，将 `dist/`、`manifest.json`、`config.json` 复制到 BDS 插件目录。

### 5.2 BDS 侧验证

1. 启动 BDS，确保 `Alices Mod` 插件加载成功。
2. 执行 `/mcagent info`，确认“已注册工具”数量包含 4 个 inventory 工具。
3. 在 Agent Core 连接成功后，查看日志 `[Alice Mod] 已向 Agent Core 注册 N 个工具`。

### 5.3 Agent Core 验证

1. Agent Core 发送 `register_tools` notification 后，工作区工具列表应包含：
   - `drop_item`
   - `equip_item`
   - `take_from_container`
   - `put_to_container`
2. 通过 `tool_call` 调用任意 V5 工具，确认返回 `success` 与正确字段。
