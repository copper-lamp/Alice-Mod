## 第1章 需求

### 1.1 模块定位

V6 是 Adapter Core BE 执行 AI 引擎（C5）的**第四、七子模块合并迭代**，负责把 Agent Core 下发的生存维持意图与方块改造意图转换为游戏内可执行操作。本版本交付 7 个工具：

| 工具 | 说明 | 分类 |
|------|------|:----:|
| `eat` | 吃东西恢复饥饿值，未指定食物时自动选择饱和度最高的食物 | survival |
| `sleep` | 搜索附近床并入睡，支持限时睡眠与指定床位置 | survival |
| `use_item` | 使用物品，支持普通使用、喝药水、投掷三种模式 | survival |
| `mine_block` | 挖掘指定坐标方块，自动选择最合适工具 | block |
| `place_block` | 在指定坐标放置方块 | block |
| `use_block` | 右键使用方块（门、拉杆、工作台等） | block |
| `area_operation` | 对区域逐个方块执行填充/清空/破坏/连锁挖掘 | block |

核心设计原则：

- **area_operation 不使用 `/fill` 等服务端指令**，而是拆解为一系列 `mine_block` / `place_block` 调用，由假人逐格执行。
- **混合决策边界**：常规情况由 BE 自主完成（工具选择、材料选择、路径规划）；遇到死胡同时，BE 先尝试背包内替代方案（换工具、换材料），仍无法解决则返回明确错误原因给 LLM 重新决策。
- **配置化上限**：`area_operation` 单次操作方块上限通过 `config.json` 的 `max_area_operation_blocks` 控制，默认 256。

### 1.4 接口需求

#### 1.4.1 工具层接口

7 个工具均实现 [IToolModule](file:///d:/McAgent/packages/adapter-bedrock/src/registry/tool-module.types.ts#L116) 接口，通过 `metadata()` 返回 JSON Schema，通过 `execute(params, ctx)` 执行。

工具返回统一格式：

```typescript
interface ToolResult {
  success: boolean;
  data?: Record<string, any>;   // 操作结果
  error?: string;               // 失败原因
  duration_ms: number;          // 执行耗时
}
```

每个工具返回需包含：

- **成功/失败原因**
- **最终状态**：实际操作的物品/方块、数量、剩余数量
- **时间消耗**：`duration_ms`
- **资源消耗**：饥饿消耗、工具耐久消耗等

#### 1.4.2 引擎层接口

```typescript
interface SurvivalEngine {
  eat(foodName?: string): EatResult;
  sleep(action: 'sleep' | 'wake', bedPos?: Vec3, maxWaitMs?: number): SleepResult;
  useItem(itemName: string, mode: 'use' | 'drink' | 'throw', target?: Vec3): UseItemResult;
}

interface BlockOperationEngine {
  mineBlock(pos: Vec3): MineResult;
  placeBlock(pos: Vec3, blockName: string): PlaceResult;
  useBlock(pos: Vec3): UseBlockResult;
  areaOperation(mode: AreaMode, from: Vec3, to?: Vec3, blockName?: string, radius?: number): AreaResult;
}

interface ToolSelector {
  /** 根据方块类型与背包内容选择最优工具 */
  selectToolForBlock(blockName: string, inventory: ItemStack[]): ToolRecommendation;
  /** 选择可替代材料（如 oak_planks 不足时用 birch_planks） */
  selectAlternativeBlock(blockName: string, inventory: ItemStack[]): string | null;
}
```

#### 1.4.3 JSON Schema 定义

详见 [BE-01-需求文档.md](../BE-01-需求文档.md#65-生存工具3-个) 第 6.5 节与第 6.6 节。Schema 中 `is_movement` 字段定义如下：

| 工具 | is_movement | 原因 |
|------|:-----------:|------|
| `eat` | false | 原地食用 |
| `use_item` | false | 原地使用 |
| `sleep` | true | 需移动到床边 |
| `mine_block` | true | 需移动到目标方块旁 |
| `place_block` | true | 需移动到目标位置旁 |
| `use_block` | true | 需移动到目标方块旁 |
| `area_operation` | true | 需在区域内移动 |

### 1.5 依赖关系

- **依赖 C3 假人管理**：通过 `BotAccess.getBotPlayer()` 获取 `Player` 对象，操作后调用 `BotManager.saveInventory()` 持久化。
- **依赖 C4 状态上报**：饥饿、装备、位置变化会反映在下一次 `status_report` 中。
- **依赖 C5.2 移动系统**：所有 `is_movement=true` 的工具在执行前调用 `move_to` 靠近目标。
- **依赖 C5.3 背包操作引擎**：`mine_block` 需要切换工具，`place_block` 需要选择方块，`eat` 需要选择食物，均依赖 `InventoryEngine` 的查找/选择能力。
- **依赖 C5.1 寻路系统**：移动系统内部复用 A* 寻路，方块操作中可能需要临时绕行或寻找替代挖掘面。
- **依赖 C8 配置接入**：`max_area_operation_blocks` 从 `config.json` 读取。
- **被 V7 战斗/生物交互依赖**：`set_combat_mode` / `interact_entity` 等工具需要移动到目标附近，部分场景需要破坏障碍物或放置方块。

## 第2章 边界与约束

### 2.1 LLM 与 BE 决策边界

| 场景 | BE 自主行为 | 上报 LLM 条件 |
|------|------------|--------------|
| 食物选择 | 按饱和度评分自动选择 | 背包无食物时 |
| 工具选择 | 按方块类型自动选择最优工具 | 无合适工具且方块不可手挖时 |
| 材料替代 | 同类型方块（如各种木板）自动替换 | 完全无材料时 |
| 床搜索 | 自动搜索半径 32 格内可用床 | 范围内无床或无法入睡时 |
| 区域操作 | 自动分解并逐格执行 | 超上限、遇到不可破坏方块、材料耗尽时 |
| 投掷目标 | 按给定坐标投掷 | 目标超出射程或不可达时 |

### 2.2 配置项

| 配置键 | 默认值 | 说明 |
|--------|:------:|------|
| `survival.eat.max_search_radius` | 0 | 食物只在背包内搜索，不搜索世界 |
| `survival.sleep.max_bed_search_radius` | 32 | 自动搜索床的半径 |
| `survival.sleep.max_wait_ms` | 60000 | sleep 默认最大等待时间 |
| `block.max_area_operation_blocks` | 256 | area_operation 单次最大方块数 |
| `block.tool_durability_threshold` | 5 | 工具耐久低于此值时尝试换工具 |
| `block.allow_alternative_materials` | true | 是否允许同功能材料替代 |

## 第3章 风险与应对

| 风险 | 概率 | 影响 | 应对措施 |
|:----:|:----:|:----:|----------|
| LLSE `SimulatedPlayer` 挖掘/放置 API 不稳定 | 中 | 高 | 使用 `simulateDestroyBlock` / `simulatePlaceBlock`；操作后校验世界状态 |
| `area_operation` 大区域导致 tick 阻塞 | 中 | 中 | 限制单次上限 256；内部每操作 8 格 yield 一 tick |
| 方块放置后状态不一致（phantom 方块） | 中 | 中 | 放置后通过 `world.getBlock()` 二次确认 |
| 工具耐久耗尽导致挖掘中断 | 中 | 中 | 挖掘前检查耐久；低耐久自动切换；无备用工具时返回错误 |
| sleep 被怪物打断或无法入睡 | 中 | 低 | 检查怪物距离；失败时返回明确原因 |
| 食物名称匹配歧义 | 低 | 中 | 精确匹配优先；存在多个候选时返回列表供 LLM 确认 |
| 连锁挖掘（vein）误入巨大矿脉导致超时 | 低 | 高 | vein 模式同样受 `max_area_operation_blocks` 限制 |
