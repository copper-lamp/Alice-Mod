### 1.3 数据流

#### 1.3.1 `eat` 数据流

```
[Agent Core 调用 eat]
        │
        ▼
[eat tool] ──> 解析 food_name
        │
        ▼
[SurvivalEngine.eat(foodName?)]
        │
        ├── 指定 food_name ──> InventoryEngine.find(foodName)
        └── 未指定 ──> FoodSelector.selectBest(inventory)
        │
        ▼
[Player.selectSlot(slot)] ──> 执行吃动画
        │
        ▼
[等待食用完成 / 超时]
        │
        ▼
[返回 ToolResult：item / hunger_restored / saturation_restored]
```

#### 1.3.2 `mine_block` 数据流

```
[Agent Core 调用 mine_block]
        │
        ▼
[mine-block tool] ──> 解析 (x, y, z)
        │
        ▼
[BlockOperationEngine.mineBlock(pos)]
        │
        ├── move_to 靠近目标（distance ≤ 5）
        │
        ▼
[WorldAccess.getBlock(pos)] ──> 获取 blockName
        │
        ▼
[ToolSelector.selectToolForBlock(blockName, inventory)]
        │
        ▼
[InventoryEngine.selectSlot(toolSlot)]
        │
        ▼
[Player.simulateDestroyBlock(pos)] ──> 等待方块破坏
        │
        ▼
[BlockValidator.confirmBroken(pos)]
        │
        ▼
[BotManager.saveInventory(botName)]
        │
        ▼
[返回 ToolResult：block / drops / tool_damage / duration_ms]
```

#### 1.3.3 `area_operation` 数据流

```
[Agent Core 调用 area_operation]
        │
        ▼
[area-operation tool] ──> 解析 mode / from / to / block_name / radius
        │
        ▼
[AreaPlanner.buildQueue(mode, from, to, radius)]
        │
        ├── 检查体积是否超过 max_area_operation_blocks
        ├── fill/clear/break: 按层/按列生成有序坐标队列
        └── vein: BFS 搜索相连同类型方块队列
        │
        ▼
[BlockOperationEngine.areaOperation(queue, mode)]
        │
        ├── 遍历队列中的每个坐标
        ├── fill  ──> placeBlock(pos, blockName)
        ├── clear ──> mineBlock(pos)（忽略掉落物）
        ├── break ──> mineBlock(pos)（汇总掉落物）
        └── vein  ──> mineBlock(pos)
        │
        ▼
[每 8 个操作 yield 一次，避免 tick 阻塞]
        │
        ▼
[返回 ToolResult：mode / total / success / fail / duration_ms]
```

## 第2章 关键技术选型

| 技术点 | 选型 | 原因 |
|--------|------|------|
| 挖掘 API | `SimulatedPlayer.simulateDestroyBlock(pos)` | 最接近玩家真实挖掘行为 |
| 放置 API | `SimulatedPlayer.simulatePlaceBlock(pos)` 或装备物品后右键 | 统一模拟玩家放置 |
| 使用方块 API | `SimulatedPlayer.simulateInteract()` | 模拟右键交互 |
| 工具选择 | 内置方块→工具映射表 + 背包可用性过滤 | 不依赖 LLM 逐个决策 |
| 材料替代 | 同标签（tag）或同大类（planks/stone）匹配 | 提高成功率，减少 LLM 往返 |
| 床搜索 | `WorldAccess.getEntities({ type: 'bed' })` 或世界方块扫描 | 优先使用实体查询，失败则方块扫描 |
| 食物数据 | 内置食物营养值表 | LLSE 不暴露食物属性，需要硬编码常用食物 |
| 进度反馈 | 计数器 + 定时器 | area_operation 需持续反馈，但工具为同步返回 |

### 3.2 `FoodSelector` 评分算法

```typescript
function scoreFood(food: FoodInfo): number {
  return food.hungerRestored * 2 + food.saturationRestored * 4;
}
```

- 饱和度权重更高，因为高饱和度能延长饥饿下降时间。
- 过滤掉会带来负面效果的食物（如腐肉）除非用户显式指定。
- 优先选择堆叠数量少的食物，避免浪费高价值食物（如金苹果）。

### 3.3 `AreaPlanner` 区域遍历顺序

对于 `fill` / `clear` / `break` 模式：

1. 按 Y 轴从低到高（或高到低）分层，避免脚下悬空导致坠落。
2. 每层内按 X-Z 蛇形遍历，减少移动距离。
3. 生成队列后立即检查体积上限。

对于 `vein` 模式：

1. 从 `from` 开始 BFS，只扩展与起点方块类型相同的相邻方块。
2. 限制搜索半径（默认 16）和总体积上限。
3. 按与起点距离升序排列，优先挖掘近处。

### 3.4 工具选择映射表（示例）

| 方块类型 | 首选工具 | 可用手挖 |
|----------|:--------:|:--------:|
| stone, cobblestone | pickaxe | 否（极慢） |
| dirt, grass_block | shovel | 是 |
| log, planks | axe | 是 |
| leaves, wool | shears / hoe | 是 |
| glass | pickaxe (精准采集) | 否 |
| obsidian | diamond_pickaxe | 否 |
| bedrock | - | 不可破坏 |

## 第4章 与周边模块集成

### 4.1 与 V4 移动系统集成

- 所有 `is_movement=true` 的工具在操作前调用 `aiEngine.moveTo()`。
- `mine_block` / `place_block` 目标距离分别按 5 格 / 3 格设定。
- `area_operation` 内部每个坐标操作前会调用 `move_to`（已被单点工具封装）。

### 4.2 与 V5 背包操作引擎集成

- `InventoryEngine.find()` / `findAll()` / `selectSlot()` 被 SurvivalEngine 和 BlockOperationEngine 复用。
- 工具耐久检查通过 `ItemStack.getAuxValue()` 或 `ItemStack.getDamage()` 实现。
- 操作完成后调用 `BotManager.saveInventory()` 持久化。

### 4.3 与 C8 配置接入集成

```typescript
interface V6Config {
  survival: {
    eat: { max_search_radius: number };
    sleep: { max_bed_search_radius: number; max_wait_ms: number };
  };
  block: {
    max_area_operation_blocks: number;
    tool_durability_threshold: number;
    allow_alternative_materials: boolean;
  };
}
```

配置变更后无需重启，工具执行时动态读取。

### 4.4 与 C4 状态上报集成

- `eat` 后更新饥饿/饱和度字段。
- `sleep` 后更新位置（床位置）。
- `mine_block` / `place_block` 后更新背包快照与位置。
- `area_operation` 因耗时较长，状态上报可能覆盖整个操作期间。

## 第5章 错误码设计

| 错误码 | 说明 | 触发场景 |
|:------:|------|----------|
| `NO_FOOD` | 背包中没有可食用物品 | `eat` 未指定或指定食物不存在 |
| `CANNOT_EAT` | 指定物品不可食用 | `eat` 指定了非食物 |
| `NO_BED` | 范围内没有可用床 | `sleep` 自动搜索失败 |
| `MONSTERS_NEARBY` | 床附近有怪物 | `sleep` 条件检查失败 |
| `NOT_SLEEP_TIME` | 当前不是夜晚/雷暴 | `sleep` 条件检查失败 |
| `ITEM_NOT_FOUND` | 背包中没有指定物品 | `use_item` / `place_block` |
| `BLOCK_UNBREAKABLE` | 方块不可破坏 | `mine_block` 遇到基岩等 |
| `NO_SUITABLE_TOOL` | 无合适工具且手挖不可行 | `mine_block` |
| `TOOL_DURABILITY_LOW` | 工具耐久不足 | 挖掘前检查 |
| `PLACE_BLOCKED` | 目标位置无法放置 | `place_block` 无邻接面或被实体占据 |
| `ALTERNATIVE_NOT_FOUND` | 无可用替代材料 | `place_block` 材料不足且替代失败 |
| `AREA_TOO_LARGE` | 区域超过配置上限 | `area_operation` 体积检查 |
| `VEIN_TOO_LARGE` | 连锁矿脉超过上限 | `area_operation` vein 模式 |
| `OPERATION_TIMEOUT` | 操作超时 | 任何长时间工具 |
