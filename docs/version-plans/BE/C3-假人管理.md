## 第1章 需求

### 1.1 模块定位

C3 假人管理模块是 Adapter Core BE 的**核心执行载体**。Agent Core 下发的全部工具调用最终都需要一个或多个游戏内玩家实体来执行。本模块负责：

- 在 BDS 中创建、销毁模拟玩家（SimulatedPlayer）
- 维护多假人实例的生命周期与状态隔离
- 提供假人上线/下线、传送、背包同步、死亡重生等基础能力
- 向工具执行层暴露统一的假人访问接口（`BotAccess`）

### 1.3 非功能需求

|     编号    | 指标       |           要求           | 测量方式                |
| :-------: | -------- | :--------------------: | ------------------- |
| C3-NFR-01 | 最大假人数    |           不限           | 在线假人计数              |
| C3-NFR-02 | 上线耗时     |          < 3s          | 从调用 online 到玩家实体可交互 |
| C3-NFR-03 | 数据持久化成功率 |          ≥ 99%         | 保存/加载操作成功比例         |
| C3-NFR-04 | 内存泄漏     | 72h 运行后假人相关内存增长 < 20MB | 内存监控                |
| C3-NFR-05 | 崩溃隔离     |  单个假人操作异常不导致 BDS 主进程崩溃 | 异常注入测试              |

### 1.4 接口需求

工具执行层通过 `ToolContext.bot` 访问假人能力。当前 `BotAccess` 接口尚未完全定义，C3 阶段至少需提供：

```typescript
interface BotAccess {
  getActiveBot(): BotInstance | null;           // 获取当前工具执行关联的假人
  listBots(): BotInfo[];                        // 列出所有假人
  createBot(config: BotConfig): BotInstance;    // 创建假人
  destroyBot(name: string): boolean;             // 销毁假人
  getBot(name: string): BotInstance | null;      // 按名称获取假人
}
```

### 1.5 依赖关系

- 依赖 C1 TCP 客户端：无（本模块不直接依赖网络层）
- 依赖 C2 工具注册：工具调用时通过 `ToolContext.bot` 使用
- 被 C4 状态上报依赖：`StatusReporter` 需要获取在线假人列表
- 被 C5 执行 AI 引擎依赖：所有工具最终操作游戏内假人实体

### 2.4 持久化设计

假人数据保存到 `plugins/mcagent-adapter-be/data/bots/<name>.json`：

```json
{
  "_version": 1,
  "name": "AliceBot",
  "pos": { "x": 0, "y": 64, "z": 0, "dimid": 0 },
  "direction": 0,
  "gameMode": 5,
  "owner": "",
  "isOnline": false,
  "operation": "",
  "opInterval": 1000,
  "opMaxTimes": 1,
  "opLength": 1000,
  "syncTarget": ""
}
```

背包数据保存到 `plugins/mcagent-adapter-be/data/inventories/<name>.snbt`，使用 NBT → SNBT 序列化，包含 `Inventory` / `Armor` / `Offhand` 三个标签。

### 2.5 与 Agent Core 的交互

C3 不直接向 Agent Core 发送消息，但通过以下方式间接交互：

- `ToolContext.bot` 被注入到工具执行上下文中
- 假人死亡、上线、下线等状态变化可通过 `sendEvent` 推送事件通知
- `StatusReporter` 扫描在线假人并上报状态

### 2.6 关键技术选型

| 技术点   | 选型                          | 原因                |
| ----- | --------------------------- | ----------------- |
| 假人生成  | `mc.spawnSimulatedPlayer()` | LLSE 原生模拟玩家 API   |
| 位置同步  | `FloatPos`                  | LLSE 要求坐标必须为该类型   |
| 寻路    | `pl.simulateNavigateTo()`   | BDS 内置寻路，优先使用原生能力 |
| 背包序列化 | NBT / SNBT                  | 保留完整物品 NBT 数据     |
| 文件存储  | LLSE `File` API             | 与 BDS 运行时兼容       |

### 3.4 关键实现细节

#### 3.4.1 假人上线流程

```typescript
// BotInstance.online()
if (mc.getPlayer(this.name) != null) return true;

const spawnPos = new FloatPos(x, y, z, dimid);
const pl = mc.spawnSimulatedPlayer(this.name, spawnPos);
if (!pl) return false;

this._status = BotStatus.ONLINE;
pl.teleport(spawnPos);       // 二次校正位置
this.applyDirection();
this.applyGameMode();
if (this._operation !== '') this.startOperationLoop();
```

#### 3.4.2 死亡处理流程

```typescript
// BotManager.onPlayerDie()
const bot = BotManager.get(player.realName);
if (!bot) return;

if (bot.increaseDeathCount()) {
  // 20 秒内死亡超过 5 次，自动下线保护
  BotManager.offline(name);
  return;
}

bot.offline(false);
setTimeout(() => bot.online(), 500);
```

#### 3.4.3 背包序列化

```typescript
// BotInstance.serializeInventory()
const compound = new NbtCompound();
const plNbt = pl.getNbt();
compound.setTag('Inventory', plNbt.getTag('Inventory'));
compound.setTag('Armor', plNbt.getTag('Armor'));
compound.setTag('Offhand', plNbt.getTag('Offhand'));
return compound.toSNBT();
```

## 第4章 风险与应对

|                 风险                 |  概率 |  影响 | 应对措施                     |
| :--------------------------------: | :-: | :-: | ------------------------ |
| BDS `spawnSimulatedPlayer` 在高负载下失败 |  中  |  高  | 失败时重试 3 次，仍失败则标记 offline |
|           多假人并发导致 TPS 下降           |  中  |  中  | 限制最大假人数为 3，tick 同步按需启用   |
|         NBT 序列化在版本升级后兼容性问题         |  低  |  中  | 持久化文件加 `_version`，升级时做迁移 |
|            假人死亡重生期间被重复击杀           |  中  |  中  | 重生时临时设置无敌或传送到安全位置        |
|   `ToolContext.bot` 未注入导致工具无法定位假人  |  高  |  高  | V10 必须完成 BotAccess 桥接    |
