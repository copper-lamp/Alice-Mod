## 第1章 概述

### 1.1 模块定位

V6-T 是 **Adapter Core BE 的临时开发/测试支撑模块**，不进入正式版本发布包，用于在 BDS 运行期间通过游戏内 GUI 直接调用已实现的工具，验证工具可用性。它解决以下痛点：

- **无需启动 Agent Core**：断网、AC 未就绪或 LLM 未配置时，仍能快速验证 BE 工具实现。
- **降低测试门槛**：策划、测试人员无需记忆 JSON-RPC 参数，通过表单即可触发工具。
- **快速回归**：V4-V6 共 14 个工具交付后，需要一条命令就能覆盖全部工具的最小可用性验证。

### 1.2 测试范围

本工具覆盖 **V4（移动）、V5（背包）、V6（生存+方块）** 已交付的 14 个执行层工具：

| 版本 | 类别 | 工具 | 说明 |
|:----:|:----:|------|------|
| V4 | 移动 | `move_to` | 移动到坐标/实体/方块 |
| V4 | 移动 | `ride` | 骑乘实体 |
| V4 | 移动 | `dismount` | 下坐骑 |
| V5 | 背包 | `drop_item` | 丢弃物品 |
| V5 | 背包 | `take_from_container` | 从容器取出 |
| V5 | 背包 | `put_to_container` | 放入容器 |
| V5 | 背包 | `equip_item` | 装备/卸下 |
| V6 | 生存 | `eat` | 进食 |
| V6 | 生存 | `sleep` | 睡觉/起床 |
| V6 | 生存 | `use_item` | 使用物品 |
| V6 | 方块 | `mine_block` | 挖掘方块 |
| V6 | 方块 | `place_block` | 放置方块 |
| V6 | 方块 | `use_block` | 使用方块 |
| V6 | 方块 | `area_operation` | 区域方块操作 |

> 注：V6-T 本身不测试 V7-V9 工具，但架构上预留扩展接口，后续可随版本迭代追加菜单项。

### 1.3 使用场景

| 场景 | 使用者 | 目的 |
|------|:------:|------|
| 新工具冒烟测试 | 开发者 | 验证工具注册、参数解析、执行路径无异常 |
| 回归测试 | QA / 开发者 | 每次合并前快速跑一遍 V4-V6 工具 |
| 现场调试 | 运维 | 玩家反馈某功能异常时，在游戏内复现并查看返回结果 |
| 教学演示 | 策划 | 向非技术人员展示假人能做什么 |

## 第2章 功能需求

### 2.1 总体需求

| 编号 | 需求 | 说明 | 优先级 |
|:----:|------|------|:------:|
| V6T-FR-01 | 命令入口 | 玩家输入 `/mcagent test` 打开 GUI 测试主菜单 | P0 |
| V6T-FR-02 | 分类菜单 | 主菜单按「移动 / 背包 / 生存 / 方块 / 报告」分类展示 | P0 |
| V6T-FR-03 | 工具参数表单 | 选中工具后，根据 `ToolMetadata.input_schema` 自动生成 CustomForm | P0 |
| V6T-FR-04 | 参数默认值 | 坐标类参数可默认填充玩家当前位置或准心目标位置 | P1 |
| V6T-FR-05 | 工具执行 | 提交表单后调用 `ToolManager.executeTool()` 执行真实工具逻辑 | P0 |
| V6T-FR-06 | 结果展示 | 执行完成后通过 ModalForm / 聊天文本展示 `ToolResult` | P0 |
| V6T-FR-07 | 一键全部测试 | 提供「全部冒烟」按钮，顺序执行预设的最小参数用例 | P1 |
| V6T-FR-08 | 测试报告 | 汇总最近 N 次测试结果，按通过/失败/超时分类 | P1 |
| V6T-FR-09 | 假人选择 | 若存在多个假人，允许在菜单中选择目标假人 | P0 |
| V6T-FR-10 | 权限控制 | 仅 OP / 白名单玩家可打开测试菜单，防止普通玩家误触发 | P0 |
| V6T-FR-11 | 开关控制 | 通过 `config.json` 的 `gui_test.enabled` 控制是否启用 | P1 |

### 2.2 菜单结构

```
/mcagent test
├── 选择目标假人（多假人时显示）
├── 移动工具
│   ├── move_to
│   ├── ride
│   └── dismount
├── 背包工具
│   ├── drop_item
│   ├── take_from_container
│   ├── put_to_container
│   └── equip_item
├── 生存工具
│   ├── eat
│   ├── sleep
│   └── use_item
├── 方块工具
│   ├── mine_block
│   ├── place_block
│   ├── use_block
│   └── area_operation
├── 全部冒烟测试
├── 查看测试报告
└── 关闭
```

### 2.3 参数表单需求

对于每个工具，表单生成器需解析 `input_schema.properties`，映射为 LLSE CustomForm 控件：

| Schema 类型 | 控件 | 默认值策略 |
|:-----------:|------|-----------|
| `string` + `enum` | Dropdown / StepSlider | 取 `enum` 第一项或 `default` |
| `string`（无 enum） | Input | 空字符串，或按字段名智能填充（如 `block_name` 默认 `stone`） |
| `number`（坐标） | Input（文本） | 玩家当前位置 / 准心指向方块坐标 |
| `number`（其他） | Input / Slider | `default` 或 0 |
| `boolean` | Switch | `default` 或 false |
| `array` | Input（逗号分隔） | 空 |
| `object`（坐标对象） | 3 个 Input（x/y/z） | 玩家当前位置 |

必填参数（`required` 中列出）在表单标题中标注 `*`。

### 2.4 一键冒烟测试用例

| 工具 | 默认参数 | 期望结果 |
|------|----------|----------|
| `move_to` | 目标 = 玩家前方 3 格 | success=true，距离 < 3 |
| `ride` | 目标 = 最近可骑乘实体 | success=true 或无可骑乘实体的明确错误 |
| `dismount` | 无 | success=true 或不在骑乘上的明确错误 |
| `drop_item` | 丢弃主手 1 个 | success=true |
| `take_from_container` | 玩家前方箱子全部取出 | success=true 或容器不存在 |
| `put_to_container` | 玩家前方箱子放入主手全部 | success=true 或容器不存在 |
| `equip_item` | 头部装备主手物品 | success=true |
| `eat` | 自动选择食物 | success=true 或背包无食物 |
| `sleep` | 自动搜索床 | success=true 或不可入睡 |
| `use_item` | 喝主手药水 | success=true 或主手不是药水 |
| `mine_block` | 玩家准心方块 | success=true 或空气/不可破坏 |
| `place_block` | 玩家脚下放置泥土 | success=true 或材料不足 |
| `use_block` | 玩家准心方块 | success=true |
| `area_operation` | clear 3×3×3 区域（以玩家为中心） | success=true 或区域过大 |

## 第3章 非功能需求

| 编号 | 指标 | 要求 | 测量方式 |
|:----:|------|:----:|----------|
| V6T-NFR-01 | 菜单打开耗时 | ≤ 500ms | 从点击命令到主菜单显示 |
| V6T-NFR-02 | 表单生成耗时 | ≤ 200ms | 从选择工具到 CustomForm 显示 |
| V6T-NFR-03 | 工具执行等待提示 | 立即显示 | 提交表单后先发送 "执行中..." 文本 |
| V6T-NFR-04 | 结果返回提示 | 执行完成 1s 内弹出 | 从工具返回到 ModalForm 显示 |
| V6T-NFR-05 | 并发限制 | 同一玩家同时只能执行 1 个工具 | 防止表单状态混乱 |
| V6T-NFR-06 | 崩溃隔离 | 单个工具抛异常不影响 BDS | try/catch 包裹并展示错误 |
| V6T-NFR-07 | TPS 影响 | 菜单操作不占主 tick | 工具执行本身的 tick 占用由被测工具负责 |
| V6T-NFR-08 | 内存占用 | 测试报告最多保留 100 条 | 超过自动清理 |

## 第4章 接口需求

### 4.1 对 ToolManager 的依赖

GUI 测试工具复用已有的工具执行能力：

```typescript
// src/registry/tool-manager.ts
async executeTool(
  name: string,
  params: Record<string, any>,
  ctx: ToolContext,
): Promise<ToolResult>;
```

### 4.2 对 ToolContext 的依赖

通过 `ToolContextImpl` 构造上下文，其中 `activeBotName` 由菜单选择的假人决定：

```typescript
const ctx = new ToolContextImpl({ activeBotName: selectedBot });
```

### 4.3 对 BotManager 的依赖

| 方法 | 用途 |
|------|------|
| `BotManager.list()` | 获取假人列表，用于菜单选择 |
| `BotManager.get(name)` | 检查假人是否存在 |
| `BotManager.getAll()` | 获取在线假人列表 |

### 4.4 配置项

新增 `config.json` 配置段：

| 配置键 | 默认值 | 说明 |
|--------|:------:|------|
| `gui_test.enabled` | `true` | 是否启用 `/mcagent test` GUI |
| `gui_test.require_op` | `true` | 是否要求 OP 权限 |
| `gui_test.allowed_players` | `[]` | 白名单玩家 XUID 列表（require_op=false 时生效） |
| `gui_test.max_report_entries` | `100` | 测试报告最大保留条数 |
| `gui_test.default_target_bot` | `""` | 默认选中的假人名称，空则自动选第一个在线假人 |

## 第5章 边界与约束

### 5.1 功能边界

| 边界 | 说明 |
|------|------|
| 不替代单元测试 | GUI 测试是人工/集成层面的冒烟，不能替代 Vitest 单元测试 |
| 不替代 E2E | 不验证 Agent Core 到 BE 的全链路，只验证 BE 工具本身 |
| 不修改工具逻辑 | 测试器只负责构造参数和调用，不侵入工具内部 |
| 不处理需要复杂前置条件的工具 | 如 `sleep` 需要夜晚/床，失败时展示真实错误即可 |

### 5.2 技术约束

| 约束 | 说明 |
|------|------|
| 依赖 LLSE Form API | 使用 `mc.newSimpleForm()` / `mc.newCustomForm()` |
| 依赖工具已注册 | 工具必须在 `toolRegistry` 中已注册，未注册工具灰显或隐藏 |
| 假人必须在线 | 执行工具前检查目标假人在线，不在线时提示 |
| 临时性 | 模块代码放在 `src/test/` 下，不影响主线工具架构 |

## 第6章 风险与应对

| 风险 | 概率 | 影响 | 应对措施 |
|:----:|:----:|:----:|----------|
| LLSE CustomForm 对复杂对象支持有限 | 中 | 中 | 将 object 类型拆分为多个输入框，不依赖原生对象控件 |
| 玩家恶意调用破坏性工具 | 低 | 高 | 权限控制 + 操作范围提示 + 建议仅在测试服开启 |
| 工具执行超时导致菜单卡死 | 中 | 中 | 异步执行 + "执行中"提示，超时后展示 TIMEOUT |
| 多假人选择后上下文切换错误 | 低 | 中 | 每次执行前通过 `ToolContextImpl` 重新设置 activeBotName |
| schema 变更后表单不兼容 | 中 | 低 | 表单生成器读取运行时 schema，无需硬编码 |
