## 4. 关键实现注意事项

### 4.1 LLSE API 映射

| 高层动作 | LLSE API（实现时以 LeviLamina/LLSE 文档为准） |
|---|---|
| 移动 | `pl.simulateMoveTo(FloatPos)` |
| 疾跑 | `pl.simulateSprint(boolean)` 或 LLSE 提供的等效 API |
| 跳跃 | `pl.simulateJump()` 或等效 API |
| 游泳 | `pl.simulateMoveTo(FloatPos)` + 姿态由游戏自动处理 |
| 攀爬 | 移动到梯子/藤蔓相邻位置，游戏自动攀爬 |
| 破坏方块 | `pl.simulateDestroy()` / `pl.simulateStartDestroyingBlock()` 或等效 API |
| 放置方块 | 通过 `pl.simulateUseItemOnBlock()` 或 LLSE 提供的等效 API |
| 骑乘 | `pl.ride(entity)` 或等效 API |
| 下马 | `pl.dismount()` 或等效 API |
| 使用烟花 | `pl.simulateUseItem()`（手持烟花火箭）或等效 API |

### 4.2 异步寻路实现

```typescript
async findPath(from, to, ctx) {
  const job = new AsyncPathJob(from, to, ctx);
  while (!job.done) {
    job.step(ctx.options.asyncNodesPerTick ?? 64);
    if (!job.done) await yieldToNextTick();
  }
  return job.result;
}
```

`yieldToNextTick` 使用 `setImmediate` 或 Promise.resolve() 让出主线程。

### 4.3 缓存失效策略

- 监听 `onBlockChanged` 事件，判断变化方块所在的 Chunk，失效包含该 Chunk 的缓存条目。
- 玩家维度切换时清空全部缓存。

### 4.4 方块交互执行顺序

1. 切换到工具/方块槽位。
2. 调整朝向（破坏时面对方块，放置时找到合法邻接面）。
3. 执行破坏/放置。
4. 等待动作完成（破坏有硬度耗时，放置即时）。
5. 校验结果，失败则中断。

## 7. 风险与应对

| 风险 | 影响 | 应对措施 |
|---|---|---|
| LLSE 模拟移动 API 精度不足 | 假人卡住/掉线 | 增加位置校验与重试机制 |
| 长距离寻路阻塞 BDS tick | TPS 下降 | 异步分帧 + 快速失败 |
| 方块变化事件高频触发 | 缓存频繁失效 | Chunk 级聚合 + 防抖 |
| 鞘翅飞行动力学复杂 | 摔落/偏离目标 | 第一版仅场景触发，保守起飞/降落评估 |
| 自动挖/搭误伤建筑 | 玩家投诉 | 默认关闭 + 敏感方块黑名单 + 必须 LLM 授权 |
