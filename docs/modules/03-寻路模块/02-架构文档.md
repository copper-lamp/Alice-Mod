## 1. 总体架构

寻路模块采用**规划-执行分离**架构，与项目"LLM 做决策、执行 AI 处理细节"的原则保持一致。

```
┌─────────────────────────────────────────────────────────────┐
│  tools/movement/                                             │
│  move-to.ts · ride.ts · dismount.ts                          │
│  - 参数校验 · 授权透传 · 调用 AIEngine                        │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  AIEngine                                                    │
│  ├─ pathfinding: PathfindingEngine                           │
│  └─ movement: MovementExecutor                               │
└──────────────────────────┬──────────────────────────────────┘
                           │
        ┌──────────────────┴──────────────────┐
        ▼                                       ▼
┌──────────────────────────────┐  ┌──────────────────────────────┐
│  PathfindingEngine           │  │  MovementExecutor            │
│  (规划层)                     │  │  (执行层)                     │
│  ├─ Router                   │  │  ├─ StateMachine             │
│  ├─ GroundPathPlanner        │  │  ├─ ConditionMonitor         │
│  ├─ FlightSegmentPlanner     │  │  ├─ ActionController         │
│  ├─ BlockInteractionPlanner  │  │  ├─ BlockInteractionExecutor │
│  ├─ PathCache                │  │  └─ DynamicReplan            │
│  └─ PathValidator            │  │                              │
└──────────────────────────────┘  └──────────────────────────────┘
```

## 2. 模块职责

| 模块 | 路径 | 职责 |
|---|---|---|
| `PathfindingEngine` | `ai/pathfinding/engine.ts` | 寻路总入口，协调 Router、Planner、Cache、Validator |
| `MovementRouter` | `ai/pathfinding/router.ts` | 根据目标、距离、地形、玩家状态、配置选择移动模式 |
| `GroundPathPlanner` | `ai/pathfinding/ground-planner.ts` | A* 3D 地面/水域/攀爬路径规划 |
| `FlightSegmentPlanner` | `ai/pathfinding/flight-planner.ts` | 场景触发式鞘翅+烟花段规划，接口预留完整飞行动力学 |
| `BlockInteractionPlanner` | `ai/pathfinding/block-interaction-planner.ts` | 死路时生成挖/搭方块方案 |
| `TerrainModifier` | `ai/pathfinding/terrain-modifier.ts` | 为 Planner 提供"假设改造地形后的临时世界视图" |
| `PathCache` | `ai/pathfinding/cache.ts` | LRU + Chunk 级区域缓存 |
| `PathValidator` | `ai/pathfinding/validator.ts` | 执行中验证路径节点是否仍有效 |
| `MovementExecutor` | `ai/movement/executor.ts` | 状态机驱动，调用底层 LLSE API 执行移动 |
| `MovementStateMachine` | `ai/movement/state-machine.ts` | 管理 WALKING/SPRINTING/SWIMMING/ELYTRA 等状态 |
| `ConditionMonitor` | `ai/movement/condition-monitor.ts` | 运行时条件判断与触发 |
| `ActionController` | `ai/movement/action-controller.ts` | 高层动作到 LLSE simulateXxx 的映射 |
| `BlockInteractionExecutor` | `ai/movement/block-interaction-executor.ts` | 挖/搭方块动作执行 |
| `InventoryRequirementChecker` | `ai/movement/inventory-requirement-checker.ts` | 材料与工具预检 |
| `MovementConfig` | `ai/movement/config.ts` | 运行时配置合并（默认值 + config.json + 工具参数） |

## 3. 数据流

### 3.1 寻路阶段

```
move_to(target, options)
   │
   ▼
AIEngine.pathfinding.findPath(start, goal, context)
   │
   ├─ PathCache.get(start, goal) ──命中──┐
   │                                     ▼
   │                              返回 Path
   │
   └─ 未命中
      │
      ▼
   MovementConfig.merge(options) ──► 合并 config.json 默认值
      │
      ▼
   MovementRouter.selectMode(start, goal, ctx)
      │
      ├─ 模式 = GROUND ──► GroundPathPlanner.findPath()
      ├─ 模式 = ELYTRA ──► GroundPathPlanner(到起飞点)
      │                    + FlightSegmentPlanner(起飞-滑翔-降落)
      │                    + GroundPathPlanner(降落点到终点)
      └─ 模式 = RIDE  ──► 预留扩展
      │
      ▼
   若 no_path 且 allowBreak/allowPlace
      │
      ▼
   BlockInteractionPlanner.generateActions()
      │
      ▼
   InventoryRequirementChecker.verify()
      │
      ▼
   PathCache.put(path)
      │
      ▼
   返回 PathResult
```

### 3.2 执行阶段

```
AIEngine.movement.execute(path, context)
   │
   ▼
MovementExecutor.run()
   │
   ├─ 遍历 Path.segments
   │    ├─ 进入 segment 对应 State
   │    ├─ ActionController 驱动 LLSE API
   │    └─ BlockInteractionExecutor 处理挖/搭段
   │
   ├─ ConditionMonitor.tick() 每 100ms
   │    ├─ 遇敌? → pause / retreat
   │    ├─ 低血量? → stop / eat
   │    ├─ 路径被挡? → DynamicReplan
   │    └─ 材料耗尽? → interrupted
   │
   └─ 返回 MoveResult
```

## 4. 关键接口定义

### 4.1 通用类型

```typescript
interface Vec3 { x: number; y: number; z: number; }

interface BoundingBox {
  min: Vec3;
  max: Vec3;
}

interface ItemSnapshot {
  name: string;
  count: number;
  slot: number;
}
```

### 4.2 移动模式

```typescript
type MoveMode =
  | 'walk'
  | 'sprint'
  | 'sprint_jump'
  | 'swim'
  | 'climb'
  | 'elytra'
  | 'ride'
  | 'boat'
  | 'break_block'
  | 'place_block';
```

### 4.3 路径模型

```typescript
interface MoveSegment {
  mode: MoveMode;
  waypoints: Vec3[];
  estimatedCost: number;        // 预估时间（ms）/ 饥饿消耗
  requiredItems?: string[];     // 需要的物品（鞘翅、烟花、方块）
  conditions?: IPathCondition[]; // 进入该段的前提条件
  blockActions?: BlockActionPlan[]; // 挖/搭动作序列
}

interface Path {
  segments: MoveSegment[];
  totalCost: number;
  totalDistance: number;
  isPartial: boolean;          // 是否只找到部分路径
}

interface PathResult {
  success: boolean;
  path?: Path;
  reason?: 'success' | 'no_path' | 'timeout' | 'too_far' | 'cancelled' | 'missing_materials';
  nodeCount: number;
  durationMs: number;
}
```

### 4.4 寻路上下文

```typescript
interface PathContext {
  playerHealth: number;
  playerHunger: number;
  playerPos: Vec3;
  inventory: ItemSnapshot[];
  dimension: string;
  hostileEntities: EntityRef[];
  options: PathOptions;
}

interface PathOptions {
  timeout?: number;
  avoidHostile?: boolean;
  allowBreak?: boolean;
  allowPlace?: boolean;
  allowSwim?: boolean;
  allowElytra?: boolean;
  maxBlocksToBreak?: number;
  maxBlocksToPlace?: number;
  preferredBlock?: string;
  maxRange?: number;
}
```

### 4.5 寻路引擎

```typescript
interface IPathfindingEngine {
  findPath(from: Vec3, to: Vec3, ctx: PathContext): Promise<PathResult>;
  invalidateCache(region?: BoundingBox): void;
}

interface IGroundPathPlanner {
  findPath(from: Vec3, to: Vec3, ctx: PathContext): Promise<PathResult>;
}

interface IFlightSegmentPlanner {
  planSegment(from: Vec3, to: Vec3, ctx: PathContext): Promise<MoveSegment | null>;
}

interface IBlockInteractionPlanner {
  generateActions(
    from: Vec3,
    to: Vec3,
    world: WorldAccess,
    options: BlockInteractionOptions
  ): BlockActionPlan[];
}
```

### 4.6 方块交互

```typescript
interface BlockInteractionOptions {
  allowBreak: boolean;
  allowPlace: boolean;
  maxBlocksToBreak: number;
  maxBlocksToPlace: number;
  preferredBlock?: string;
  unbreakableBlocks: Set<string>;
  protectedBlocks: Set<string>;
}

interface BlockActionPlan {
  type: 'break' | 'place';
  targetPos: Vec3;
  blockName?: string;        // 破坏目标或放置方块名称
  toolName?: string;         // 推荐工具
  face?: Vec3;               // 放置方向
  estimatedTimeMs: number;
  durabilityCost?: number;
}
```

### 4.7 执行器

```typescript
interface IMovementExecutor {
  execute(path: Path, ctx: PathContext): Promise<MoveResult>;
  pause(): void;
  resume(): void;
  stop(): void;
}

interface MoveResult {
  success: boolean;
  finalPos: Vec3;
  distanceMoved: number;
  durationMs: number;
  hungerCost: number;
  reason?: 'success' | 'interrupted' | 'blocked' | 'timeout' | 'died' | 'cancelled' | 'missing_materials';
}

interface IMovementStateMachine {
  getState(): MoveMode;
  transition(to: MoveMode, ctx: PathContext): boolean;
}

interface IConditionMonitor {
  tick(ctx: PathContext): ConditionSignal;
}

type ConditionSignal =
  | 'continue'
  | 'pause'
  | 'retreat'
  | 'replan'
  | 'stop';
```

### 4.8 条件策略

```typescript
interface IPathCondition {
  evaluate(ctx: PathContext, point: Vec3): ConditionEvaluation;
}

interface ConditionEvaluation {
  pass: boolean;
  costMultiplier?: number;   // 规划层代价调整
  reason?: string;
}
```

```json
{
  "pathfinding": {
    "enabled": true,
    "maxRange": 128,
    "cacheSize": 256,
    "asyncNodesPerTick": 64,
    "sprint": {
      "enabled": true,
      "minHunger": 7
    },
    "swim": {
      "enabled": true
    },
    "elytra": {
      "enabled": false,
      "minDistance": 30,
      "requireFirework": true
    },
    "blockInteraction": {
      "allowBreak": false,
      "allowPlace": false,
      "maxBlocksToBreak": 8,
      "maxBlocksToPlace": 8,
      "preferredPlaceBlocks": ["dirt", "cobblestone", "cobblestone_wall"],
      "unbreakableBlocks": ["bedrock", "barrier", "command_block", "end_portal_frame"],
      "protectedBlocks": ["chest", "trapped_chest", "ender_chest", "bed", "furnace"]
```

配置优先级：**工具参数 > config.json > 代码默认值**。

## 6. 高性能设计

### 6.1 路径缓存

- Key：`chunkFrom` + `chunkTo` + `dimension` + `mode` 哈希。
- Value：`Path` 对象 + 最近命中时间。
- 失效：方块变化事件触发区域缓存失效。

### 6.2 异步分帧

- 搜索节点数超过 `asyncNodesPerTick` 时，将剩余工作拆分为 tick 任务。
- 使用 `setImmediate` / 下一帧继续，避免阻塞 BDS 主线程。

### 6.3 增量重规划

- 执行时前方节点被挡，只在局部 16×16×16 区域重新规划。
- 若局部重规划失败，再回退到全局重规划。

### 6.4 快速失败

- 起点终点距离 > `maxRange` 直接返回 `too_far`。
- 目标在不可达区域（如岩浆湖中心）快速剪枝。

## 7. 安全设计

1. **授权链**：工具参数 `allowBreak`/`allowPlace` 必须显式 `true`，且配置层开关开启，才会触发方块交互。
2. **黑名单校验**：`BlockInteractionPlanner` 在生成动作前过滤 `unbreakableBlocks`。
3. **保护名单校验**：`protectedBlocks` 即使授权也不破坏，除非 LLM 额外提供覆盖参数（预留）。
4. **材料预检**：执行前检查背包，不足不进入执行。
5. **中断回退**：任何危险信号触发 `pause`/`stop`，并返回当前位置与原因。
