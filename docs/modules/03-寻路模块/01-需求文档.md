## 1. 概述

### 1.1 目标

为 McAgent Adapter Core（优先 Bedrock 版）提供一套高性能、可配置、支持全自动移动的寻路与执行模块。LLM 通过 `move_to` 等高层工具决定目标，执行 AI 自动完成寻路、避障、状态切换、条件判断以及必要时的方块交互。

### 1.2 范围

| 在范围内 | 不在范围内 |
|---|---|
| 地面/水域/攀爬 A* 3D 寻路 | 跨维度寻路（下界/末地传送到主世界） |
| 疾跑、游泳、攀爬自动切换 | 完整自由飞行动力学（第一版） |
| 场景触发式鞘翅+烟花路径 | 红石机关/门/活塞自动交互 |
| 死路时自动挖方块/搭方块（需 LLM 授权） | 玩家建筑保护/领地插件集成 |
| 运行中遇敌/低血量/摔落等条件判断 | LLM 直接控制每一个跳跃/转向 |

### 1.3 设计原则

1. **LLM 轻决策，执行 AI 重执行**：LLM 只决定目标和策略开关，不干预移动细节。
2. **可配置**：所有安全策略、行为开关、黑名单、性能参数均能通过配置文件或工具参数调整。
3. **可失败、有反馈**：寻路或执行失败时返回明确原因与最终状态。
4. **高性能**：不阻塞 BDS tick，长距离路径可异步分帧计算。
5. **安全优先**：破坏/放置方块必须经 LLM 显式授权，并有默认不可破坏方块黑名单。

## 2. 功能需求

### 2.1 基础寻路

| 编号 | 需求 | 验收标准 |
|---|---|---|
| PF-F-01 | 支持坐标、实体、方块三类目标 | `move_to` 三种 `target_type` 均能正确解析目标位置 |
| PF-F-02 | 支持 A* 3D 地面寻路 | 平坦 50×50 区域路径为直线，50 格内耗时 < 5ms |
| PF-F-03 | 支持 1 格高差自动跳跃 | 路径包含跳跃节点，成功通过 |
| PF-F-04 | 支持 2 格高差处理 | 2 格高墙可绕行或搭方块，不卡死 |
| PF-F-05 | 支持水域游泳 | 河流路径包含游泳段，通过后继续行走 |
| PF-F-06 | 支持梯子/藤蔓攀爬 | 垂直路径可沿攀爬物上升/下降 |
| PF-F-07 | 自动避开岩浆、仙人掌、岩浆块 | 路径不经过危险方块，无法绕行时返回不可达 |
| PF-F-08 | 路径平滑 | 去除共线冗余节点，减少转头次数 |
| PF-F-09 | 最大搜索范围限制 | 直线距离 > 128 格直接返回 `too_far`，耗时 < 5ms |

### 2.2 移动模式与状态切换

| 编号 | 需求 | 验收标准 |
|---|---|---|
| PF-F-10 | 自动选择走路/疾跑 | 平地长距离默认疾跑，临近目标/饥饿低时自动切换走路 |
| PF-F-11 | 疾跑受饥饿与配置约束 | 饥饿 ≤ 6 或配置禁用疾跑时，自动切换走路 |
| PF-F-12 | 游泳自动切换 | 进入水源后自动切换游泳姿态与移动速度 |
| PF-F-13 | 攀爬自动切换 | 接近梯子/藤蔓时自动切换攀爬状态 |
| PF-F-14 | 支持骑乘 | `ride` / `dismount` 工具能正确上下马/船/猪 |

### 2.3 条件判断

#### 2.3.1 规划层条件（影响路径代价）

| 编号 | 需求 | 说明 |
|---|---|---|
| PF-F-15 | 避开敌对生物 | `avoidHostile=true` 时路径绕开 16 格内敌对生物 |
| PF-F-16 | 饥饿感知 | 饥饿低时增加疾跑代价，优先走路 |
| PF-F-17 | 摔落保护 | 避免过大落差（> 3 格），优先绕行或搭方块 |
| PF-F-18 | 危险地形规避 | 岩浆、仙人掌、岩浆块默认 ∞ 代价 |

#### 2.3.2 执行层条件（运行时动态决策）

| 编号 | 需求 | 说明 |
|---|---|---|
| PF-F-19 | 遇敌检测 | 敌对生物进入 8 格内可暂停或撤退 |
| PF-F-20 | 低血量保护 | 血量 ≤ 配置阈值时停止移动并触发逃跑/进食 |
| PF-F-21 | 饥饿保护 | 饥饿 ≤ 配置阈值时停止疾跑 |
| PF-F-22 | 摔落检测 | 检测到即将摔落且高度危险时停止或转向 |
| PF-F-23 | 路径失效检测 | 前方节点被阻挡时触发局部重规划 |

### 2.4 鞘翅与烟花（场景触发式）

| 编号 | 需求 | 验收标准 |
|---|---|---|
| PF-F-24 | 场景触发起飞 | 跨越大型峡谷/山脉/海域时，评估后自动插入鞘翅段 |
| PF-F-25 | 烟花推进 | 滑翔过程中自动使用烟花火箭维持/提升高度 |
| PF-F-26 | 安全降落 | 选择平缓区域降落，避免摔落伤害 |
| PF-F-27 | 材料检查 | 起飞前检查背包是否有鞘翅与烟花火箭，否则不触发 |

### 2.5 死路方块交互（挖/搭）

| 编号 | 需求 | 验收标准 |
|---|---|---|
| PF-F-28 | LLM 授权控制 | `allowBreak` / `allowPlace` 默认 `false`，未授权不破坏/放置 |
| PF-F-29 | 自动挖掘障碍 | 死路时自动规划最少方块破坏方案 |
| PF-F-30 | 自动搭桥/搭梯 | 跨越沟壑或攀登时自动规划最少方块放置方案 |
| PF-F-31 | 材料预检 | 规划阶段检查背包材料与工具，不足时返回 `missing_materials` |
| PF-F-32 | 不可破坏方块过滤 | 基岩、黑曜石等默认不可破坏，可配置扩展黑名单 |
| PF-F-33 | 工具选择 | 挖掘时自动选择背包内最优工具 |
| PF-F-34 | 保护玩家建筑 | 默认不破坏容器、床、门、陷阱箱等敏感方块，可配置 |

| PF-F-37 | 启用/禁用游泳 | `true` | `config.json` / `move_to` 参数 |

## 3. 性能需求

| 编号 | 指标 | 目标值 | 测量方式 |
|---|---|---|---|
| PF-P-01 | 32 格内寻路耗时 | < 10ms | `performance.now()` |
| PF-P-02 | 128 格内寻路耗时 | < 50ms | `performance.now()` |
| PF-P-03 | 路径缓存命中率 | > 30% | 重复移动统计 |
| PF-P-04 | 异步规划不阻塞 tick | 单帧计算节点 ≤ 64 | 帧耗时监控 |
| PF-P-05 | TPS 影响 | ≥ 19 | BDS `/tps` |
| PF-P-06 | 内存占用 | 缓存 + 异步队列 < 20MB | 运行时统计 |

## 4. 接口需求

### 4.1 对工具层暴露

`move_to` 工具的参数已定义于 [移动工具设计](../../tools/06-移动工具设计.md)，本模块需支持以下 `options` 字段：

```typescript
{
  timeout?: number;
  avoidHostile?: boolean;
  allowBreak?: boolean;
  allowPlace?: boolean;
  allowSwim?: boolean;
  allowElytra?: boolean;
  maxBlocksToBreak?: number;
  maxBlocksToPlace?: number;
  preferredBlock?: string;
  maxRange?: number;
  pathfinding?: 'astar';
}
```

### 4.2 对 AIEngine 暴露

```typescript
interface IPathfindingEngine {
  findPath(from: Vec3, to: Vec3, ctx: PathContext): Promise<PathResult>;
  invalidateCache(region?: BoundingBox): void;
}

interface IMovementExecutor {
  execute(path: Path, ctx: PathContext): Promise<MoveResult>;
  pause(): void;
  resume(): void;
  stop(): void;
}
```

## 5. 安全与约束

1. **授权原则**：任何方块破坏/放置必须经 LLM 显式授权。
2. **黑名单原则**：默认不可破坏方块见附录 A，敏感方块保护名单见附录 B。
3. **资源保护**：自动搭桥时优先使用泥土/圆石等廉价方块，避免消耗珍贵材料。
4. **中断安全**：执行过程中玩家被攻击、工具损坏、材料耗尽时，必须安全中断并返回状态。
