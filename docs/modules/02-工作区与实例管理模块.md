## 第一部分：需求文档

### 1.1 模块定位

V3 实现三个紧密关联的子系统，共同构成 Agent Core 的**连接管理层**：

| 子系统 | 职责 | 核心产出 |
|--------|------|----------|
| **工作区管理器** | 管理 Adapter Core 连接的生命周期、状态切换、会话隔离 | `WorkspaceManager` |
| **实例管理器** | 管理 `mcagent_instance.json` 配置文件的导入/校验/CRUD/离线访问 | `InstanceManager` |
| **工具注册** | 接收 Adapter Core 的 `register_tools` 通知，动态维护工作区工具列表 | `ToolRegistry` |

              TCP 连接建立

### 2.3 工作区数据结构

```typescript
interface Workspace {
  id: string;                    // 唯一 ID（UUID）
  name: string;                  // 显示名称（默认 = instance_id）
  instanceId: string;            // Adapter Core 实例 ID
  connectionId: string | null;   // 关联的 TcpConnection ID（null=离线）
  
  // 状态
  state: WorkspaceState;         // offline | connecting | online
  
  // 版本信息
  edition: 'bedrock' | 'java' | null;
  protocolVersion: string | null;
  modVersion: string | null;
  
  // 工具
  tools: ToolDefinition[];       // 已注册的工具列表
  
  // 来源
  source: 'auto' | 'manual';     // 自动创建 or 手动导入
  
  // 会话隔离数据
  session: {
    conversationHistory: Message[];
    memoryContext: Record<string, unknown>;
  };
  
  // 持久化
  persisted: boolean;            // 是否已保存到 InstanceManager
  createdAt: number;
  updatedAt: number;
  lastOnlineAt: number | null;   // 最后在线时间
}
```

### 3.3 集成方式：TcpServer ↔ WorkspaceManager

TcpServer 的事件需要连接到 WorkspaceManager：

```typescript
// 在 Agent Core 初始化时
const tcpServer = new TcpServer(options)
const workspaceManager = new WorkspaceManager()
const instanceManager = new InstanceManager()

// TCP 连接建立 → 创建工作区
tcpServer.on(ServerEvent.ConnectionOpened, ({ clientId }) => {
  const conn = tcpServer.getConnection(clientId)
  // 等待握手完成...
})

// TCP 连接状态变更 → 同步工作区状态
tcpServer.on(ServerEvent.ConnectionStateChange, ({ clientId, newState, prevState }) => {
  const conn = tcpServer.getConnection(clientId)
  if (newState === 'connected') {
    workspaceManager.setWorkspaceOnline(conn.instanceId, clientId)
  }
})

// TCP 连接关闭 → 工作区离线
tcpServer.on(ServerEvent.ConnectionClosed, ({ clientId }) => {
  workspaceManager.setWorkspaceOfflineByConnectionId(clientId)
})

// 接收到 register_tools 通知 → 注册工具
// 通过 TcpConnection.setHandler 设置消息处理器
```

### 3.7 常问问题

#### Q: 工作区和实例的关系是什么？
工作区是运行时概念，对应一个**已连接的 Adapter Core 会话**。实例是配置概念，对应一个**可连接的 Adapter Core 配置**。一个实例可以多次连接（每次创建新的工作区），也可以离线查看（通过实例配置文件）。

#### Q: 工具注册是覆盖还是追加？
**覆盖**。每次 `register_tools` 通知会替换该工作区的全部工具列表。这是因为工具列表变化通常是整体变更（模组重载、版本切换），做差量更新过于复杂且容易出错。

#### Q: 多个工作区可以有相同的 instance_id 吗？
不允许。每个 instance_id 在工作区管理器中是唯一的。如果同一 instance_id 的连接断开后重连，会复用之前的 Workspace 对象。如果同一 instance_id 尝试建立新连接而旧连接仍在，后者会被拒绝。

#### Q: 离线访问支持哪些操作？
- 查看已保存的实例配置列表
- 浏览实例关联的最后已知工具列表
- 编辑实例配置（名称/标签等）
- 不依赖游戏进程运行
