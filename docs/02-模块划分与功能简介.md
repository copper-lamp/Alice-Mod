### 1.2 关键说明

| 要点 | 说明 |
|------|------|
| **两大模块** | Agent Core（桌面端，含 TCP 服务端）、Adapter Core（模组，含 TCP 客户端） |
| **BE/JE 独立实现** | Adapter Core 的基岩版（LeviLamina）和 Java 版（Fabric）代码完全独立，不共享实现代码 |
| **工具是模组的一部分** | 108+ 个工具由 Adapter Core 实现，通过 TCP 注册给 Agent Core，不作为独立模块 |
| **Shared 是库，不是模块** | 类型定义、工具 Schema、常量等作为共享库被 Agent Core 引用，不独立部署；JE 参照定义自行实现等价类型 |
| **TCP 协议不是独立库** | TCP 服务端内置在 Agent Core 中，TCP 客户端由 BE/JE 各自参照通信规范自实现 |

## 第2章 模块A：Agent Core（11个模块）

Agent Core 是基于 Electron + TypeScript/Node.js 的桌面应用，是整个系统的"大脑"。不依赖游戏运行，可独立启动。

### A1 UI 界面模块

| 属性 | 说明 |
|------|------|
| **位置** | Agent Core / 渲染进程 |
| **依赖** | A4 (工作区管理器)、A2 (LLM 调度) |
| **实现** | React + Electron IPC |

主界面包含 6 个面板：

| 面板 | 功能 |
|------|------|
| 主控制面板 | 连接状态、工作区切换、启动/停止 |
| LLM 对话面板 | 消息流历史、LLM 思考过程可视化 |
| 工具调用面板 | 工具调用链、参数、结果实时显示 |
| 游戏状态面板 | 血量/饥饿/位置/装备/背包摘要实时更新 |
| 配置面板 | LLM 模型/TCP 端口/QQ 设置/实例管理 |
| 记忆浏览器 | 按类型/标签检索、查看、编辑、删除记忆 |

### A2 LLM 调度模块

| 属性 | 说明 |
|------|------|
| **位置** | Agent Core / 主进程 |
| **依赖** | A3 (提示词系统)、TCP Bridge (B1)、A4 (工作区管理) |
| **实现** | TypeScript / Provider 插件模式 |

职责：

1. **Provider 管理**：统一接口接入 OpenAI / Claude / Gemini / Ollama 等模型
2. **消息路由**：将用户输入/事件转给 LLM，解析 LLM 返回的工具调用
3. **Function Calling 解析**：提取 LLM 返回的 JSON 工具调用参数，构造 JSON-RPC Batch
4. **结果回注**：将工具执行结果注入下一轮 LLM 上下文
5. **模型切换**：运行时支持模型热切换

**Provider 接口**：

```typescript
interface LLMProvider {
  chat(messages: Message[], tools?: ToolDefinition[]): Promise<LLMResponse>;
  chatStream(messages: Message[], tools?: ToolDefinition[]): AsyncIterable<LLMChunk>;
}
```

| Provider | 说明 |
|----------|------|
| OpenAI | 兼容 OpenAI / Azure OpenAI / DeepSeek 等 API |
| Claude | Anthropic Messages API |
| Gemini | Google Gemini API |
| Ollama | 本地 Ollama 部署模型 |

### A3 提示词系统模块

| 属性 | 说明 |
|------|------|
| **位置** | Agent Core / 主进程 |
| **依赖** | 无（被 A2 调用） |
| **实现** | TypeScript / 模板引擎 |

职责：

1. **系统提示词模板**：角色设定、行为规范、输出格式
2. **基础状态注入**：每轮对话前自动采集并注入基础状态（~150 tokens）
3. **工具列表动态组装**：从工作区管理器获取当前已注册工具，转换为 LLM Function Calling 格式
4. **上下文窗口管理**：超出 ~2048 tokens 时自动截断，支持滑动窗口

### A5 TCP 服务端与实例管理器模块

| 属性 | 说明 |
|------|------|
| **位置** | Agent Core / 主进程 |
| **依赖** | Shared (D1~D4) |
| **实现** | TypeScript / Node.js net 模块 |

职责（TCP 服务端部分）：

1. TCP 端口监听（默认 27541），接受 Adapter Core 连接
2. 通过握手协议验证身份（instance_id + auth_token）
3. 粘包处理（按 `\n` 分隔消息帧）
4. 维护连接池，每个连接映射到一个工作区
5. 将工具调用请求转发给对应 Adapter Core
6. 接收状态上报和事件通知
7. 心跳管理（服务端定时 Ping，默认 10 秒间隔）
8. 超时检测（30 秒无响应断开）
9. 断线自动重连（指数退避 1s/2s/4s/8s/16s，最多 5 次）
10. 重连成功后恢复工具列表注册

职责（实例管理器部分）：

1. **JSON 入口文件管理**：
   - 用户通过文件选择对话框导入 `mcagent_instance.json`
   - 校验文件格式和 schema_version
   - 防止重复添加（按 instance_id 检查）
2. **离线访问**：
   - 读取 JSON 文件中的 database 路径
   - 连接 SQLite 数据库，浏览/编辑记忆和配置
   - 不启动游戏也可修改 LLM 模型、工具权限等
3. **实例列表管理**：
   - 添加/删除/编辑实例
   - 显示在线/离线状态

## 第3章 模块B：Adapter Core（基岩版 / Java 版）

Adapter Core 是运行在游戏服务器内的模组/插件，负责执行所有游戏内操作。

**BE/JE 独立实现**：基岩版（LeviLamina 插件）和 Java 版（Fabric 模组）代码完全独立，不共享实现代码。但模块划分一致，接口定义一致。

### 4.1 模块清单

| 编号 | 模块 | 说明 | BE/JE 代码复用 |
|------|------|------|:---:|
| C1 | TCP 客户端模块 | 连接 Agent Core，通信 | ❌ 独立实现 |
| C2 | 工具注册模块 | 注册 108+ 工具到 Agent Core | ❌ 独立实现 |
| C3 | 假人管理模块 | 创建/销毁游戏假人 | ❌ 独立实现 |
| C4 | 状态上报模块 | 周期性上报 + 事件通知 | ❌ 独立实现 |
| C5 | 执行AI 引擎 | 7 个子模块 | ❌ 独立实现 |
| C6 | 数据持久化模块 | SQLite / Chroma / 文件管理 | ❌ 独立实现 |
| C7 | JSON 入口生成模块 | 生成 mcagent_instance.json | ❌ 独立实现 |
| C8 | 配置接入模块 | 游戏内配置接口 | ❌ 独立实现 |

### C5 执行AI 引擎模块

此为 Adapter Core 的核心模块，包含 7 个子模块，**BE/JE 各自独立实现全部逻辑**。

```
执行AI 引擎（C5）
├── C5.1 寻路系统 (A* 3D)
├── C5.2 移动系统 (坐标/视角)
├── C5.3 背包操作引擎 (物品/装备)
├── C5.4 方块操作引擎 (挖掘/放置/区域)
├── C5.5 战斗系统 (行为树)
├── C5.6 生物交互引擎 (喂食/繁殖/交易)
└── C5.7 生存操作引擎 (饮食/睡眠/使用)
```

| 移动状态 | 走路/疾跑/潜行切换 |

## 第4章 共享库：Shared

Shared 是纯 TypeScript 类型定义库，不包含运行时逻辑，不独立部署，被 Agent Core 引用。

| 编号 | 模块 | 说明 |
|------|------|------|
| D1 | 类型定义 | JSON-RPC 2.0 核心类型（Request/Response/Notification/Batch） |
| D2 | 工具 Schema | ToolDefinition、ToolCategory 等类型定义 |
| D3 | 协议工具 | 消息校验、错误码查询、版本比较、Batch 检查 |
| D4 | 常量与枚举 | 协议常量、错误码枚举、事件类型枚举 |

> **注意**：Shared 库目前仅用于 Agent Core 和 TCP Bridge。Adapter Core 的 Java 版（Fabric）不直接引用 TypeScript 的 Shared 库，而是参照其中的定义用 Java 类型体系自行实现等价接口。

### D1 类型定义模块

interface JsonRpcRequest {
  jsonrpc: '2.0';

}

}
