## 第1章 概述

### 1.1 设计目标

任务系统是 McAgent 的"执行引擎"，使 LLM 能够创建、分解、调度和监控任务，实现从高层目标到具体操作的完整闭环。其设计遵循以下原则：

| 原则 | 说明 |
|------|------|
| **统一模型** | 所有任务共享统一的数据模型和状态机，4 种任务类型通过配置区分 |
| **分层调度** | 调度器负责任务排队和依赖管理，执行器负责具体执行 |
| **可重试** | 失败任务支持指数退避重试，不丢失 |
| **可持久化** | 任务状态存储在 SQLite，进程重启后恢复 |
| **LLM 可管理** | LLM 通过工具创建/查询/更新/取消任务，全程可控 |
| **事件驱动** | 支持条件触发（状态/时间/事件/表达式） |

### 1.3 数据流

```
LLM 创建任务:
  task_create({ name: "挖10个钻石", type: "simple", action: ... })
    → TaskManager.create()
      → 验证参数 → 写入 SQLite → 加入调度队列
    → 返回 task_id

任务执行:
  TaskScheduler 循环检查队列
    → 检查依赖是否满足
    → 交给 TaskExecutor.execute()
      → switch(task.type):
          simple → 调用 action 对应的工具
          composite → 依次执行子任务
          loop → 按循环配置重复执行
          conditional → 等待条件满足后执行
    → 成功: 标记 completed, 触发后续任务
    → 失败: 检查重试配置, 指数退避重试

LLM 监控:
  task_query({ status: "running" })
    → TaskManager.query()
    → 返回任务列表和进度
```

### 2.2 任务类型

```typescript
type TaskType =
  /** 简单任务 — 单个一次性操作，直接调用一个工具 */
  | 'simple'
  /** 复合任务 — 包含多个子任务，按依赖关系依次执行 */
  | 'composite'
  /** 循环任务 — 按次数/间隔/条件重复执行 */
  | 'loop'
  /** 条件任务 — 等待指定条件满足后执行 */
  | 'conditional';
```

### 2.3 任务状态

```typescript
type TaskStatus =
  | 'pending'       // 等待执行
  | 'running'       // 执行中
  | 'paused'        // 已暂停
  | 'completed'     // 已完成
  | 'failed'        // 已失败（重试耗尽）
  | 'cancelled';    // 已取消
```

**状态机**：

```
                   ┌──────────────────────────────┐
                   │         pending              │
                   └──┬──────────┬──────────┬─────┘
                      │          │          │
              调度执行 │   暂停   │   取消   │
                      ▼          ▼          ▼
               ┌──────────┐ ┌───────┐ ┌──────────┐
               │ running  │ │paused │ │cancelled │
               └──┬───┬───┘ └──┬────┘ └──────────┘
                  │   │      恢复
              完成 │   │失败    │
                  ▼   ▼       ▼
           ┌──────────┐ ┌───────┐
           │completed │ │failed │
           └──────────┘ └───────┘
               ↑
          failed 但重试后 → pending
```

### 4.3 调度事件

```
调度器生命周期:

  启动(start) →
    调度循环开始 →
      [pollIntervalMs 间隔轮询]
        → 检查队列 → 有任务 →
          检查并发数 → 未达上限 →
            检查依赖 → 满足 →
              执行任务 (emit 'task_started')
                → 完成 (emit 'task_completed')
                  → 检查后续依赖任务 (emit 'task_dependency_met')
                → 失败 (emit 'task_failed')
                  → 检查重试 → 可重试 → 放回队列
                  → 不可重试 → 标记 failed
        → 无任务 → (emit 'queue_empty')
      → 继续循环
  停止(stop) →
    调度循环结束
```

### 5.1 引擎接口

```typescript
class TaskExecutor {
  /**
   * 根据任务类型选择对应的执行器
   */
  async execute(task: Task, context: ExecutionContext): Promise<TaskResult> {
    switch (task.type) {
      case 'simple':
        return this.executeSimple(task, context);
      case 'composite':
        return this.executeComposite(task, context);
      case 'loop':

      case 'conditional':
```

```typescript
class SimpleTaskExecutor implements TaskRunner {
  /**
   * 简单任务执行：
   * 1. 从 task.action 获取 toolName 和 parameters
   * 2. 通过 context.callTool() 调用工具
   * 3. 收集执行结果
   * 4. 如果工具失败且 task 配置了重试，抛出异常交给调度器处理
   */
  async execute(task: Task, ctx: ExecutionContext): Promise<TaskResult> {
    const startTime = Date.now();

    if (!task.action) {
      return { success: false, error: '简单任务必须包含 action', durationMs: 0 };
    }

    try {
      const result = await ctx.callTool(task.action.toolName, task.action.parameters);
      return {
        success: true,
        data: result,
        durationMs: Date.now() - startTime,
      };
    } catch (e: any) {
      return {
        success: false,
        error: e.message,
        durationMs: Date.now() - startTime,
      };
    }
  }
}

class CompositeTaskExecutor implements TaskRunner {
  /**
   * 复合任务执行：
   * 1. 按 subtaskIds 顺序执行子任务
   * 2. 每完成一个子任务，更新进度
   * 3. 如果某个子任务失败，复合任务也标记为失败
   * 4. 子任务的依赖关系由调度器管理
   */
  async execute(task: Task, ctx: ExecutionContext): Promise<TaskResult> {
    if (!task.subtaskIds || task.subtaskIds.length === 0) {
      return { success: false, error: '复合任务必须包含子任务', durationMs: 0 };
    }

    const results: any[] = [];
    const startTime = Date.now();

    for (let i = 0; i < task.subtaskIds.length; i++) {
      const subtaskId = task.subtaskIds[i];
      const subtask = await ctx.getSubTask(subtaskId);

      if (!subtask) {
        return {
          success: false,
          error: `子任务 ${subtaskId} 不存在`,
          durationMs: Date.now() - startTime,
        };
      }

      // 将子任务加入调度队列
      // 复合任务本身等待子任务全部完成后才标记完成
      // 实际由调度器管理，这里简化逻辑
    }

    return {
      success: true,
      data: results,
      durationMs: Date.now() - startTime,
    };
  }

  async estimateDuration(task: Task): Promise<number> {
    if (!task.subtaskIds) return 0;
    let total = 0;
    for (const id of task.subtaskIds) {
      const subtask = await getTaskById(id);
      if (subtask?.timeout) total += subtask.timeout;
    }
    return total;
  }
}

class LoopTaskExecutor implements TaskRunner {
  /**
   * 循环任务执行：
   * 1. 按 loopConfig.mode 决定循环方式
   * 2. count 模式：执行 N 次
   * 3. interval 模式：每间隔 N 秒执行一次
   * 4. condition 模式：条件为 true 时执行
   * 5. 每次循环后检查中止信号
   * 6. 达到 maxIterations 后自动停止
   */
  async execute(task: Task, ctx: ExecutionContext): Promise<TaskResult> {
    if (!task.loopConfig || !task.action) {
      return { success: false, error: '循环任务必须包含 loopConfig 和 action', durationMs: 0 };
    }

    const { mode, count, interval, condition, maxIterations = 100 } = task.loopConfig;
    const results: any[] = [];
    const startTime = Date.now();
    let iteration = 0;

    while (iteration < maxIterations) {
      // 检查中止信号
      if (ctx.abortSignal?.aborted) {
        return {
          success: false,
          error: '任务被中止',
          durationMs: Date.now() - startTime,
        };
      }

      // 检查循环条件
      if (mode === 'count' && iteration >= (count ?? 1)) break;
      if (mode === 'condition' && !this.evaluateCondition(condition!, task)) break;
      // mode === 'interval' 永真，由外部控制

      // 执行操作
      const result = await ctx.callTool(task.action.toolName, task.action.parameters);
      results.push(result);
      iteration++;

      // 更新进度
      if (mode === 'count' && count) {
        await ctx.updateProgress(task.id, Math.min((iteration / count) * 100, 100));
      }

      // 间隔等待
      if (mode === 'interval' && interval) {
        await new Promise(resolve => setTimeout(resolve, interval * 1000));
      }
    }

    return {
      success: true,
      data: results,
      durationMs: Date.now() - startTime,
    };
  }

  private evaluateCondition(condition: string, task: Task): boolean {
    // 简单条件求值，实际实现应使用安全沙箱
    // 支持 task.progress, task.retryCount 等变量
    try {
      return new Function('task', `return ${condition}`)(task);
    } catch {
      return true; // 条件解析失败时继续执行
    }
  }
}

class ConditionalTaskExecutor implements TaskRunner {
  /**
   * 条件任务执行：
   * 1. 先评估条件是否满足
   * 2. 如果已满足，直接执行 action
   * 3. 如果不满足，轮询等待条件变化
   * 4. 等待超时后（timeout），标记为失败
   * 5. 条件满足后执行一次 action 即完成
   *
   * 条件求值通过事件系统实现，避免忙等：
   * - type='state': 订阅状态变化事件
   * - type='time': 使用定时器
   * - type='event': 直接监听事件
   * - type='expression': 调用表达式引擎
   */
  async execute(task: Task, ctx: ExecutionContext): Promise<TaskResult> {
    if (!task.condition || !task.action) {
      return { success: false, error: '条件任务必须包含 condition 和 action', durationMs: 0 };
    }

    const startTime = Date.now();

    // 等待条件满足（超时由调度器处理）
    const conditionMet = await this.waitForCondition(task.condition, task.timeout);

    if (!conditionMet) {
      return {
        success: false,
        error: '条件超时未满足',
        durationMs: Date.now() - startTime,
      };
    }

    // 执行操作
    const result = await ctx.callTool(task.action.toolName, task.action.parameters);

    return {
      success: true,
      data: result,
      durationMs: Date.now() - startTime,
    };
  }

  private async waitForCondition(condition: Condition, timeout?: number): Promise<boolean> {
    switch (condition.type) {
      case 'time': {
        const targetTime = condition.value as number;
        const delay = Math.max(0, targetTime * 1000 - Date.now());
        await new Promise(resolve => setTimeout(resolve, delay));
        return true;
      }
      case 'event':
        // 通过事件总线监听，超时后返回 false
        return this.waitForEvent(condition.value as string, timeout);
      case 'state':
      case 'expression':
        // 轮询求值（带间隔）
        return this.pollCondition(condition.value as string, timeout);
    }
  }

  private async pollCondition(expression: string, timeout?: number): Promise<boolean> {
    const deadline = timeout ? Date.now() + timeout * 1000 : Infinity;
    while (Date.now() < deadline) {
      try {
        const result = new Function('return ' + expression)();
        if (result) return true;
      } catch {}
      await new Promise(resolve => setTimeout(resolve, 1000));
    }
    return false;
  }

  private async waitForEvent(eventName: string, timeout?: number): Promise<boolean> {
    return new Promise(resolve => {
      const timer = timeout
        ? setTimeout(() => resolve(false), timeout * 1000)
        : null;
      eventBus.once(eventName, () => {
        timer && clearTimeout(timer);
        resolve(true);
      });
    });
  }
}
```

## 第9章 BE/JE 实现差异

任务系统在 Agent Core 中实现（TypeScript），BE 和 JE 共享同一套代码。

| 维度 | 说明 |
|------|------|
| TaskManager | Agent Core 侧，TypeScript，BE/JE 共享 |
| 调度器 | Agent Core 侧，TypeScript，BE/JE 共享 |
| 执行引擎 | Agent Core 侧，TypeScript，BE/JE 共享 |
| 存储 | Agent Core 侧 SQLite，BE/JE 共享读写 |
| 工具接口 | Agent Core 侧注册，BE/JE 通过 TCP 调用其他工具 |

> **注意**：任务系统本身不直接执行游戏操作。它通过 TaskExecutor 调用其他工具模块（移动/背包/方块等），这些工具模块在 Adapter Core 中实现。因此任务系统的 BE/JE 差异完全取决于被调用工具的差异，任务系统本身无差异。

```
                                      callTool() → 实际工具
                                            │
                                    ┌───────┴───────┐
                                    │  成功 / 失败    │
                                    └───────┬───────┘
                                            │
                                   调度器处理结果
                                    ├─ 成功 → 触发后续依赖任务
                                    └─ 失败 → 检查重试 → 放回队列
```
