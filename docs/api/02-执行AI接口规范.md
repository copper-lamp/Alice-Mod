## 第1章 概述

### 1.1 设计目标

执行AI 是 Adapter Core 的核心，负责将 LLM 的工具调用转化为真实的游戏操作。其接口规范设计遵循以下原则：

| 原则 | 说明 |
|------|------|
| **模块化** | 每个工具是一个独立模块，可单独开发、测试、部署 |
| **可扩展** | 用户可自定义工具，无需修改核心代码 |
| **自动注册** | 模组自动扫描工具模块目录，无需手动配置 |
| **统一接口** | 所有工具遵循相同接口规范，BE/JE 各自实现 |
| **假人驱动** | 工具直接控制假人行为，假人是执行AI 的唯一操作对象 |

### 1.2 架构

```
                        LLM（大脑）
                           │ 工具调用 (JSON-RPC)
                           ▼
┌──────────────────────────────────────────────────┐

│  │  └─────────┘ └─────────┘ └─────────┘   │    │
```

### 1.3 关键概念

| 概念 | 说明 |
|------|------|
| **工具（Tool）** | 一个自包含的功能模块，接受参数、执行操作、返回结果 |
| **工具注册表** | 管理所有已注册工具的容器，支持自动发现和热加载 |
| **执行AI 引擎** | 底层能力集合（寻路/移动/背包等），由工具调用 |
| **假人** | 执行AI 的唯一操作对象，一个工具实例对应一个假人 |
| **执行上下文** | 工具执行时传入的环境，包含假人引用、游戏状态等 |

## 第2章 工具模块接口

### 2.1 概述

每个工具是一个独立的模块，通过统一的接口定义。模块化设计使得工具可以：

- 独立开发、测试、部署
- 热加载（不重启模组即可添加/更新工具）
- 跨项目复用（同一工具模块可在不同服务器使用）

### 2.2 BE（基岩版）工具接口

#### TypeScript 接口定义

```typescript
/**
 * 工具模块接口
 * 所有工具必须实现此接口
 */
interface IToolModule {
  /** 工具元数据 */
  readonly metadata: ToolMetadata;

  /**
   * 工具执行函数
   * @param params 调用参数，由 input_schema 定义
   * @param ctx 执行上下文，包含假人引用、游戏状态等
   * @returns 执行结果
   */
  execute(params: Record<string, any>, ctx: ExecutionContext): Promise<ToolResult>;
}

/** 工具元数据 */
interface ToolMetadata {
  /** 工具名称，全局唯一，snake_case */
  name: string;

  /** 工具描述，LLM 理解用途 */
  description: string;

  /** 工具分类 */
  category: ToolCategory;

  /** 参数 JSON Schema */
  input_schema: JSONSchema;

  /** 返回值 JSON Schema（可选，建议提供） */
  output_schema?: JSONSchema;

  /** 执行配置 */
  config?: ToolConfig;
}

/** 工具分类 */
type ToolCategory =
  | 'perception' | 'movement' | 'inventory' | 'entity'
  | 'survival' | 'block' | 'chat' | 'qq' | 'memory' | 'task';

/** 执行配置 */
interface ToolConfig {
  /** 默认超时（毫秒） */
  timeout_default_ms?: number;
  /** 最大超时 */
  timeout_max_ms?: number;
  /** 是否为移动类工具（影响依赖分析） */
  is_movement?: boolean;
  /** 是否异步执行（无需等待结果） */
  is_async?: boolean;
  /** 是否需要假人（true=需要假人实例，false=纯逻辑工具） */
  requires_bot?: boolean;
}

/** 执行上下文 */
interface ExecutionContext {
  /** 假人实例（工具操作的目标） */
  bot: Bot;
  /** 游戏世界状态快照 */
  world: WorldSnapshot;
  /** 游戏 API */
  api: GameAPI;
  /** 日志记录器 */
  logger: Logger;
  /** 信号量（用于中断执行） */
  abortSignal: AbortSignal;
}

/** 执行结果 */
interface ToolResult {
  /** 是否成功 */
  success: boolean;
  /** 返回数据 */
  data?: Record<string, any>;
  /** 错误信息 */
  error?: string;
  /** 执行耗时（毫秒） */
  duration_ms?: number;
}
```

#### 基础工具模块实现示例

```typescript
// tools/movement/move_to.ts
import { IToolModule, ToolMetadata, ExecutionContext, ToolResult } from '../core/tool-interface';

export class MoveToTool implements IToolModule {
  readonly metadata: ToolMetadata = {
    name: 'move_to',
    description: '走到指定坐标，自动寻路',
    category: 'movement',
    input_schema: {
      type: 'object',
      properties: {
        x: { type: 'number', description: '目标 X 坐标' },
        y: { type: 'number', description: '目标 Y 坐标' },
        z: { type: 'number', description: '目标 Z 坐标' },
        dimension: {
          type: 'string',
          enum: ['overworld', 'nether', 'end'],
          default: 'overworld',
        },
      },
      required: ['x', 'y', 'z'],
    },
    output_schema: {
      type: 'object',
      properties: {
        position: {
          type: 'object',
          properties: {
            x: { type: 'number' },
            y: { type: 'number' },
            z: { type: 'number' },
          },
        },
        duration_ms: { type: 'number' },
        hunger_cost: { type: 'number' },
      },
    },
    config: {
      timeout_default_ms: 30000,
      timeout_max_ms: 120000,
      is_movement: true,
      requires_bot: true,
    },
  };

  async execute(params: Record<string, any>, ctx: ExecutionContext): Promise<ToolResult> {
    const { x, y, z, dimension } = params;
    const startTime = Date.now();

    try {
      // 调用执行AI 引擎的寻路系统
      const path = await ctx.api.pathfinding.findPath(ctx.bot.position, { x, y, z }, dimension);

      // 沿路径移动
      const result = await ctx.bot.movement.followPath(path, {
        timeout: this.metadata.config?.timeout_default_ms ?? 30000,
        signal: ctx.abortSignal,
      });

      return {
        success: true,
        data: {
          position: result.position,
          duration_ms: Date.now() - startTime,
          hunger_cost: result.hungerCost,
        },
        duration_ms: Date.now() - startTime,
      };
    } catch (error) {
      return {
        success: false,
        error: `移动失败: ${error.message}`,
        data: {
          position: ctx.bot.position,
          duration_ms: Date.now() - startTime,
        },
        duration_ms: Date.now() - startTime,
      };
    }
  }
}
```

### 2.3 JE（Java 版）工具接口

#### Java 接口定义

```java
/**
 * 工具模块接口
 */
public interface IToolModule {
    /**
     * 获取工具元数据
     */
    ToolMetadata getMetadata();

    /**
     * 执行工具
     * @param params 调用参数
     * @param ctx 执行上下文
     * @return 执行结果

/** 工具元数据 */
public class ToolMetadata {

}

/** 工具分类 */
public enum ToolCategory {
    PERCEPTION, MOVEMENT, INVENTORY, ENTITY,
    SURVIVAL, BLOCK, CHAT, QQ, MEMORY, TASK
}

/** 执行上下文 */
public class ExecutionContext {
    private Bot bot;              // 假人实例
    private WorldSnapshot world;  // 世界状态

    private Logger logger;        // 日志记录器
    private AtomicBoolean abortFlag;  // 中断信号
```

#### Java 工具模块示例

```java
// com.mcagent.tools.movement.MoveToTool.java
@ToolModule(
    name = "move_to",
    description = "走到指定坐标，自动寻路",
    category = ToolCategory.MOVEMENT
)
public class MoveToTool implements IToolModule {

    @Override
    public ToolMetadata getMetadata() {
        return ToolMetadata.builder()
            .name("move_to")
            .description("走到指定坐标，自动寻路")
            .category(ToolCategory.MOVEMENT)
            .inputSchema(Map.of(
                "type", "object",
                "properties", Map.of(
                    "x", Map.of("type", "number", "description", "目标 X 坐标"),
                    "y", Map.of("type", "number", "description", "目标 Y 坐标"),
                    "z", Map.of("type", "number", "description", "目标 Z 坐标")
                ),
                "required", List.of("x", "y", "z")
            ))
            .config(ToolConfig.builder()
                .timeoutDefaultMs(30000)
                .timeoutMaxMs(120000)
                .isMovement(true)
                .build())
            .build();
    }

    @Override
    public CompletableFuture<ToolResult> execute(Map<String, Object> params, ExecutionContext ctx) {
        long startTime = System.currentTimeMillis();
        double x = (double) params.get("x");
        double y = (double) params.get("y");
        double z = (double) params.get("z");

        return ctx.getApi().getPathfinding().findPath(ctx.getBot().getPosition(), new Vec3(x, y, z))
            .thenCompose(path -> ctx.getBot().getMovement().followPath(path))
            .thenApply(result -> ToolResult.success()
                .withData("position", result.getPosition())
                .withData("duration_ms", System.currentTimeMillis() - startTime)
                .withData("hunger_cost", result.getHungerCost())
                .build())
            .exceptionally(err -> ToolResult.failure()
                .withError("移动失败: " + err.getMessage())
                .build());
    }
}
```

## 第3章 工具注册机制

### 3.1 自动发现

工具注册表（ToolRegistry）在模组启动时自动扫描指定目录，发现并加载所有工具模块。

#### BE（基岩版）扫描规则

```
插件目录/
├── tools/                          ← 工具根目录（自动扫描）
│   ├── index.ts                    ← 可选：批量导出所有工具
│   │
│   ├── perception/                 ← 按分类分目录
│   │   ├── look_around.ts
│   │   ├── look_at_block.ts
│   │   └── ...
│   │
│   ├── movement/
│   │   ├── move_to.ts
│   │   ├── move_direction.ts
│   │   └── ...
│   │
│   ├── custom/                     ← 用户自定义工具
│   │   ├── my_custom_tool.ts       ← 用户自己写的工具
│   │   └── ...
│   │
│   └── plugins/                    ← 第三方插件工具
│       ├── farming/                ← 种植插件
│       │   └── plant_crop.ts
│       └── ...
│
└── mcagent_instance.json           ← 生成的入口文件
```

**扫描流程**：

```
模组启动
    ↓
ToolRegistry.init()
    ├── 扫描 tools/ 目录下所有 .ts/.js 文件
    ├── 对每个文件：
    │   ├── 动态导入模块
    │   ├── 检查是否实现 IToolModule 接口
    │   ├── 验证 metadata 完整性（name/description/input_schema）
    │   ├── 检查 name 是否唯一（冲突则报错）
    │   └── 注册到工具注册表
    ├── 生成工具列表 → 通过 TCP 注册到 Agent Core
    └── 启动文件监听（热加载）
```

**热加载机制**：

```
文件监听器（fs.watch）
    ├── 文件变更 → 重新加载模块
    ├── 新增文件 → 扫描并注册
    ├── 删除文件 → 注销工具
    └── 变更后 → 发送 update_tools 到 Agent Core
```

#### JE（Java 版）扫描规则

```
mods/
├── mcagent-core.jar                  ← 核心模组（内置工具）
│
├── mcagent-custom-tools.jar          ← 用户自定义工具包
│   └── com/mcagent/custom/
│       ├── MyCustomTool.class
│       └── ...
│
└── mcagent-farming-plugin.jar        ← 第三方插件
    └── com/mcagent/plugins/farming/
        ├── PlantCropTool.class
        └── ...
```

**扫描流程**：

```
模组加载
    ↓
ToolRegistry.init()
    ├── 通过注解扫描（@ToolModule）
    │   ├── 扫描当前 JAR 内的注解
    │   └── 扫描依赖 JAR 中的注解（SPI 机制）
    ├── 对每个注解类：
    │   ├── 实例化并验证
    │   ├── 检查 name 唯一性
    │   └── 注册到工具注册表
    ├── 生成工具列表 → 通过 TCP 注册到 Agent Core
    └── （Java 版热加载待定，Fabric 热加载支持有限）
```

### 3.2 工具注册表 API

```typescript
// BE（TypeScript）工具注册表
class ToolRegistry {
  /** 注册单个工具 */
  register(tool: IToolModule): void;

  /** 批量注册工具 */
  registerAll(tools: IToolModule[]): void;

  /** 注销工具 */
  unregister(toolName: string): void;

  /** 获取工具 */
  get(toolName: string): IToolModule | undefined;

  /** 获取所有工具 */
  getAll(): IToolModule[];

  /** 按分类获取 */
  getByCategory(category: ToolCategory): IToolModule[];

  /** 执行工具 */
  async execute(toolName: string, params: Record<string, any>, ctx: ExecutionContext): Promise<ToolResult>;

  /** 扫描目录并注册 */
  async scanDirectory(dirPath: string): Promise<IToolModule[]>;

  /** 生成 JSON Schema 工具列表（用于 TCP 注册） */
  generateToolList(): ToolDefinition[];
}

// 使用示例
const registry = new ToolRegistry();

// 手动注册
registry.register(new MoveToTool());

// 自动扫描目录
await registry.scanDirectory('./tools');

// 注册到 Agent Core
const toolList = registry.generateToolList();
tcpClient.send({
  method: 'register_tools',
  params: { tools: toolList },
});
```

### 3.3 工具定义校验

工具注册时，注册表会进行以下校验：

| 校验项 | 规则 | 失败处理 |
|--------|------|----------|
| name | 非空，snake_case，仅含小写字母/数字/下划线 | 跳过注册，打印警告 |
| description | 非空，长度 ≤ 200 字符 | 跳过注册 |
| input_schema | 必须是有效的 JSON Schema 对象 | 跳过注册 |
| name 唯一性 | 不能与已有工具重名 | 覆盖或报错（可配置） |
| execute 方法 | 必须存在且返回 Promise/ToolResult | 跳过注册 |
| category | 必须属于 ToolCategory 枚举值 | 默认归类为 'task' |

## 第4章 执行AI 上下文

### 4.1 上下文构成

每次工具调用时，执行AI 引擎会创建一个 `ExecutionContext`，包含工具执行所需的所有依赖。

```typescript
interface ExecutionContext {
  /** 假人实例 — 工具操作的核心对象 */
  bot: Bot;

  /** 世界状态快照 — 执行时游戏状态 */
  world: WorldSnapshot;

  /** 游戏 API 集合 — 执行AI 引擎提供的底层能力 */
  api: GameAPI;

  /** 日志记录器 */
  logger: Logger;

  /** 中断信号 — 用于超时/手动取消 */
  abortSignal: AbortSignal;
}
```

### 4.2 假人接口（Bot）

假人是执行AI 的唯一操作对象，所有工具通过假人接口控制游戏行为。

```typescript
interface Bot {
  /** 假人基本信息 */
  readonly id: string;
  readonly name: string;

  /** 位置信息 */
  readonly position: Vec3;
  readonly dimension: string;

  /** 移动控制 */
  movement: BotMovement;
  /** 视角控制 */
  look: BotLook;
  /** 背包操作 */
  inventory: BotInventory;
  /** 装备操作 */
  equipment: BotEquipment;
  /** 战斗操作 */
  combat: BotCombat;
  /** 交互操作 */
  interaction: BotInteraction;
  /** 生存操作 */
  survival: BotSurvival;
}

// 各子接口定义
interface BotMovement {
  followPath(path: Vec3[], options?: MoveOptions): Promise<MoveResult>;
  moveTo(x: number, y: number, z: number): Promise<MoveResult>;
  setSprinting(sprinting: boolean): void;
  setSneaking(sneaking: boolean): void;
  jump(): void;
  stop(): void;
}

interface BotLook {
  lookAt(x: number, y: number, z: number): Promise<void>;
  lookAtEntity(entityId: string): Promise<void>;
  setYaw(yaw: number): void;
  setPitch(pitch: number): void;
}

interface BotInventory {
  getAllItems(): InventoryItem[];
  getItem(slot: number): InventoryItem | undefined;
  moveItem(fromSlot: number, toSlot: number): void;
  dropItem(slot: number, count: number): void;
  useItem(slot: number): void;
  organize(): void;
}

interface BotEquipment {
  equip(slot: EquipmentSlot): void;
  unequip(slot: EquipmentSlot): void;
  getEquipped(): Record<EquipmentSlot, InventoryItem | undefined>;
}

interface BotCombat {
  attack(entityId: string): void;
  setMode(mode: CombatMode): void;
  stop(): void;
}

interface BotInteraction {
  interactEntity(entityId: string): void;
  interactBlock(x: number, y: number, z: number): void;
  useItemOnBlock(itemSlot: number, x: number, y: number, z: number): void;
}

interface BotSurvival {
  eat(foodSlot: number): Promise<EatResult>;
  sleep(): Promise<void>;
  wakeUp(): void;
}
```

### 4.3 游戏 API 集合

执行AI 引擎提供底层能力接口，供工具调用。

```typescript
interface GameAPI {
  /** 寻路系统 */
  pathfinding: PathfindingAPI;
  /** 方块查询 */
  block: BlockAPI;
  /** 实体查询 */
  entity: EntityAPI;
  /** 世界信息 */
  world: WorldAPI;
  /** 容器操作 */
  container: ContainerAPI;
  /** 时间/天气 */
  environment: EnvironmentAPI;
}

interface PathfindingAPI {
  findPath(from: Vec3, to: Vec3, dimension?: string): Promise<Vec3[]>;
  isReachable(pos: Vec3): boolean;
  getDistance(from: Vec3, to: Vec3): number;
}

interface BlockAPI {
  getBlock(x: number, y: number, z: number): BlockInfo | undefined;
  getBlocks(x1: number, y1: number, z1: number, x2: number, y2: number, z2: number): BlockInfo[];
  findBlock(type: string, radius: number): Vec3 | undefined;
  findBlocks(type: string, radius: number): Vec3[];
}

interface EntityAPI {
  getEntities(radius?: number): EntityInfo[];
  getEntity(id: string): EntityInfo | undefined;
  findEntity(type: string, radius?: number): EntityInfo | undefined;
}

interface WorldAPI {
  getTime(): number;
```

## 第5章 工具执行流程

### 5.1 完整调用链

```
LLM 决策 → 工具调用 (JSON-RPC Batch)
    │
    ▼
Agent Core → TCP 分发 → Adapter Core
    │
    ▼
ToolRegistry.execute(toolName, params, ctx)
    │
    ├── 1. 查找工具模块
    ├── 2. 校验参数（input_schema）
    ├── 3. 创建执行上下文
    ├── 4. 设置超时（abortSignal）
    ├── 5. 执行 tool.execute(params, ctx)
    ├── 6. 捕获结果或异常
    └── 7. 返回 ToolResult
    │
    ▼
结果 → TCP → Agent Core → LLM 下一轮决策
```

### 5.2 Batch 执行

当 LLM 一次发出多个工具调用时，执行AI 引擎按依赖关系调度：

```typescript
class ExecutionEngine {
  async executeBatch(calls: ToolCall[]): Promise<ToolResult[]> {
    // 1. 分析依赖关系
    //    - 移动类工具（is_movement=true）优先执行
    //    - 依赖关系由工具名称语义决定
    //    - 无依赖的工具并行执行

    // 2. 按依赖分组
    //    - 组内顺序执行
    //    - 组间并行执行

    // 3. 收集结果，按请求顺序返回
  }
}
```

### 5.3 超时与中断

| 机制 | 说明 |
|------|------|
| 默认超时 | 每个工具可按 config.timeout_default_ms 设置 |
| 全局超时 | 引擎设置最大执行时间（避免无限等待） |
| abortSignal | 通过 AbortSignal 通知工具提前终止 |
| 中断链 | 父任务取消时，自动中断所有子工具 |

## 第7章 用户自定义工具

### 7.1 创建自定义工具

用户可以在 `tools/custom/` 目录下创建自己的工具模块，无需修改核心代码。

#### 示例：自动种植工具

```typescript
// tools/custom/auto_farm.ts
// 用户自定义工具：自动在指定位置种植作物

import { IToolModule, ToolMetadata, ExecutionContext, ToolResult } from '../core/tool-interface';

export class AutoFarmTool implements IToolModule {
  readonly metadata: ToolMetadata = {
    name: 'auto_farm',
    description: '在指定位置自动种植作物，先耕地再播种',
    category: 'block',
    input_schema: {
      type: 'object',
      properties: {
        x: { type: 'number', description: '起始 X 坐标' },
        z: { type: 'number', description: '起始 Z 坐标' },
        width: { type: 'number', description: '种植区域宽度', default: 5 },
        depth: { type: 'number', description: '种植区域深度', default: 5 },
        crop: {
          type: 'string',
          description: '作物类型',
          enum: ['wheat', 'carrot', 'potato', 'beetroot'],
          default: 'wheat',
        },
      },
      required: ['x', 'z'],
    },
    config: {
      timeout_default_ms: 120000,
      requires_bot: true,
    },
  };

  async execute(params: Record<string, any>, ctx: ExecutionContext): Promise<ToolResult> {
    const { x, z, width = 5, depth = 5, crop = 'wheat' } = params;
    const startTime = Date.now();
    let planted = 0;

    for (let dx = 0; dx < width; dx++) {
      for (let dz = 0; dz < depth; dz++) {
        // 移动到目标位置
        const targetX = x + dx;
        const targetZ = z + dz;

        // 查找耕地或创建耕地
        const block = ctx.api.block.getBlock(targetX, 63, targetZ);
        if (!block || block.name !== 'farmland') {
          // 使用锄头耕地
          await ctx.bot.interaction.useItemOnBlock('hoe_slot', targetX, 63, targetZ);
        }

        // 播种
        await ctx.bot.inventory.useItem('seeds');
        await ctx.bot.interaction.useItemOnBlock('seeds_slot', targetX, 64, targetZ);
        planted++;
      }
    }

    return {
      success: true,
      data: {
        crops_planted: planted,
        area: `${width}x${depth}`,
        duration_ms: Date.now() - startTime,
      },
      duration_ms: Date.now() - startTime,
    };
  }
}
```

### 7.2 工具加载优先级

| 优先级 | 目录 | 说明 |
|--------|------|------|
| 1（最高） | `tools/custom/` | 用户自定义工具，可覆盖内置工具 |
| 2 | `tools/plugins/` | 第三方插件工具 |
| 3（最低） | `tools/*/`（内置） | 系统内置工具 |

**覆盖规则**：如果自定义工具与内置工具同名，自定义工具优先，内置工具被覆盖。

### 7.3 工具依赖声明

工具可以声明对其他工具或模块的依赖，由注册表在加载时解析：

```typescript
interface ToolDependency {
  /** 依赖类型 */
  type: 'tool' | 'module' | 'api';
  /** 依赖名称 */
  name: string;
  /** 版本范围 */
  version?: string;
  /** 是否必需 */
  required?: boolean;
}
```

示例：

```typescript
class AutoFarmTool implements IToolModule {
  readonly metadata: ToolMetadata = {
    name: 'auto_farm',
    // ...
  };

  // 声明依赖
  readonly dependencies: ToolDependency[] = [
    { type: 'tool', name: 'move_to', required: true },
    { type: 'module', name: 'pathfinding', required: true },
    { type: 'api', name: 'block_api', required: true },

}
```

## 第9章 错误处理

### 9.1 工具执行错误码

| 错误码 | 含义 | 说明 |
|--------|------|------|
| `TOOL_NOT_FOUND` | 工具不存在 | 调用了未注册的工具 |
| `PARAM_INVALID` | 参数校验失败 | 参数不符合 JSON Schema |
| `EXECUTION_FAILED` | 执行失败 | 工具内部逻辑出错 |
| `TIMEOUT` | 执行超时 | 超过 timeout_max_ms |
| `BOT_NOT_READY` | 假人未就绪 | 假人未完全加载 |
| `GAME_NOT_READY` | 游戏未就绪 | 世界未完全加载 |
| `ABORTED` | 执行被中断 | 通过 abortSignal 取消 |
| `DEPENDENCY_MISSING` | 依赖缺失 | 工具依赖的模块未加载 |

```typescript
interface ToolError {
```

## 第10章 BE/JE 实现差异

### 10.1 接口一致性

BE 和 JE 的接口定义保持语义一致，但实现细节存在差异：

| 维度 | BE（基岩版） | JE（Java 版） |
|------|-------------|--------------|
| **语言** | TypeScript / Node.js | Java 17+ |
| **工具接口** | `IToolModule` 接口 | `IToolModule` 接口 |
| **注册机制** | 文件系统扫描 + 动态导入 | 注解扫描 + SPI |
| **热加载** | 支持（fs.watch） | 有限支持（Fabric 限制） |
| **假人 API** | LeviLamina BDS API | Minecraft Fabric API |
| **寻路算法** | A* 3D（BDS 坐标系） | A* 3D（Minecraft 坐标系） |
| **异步模型** | Promise / async-await | CompletableFuture |
| **自定义工具** | 放 .ts/.js 文件即可 | 打包 JAR 或 Java 文件 |

### 10.2 坐标系统

```
# BE 基岩版坐标系
# 与 BDS API 一致，Y 轴向上
{ x: 100.5, y: 64.0, z: -200.3 }

# JE Java 版坐标系
# 与 Minecraft Java API 一致
{ x: 100.5, y: 64.0, z: -200.3 }

# 坐标类型
type Vec3 = { x: number; y: number; z: number };
```
