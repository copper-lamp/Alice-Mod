## 第1章 概述

### 1.1 设计目标

记忆系统是 McAgent 的"经验积累层"，使 LLM 能够记住玩家习惯、地图关键点、任务经验等，实现跨会话的知识积累。

| 原则 | 说明 |
|------|------|
| **双模存储** | SQLite 处理结构化查询，Chroma 处理向量语义检索 |
| **类型化** | 5 种记忆类型，8 个记忆分支，各有明确用途 |
| **可遗忘** | 支持按重要度/时间自动清理过期记忆 |
| **可导出** | 支持 JSON 格式导入/导出，便于迁移和备份 |
| **离线可访问** | Agent Core 可直接读取数据库文件，不依赖游戏运行 |

### 1.2 架构

### 1.3 数据流

```
存储流程：
  LLM 调用 memory_store(content, type, tags, importance)
    → MemoryManager.store()
      → 生成 embedding_id (调用嵌入模型)
      → SQLiteStore.save_meta() (保存元数据到 SQLite)
      → ChromaStore.save_vector() (保存向量到 Chroma)
    → 返回记忆 ID

检索流程：
  LLM 调用 memory_recall(type, tags, keywords)
    → MemoryManager.recall()
      → SQLiteStore.query() (结构化条件过滤)
    → 返回匹配结果列表

语义检索流程：
  LLM 调用 memory_get_similar(query, type?, limit?)
    → MemoryManager.getSimilar()
      → 生成查询向量 (调用嵌入模型)
      → ChromaStore.query_similar() (向量相似度搜索)
      → SQLiteStore.get_by_ids() (回补元数据)
    → 返回排序后的记忆列表
```

## 第2章 数据模型

### 2.1 记忆实体

```typescript
/** 记忆实体 — 核心数据模型 */
interface Memory {
  /** 唯一标识（UUID v4） */
  id: string;

  /** 记忆类型 */
  type: MemoryType;

  /** 记忆所属分支 */
  branch: MemoryBranch;

  /** 记忆内容（结构化 JSON） */
  content: Record<string, any>;

  /** 标签列表 */
  tags: string[];

  /** 重要度 1-10（10=最重要，永不过期） */
  importance: number;

  /** 访问次数 */
  accessCount: number;

  /** 创建时间 */
  createdAt: number;        // unix timestamp

  /** 最后更新时间 */
  updatedAt: number;

  /** 过期时间（null=永不过期） */
  expiresAt: number | null;

  /** Chroma 向量 ID（关联用） */
  embeddingId: string | null;
}
```

### 2.2 记忆类型

```typescript
type MemoryType =
  /** 玩家习惯 — 记住玩家偏好和行为模式 */
  | 'player_habit'
  /** 地图关键点 — 记住坐标和位置信息 */
  | 'map_point'
  /** 任务经验 — 记住任务执行经验 */
  | 'task_experience'
  /** 社交关系 — 记住与其他玩家的关系 */
  | 'social'
  /** 技能 — 记住已学技能 */
  | 'skill'
  /** 地图区域 — 命名区域（基地、矿区、村庄等范围） */
  | 'map_region'
  /** 地图生态 — 生物群系和地形特征 */
  | 'map_biome';
```

| 类型 | 内容示例 | 用途 |
|------|----------|------|
| `player_habit` | `{ "player": "Steve", "preference": "钻石工具", "confidence": 0.9 }` | LLM 参考玩家偏好做决策 |
| `map_point` | `{ "name": "钻石矿脉", "x": 128, "y": 20, "z": -256, "dimension": "overworld", "found_at": "2026-07-04" }` | 导航到已知位置 |
| `task_experience` | `{ "task": "挖钻石", "lesson": "带火把和水桶", "success": true }` | 避免重复犯错 |
| `social` | `{ "player": "Steve", "relation": "trusted", "interactions": 15 }` | 决定交互方式 |
| `skill` | `{ "name": "木屋建造", "steps": [...], "materials": [...] }` | 任务分解参考 |
| `map_region` | `{ "name": "主基地", "x1": 100, "z1": 100, "x2": 200, "z2": 200, "dimension": "overworld", "type": "base" }` | 定义和管理命名区域 |
| `map_biome` | `{ "biome": "plains", "x": 150, "z": 150, "dimension": "overworld", "features": ["village_nearby"] }` | 记录生物群系信息 |

### 2.3 记忆分支

```typescript
type MemoryBranch =
  | 'character'       // 角色记忆 — 身份、人设、行为特征
  | 'emotion'         // 情绪特征 — 情绪状态、变化
  | 'environment'     // 环境记忆 — 场景、坐标、描述
  | 'experience'      // 经验库 — 交互经验、教训
  | 'knowledge'       // 知识库 — 领域知识、技能档案
  | 'user_preference' // 用户偏好 — 用户信息、偏好
  | 'emotion_log'     // 情绪记录 — 时序数据
  | 'task_archive';   // 任务存档 — 预设/自定义任务
```

### 2.4 各分支存储策略

| 分支 | 存储方式 | 检索方式 | 说明 |
|------|----------|----------|------|
| 角色记忆 | SQLite | 精确查询 | 角色配置，不常变 |
| 情绪特征 | SQLite | 精确查询 | 当前情绪状态 |
| 环境记忆 | SQLite + Chroma + 空间索引 | 精确 + 语义 + 空间 | 坐标 + 场景描述 + 空间范围查询 |
| 经验库 | Chroma | 语义检索 | 核心经验，靠语义匹配 |
| 知识库 | Chroma | 语义检索 | 知识，靠语义匹配 |
| 用户偏好 | SQLite | 精确查询 | 用户配置，不常变 |
| 情绪记录 | SQLite（时序） | 时间范围查询 | 记录变化历史 |
| 任务存档 | SQLite + Chroma | 精确 + 语义 | 任务模板和经验 |

## 第3章 MemoryManager API

### 3.1 接口定义

```typescript
class MemoryManager {
  constructor(config: MemoryConfig);

  // ─── 存储 ───

  /** 存储一条记忆 */
  async store(params: StoreParams): Promise<StoreResult>;

  /** 批量存储记忆 */
  async batchStore(params: BatchStoreParams): Promise<BatchStoreResult>;

  // ─── 检索 ───

  /** 按条件检索记忆 */
  async recall(params: RecallParams): Promise<RecallResult>;

  /** 按 ID 获取单条记忆 */
  async getById(id: string): Promise<Memory | null>;

  /** 语义相似度检索 */
  async getSimilar(params: SimilarParams): Promise<SimilarResult>;

  /** 列出所有记忆（分页） */
  async list(params: ListParams): Promise<ListResult>;

  // ─── 更新 / 删除 ───

  /** 更新记忆 */
  async update(id: string, updates: Partial<Memory>): Promise<void>;

  /** 删除记忆 */
  async forget(id: string): Promise<void>;

  /** 按条件批量删除 */
  async forgetBy(params: ForgetByParams): Promise<number>;

  // ─── 标签管理 ───

  /** 添加标签 */
  async addTag(id: string, tag: string): Promise<void>;

  /** 移除标签 */
  async removeTag(id: string, tag: string): Promise<void>;

  // ─── 统计 — 管理 ───

  /** 获取记忆统计 */
  async stats(): Promise<MemoryStats>;

  /** 清理过期记忆 */
  async cleanup(options?: CleanupOptions): Promise<CleanupResult>;

  /** 导出记忆 */
  async export(options?: ExportOptions): Promise<string>;

  /** 导入记忆 */
  async import(json: string): Promise<ImportResult>;

  // ─── 地图索引 ───

  /** 空间邻近查询（以某点为中心搜索半径内的地图特征） */
  async queryNearby(params: NearbyParams): Promise<NearbyResult>;

  /** 区域范围查询（搜索矩形区域内的地图特征） */
  async queryArea(params: AreaParams): Promise<AreaResult>;

  /** 获取区域概览（指定坐标区域的综合地图摘要） */
  async getAreaOverview(params: OverviewParams): Promise<OverviewResult>;
}
```

**嵌入生成策略**：

```typescript
class EmbeddingStrategy {
  /** 从记忆内容生成嵌入文本 */
  static buildEmbeddingText(memory: Memory): string {
    // 根据类型构建不同的嵌入文本
    switch (memory.type) {
      case 'player_habit':
        return `玩家 ${memory.content.player} 偏好 ${memory.content.preference}`;
      case 'map_point':
        return `${memory.content.name} 在 ${memory.content.x}, ${memory.content.y}, ${memory.content.z} ${memory.content.dimension}`;
      case 'task_experience':
        return `任务 ${memory.content.task}: ${memory.content.lesson}`;
      case 'social':
        return `与 ${memory.content.player} 的关系: ${memory.content.relation}`;
      case 'skill':
        return `技能 ${memory.content.name}: ${memory.content.steps?.join(', ')}`;
      case 'map_region':
        return `区域 ${memory.content.name} 范围 ${memory.content.x1},${memory.content.z1} 到 ${memory.content.x2},${memory.content.z2} 在 ${memory.content.dimension}，类型 ${memory.content.type}`;

        return `生物群系 ${memory.content.biome} 在 ${memory.content.x}, ${memory.content.z} ${memory.content.dimension}，特征 ${memory.content.features?.join(', ')}`;
      default:
        return JSON.stringify(memory.content);
    }
  }
```

## 第7章 清理策略

### 7.1 自动清理规则

| 规则 | 条件 | 行为 |
|------|------|------|
| 过期清理 | `expires_at < now()` | 删除该记忆 |
| 低重要度清理 | `importance <= 2` 且 `access_count == 0` 且 `created_at > 30 天` | 删除 |
| 数量上限 | 某类型记忆超过 `memory_max_per_type` | 删除最旧/最低重要度的 |
| 手动触发 | LLM 调用 `memory_cleanup` | 按指定模式清理 |

### 7.2 清理实现

```typescript
class CleanupEngine {
  async cleanup(options: CleanupOptions): Promise<CleanupResult> {
    const removed: Array<{ id: string; reason: string }> = [];

    // 1. 过期清理
    if (options.mode === 'expired' || options.mode === 'all') {
      const expired = await this.sqlite.query({
        filter: 'expires_at IS NOT NULL AND expires_at < ?',
        params: [Date.now() / 1000],
      });
      for (const mem of expired) {
        await this.deleteMemory(mem.id);
        removed.push({ id: mem.id, reason: 'expired' });
      }
    }

    // 2. 低重要度清理
    if (options.mode === 'low_importance' || options.mode === 'all') {
      const threshold = options.importanceThreshold ?? 2;
      const cutoff = Date.now() / 1000 - 30 * 86400; // 30 天前
      const lowImportance = await this.sqlite.query({
        filter: 'importance <= ? AND access_count = 0 AND created_at < ?',
        params: [threshold, cutoff],
        limit: 1000,
      });
      for (const mem of lowImportance) {
        await this.deleteMemory(mem.id);
        removed.push({ id: mem.id, reason: 'low_importance' });
      }
    }

    // 保留最近 N 条
    if (options.keepRecent && removed.length > 0) {
      // 确保不把所有记忆清空
    }

    return {
      removed: removed.length,
      kept: 0,
      details: removed,
    };
  }

  private async deleteMemory(id: string): Promise<void> {
    const memory = await this.sqlite.getById(id);
    if (memory?.embeddingId) {
      await this.chroma.delete(memory.embeddingId);
    }
    await this.sqlite.deleteMeta(id);
  }
}
```

## 第11章 地图索引系统

### 11.1 概述

地图索引系统是内嵌在记忆系统中的空间查询引擎，使 LLM 能够按坐标、区域和空间关系检索地图信息。它与记忆系统共享 SQLite 存储，但拥有独立的空间索引结构和查询 API。

**设计目标：**

| 目标 | 说明 |
|------|------|
| **空间聚簇** | 按 Chunk (16×16) 对齐的网格索引，加速邻近查询 |
| **多维度支持** | 每个维度（主世界/下界/末地）独立索引 |
| **区域命名** | 矩形区域定义，支持嵌套和重叠 |
| **概览能力** | 对指定区域生成结构化摘要，供 LLM 快速理解 |
| **语义融合** | 地图特征同样写入 Chroma，支持"找红色矿石"等语义查询 |

### 11.4 空间索引机制

使用 **Chunk 对齐网格**实现快速空间查询。Minecraft 的 Chunk 大小为 16×16 方块，每个特征按坐标 `(blockX >> 4, blockZ >> 4)` 计算所在 Chunk，写入 `map_spatial_grid`。

```
邻近查询流程：

  输入: center=(x=200, z=300), radius=32, dimension=overworld
  
  1. 计算 Chunk 范围:
     chunk_x 范围: (200-32)>>4 ~ (200+32)>>4  = 10 ~ 14
     chunk_z 范围: (300-32)>>4 ~ (300+32)>>4  = 16 ~ 20
  
  2. 查询 map_spatial_grid:
     WHERE chunk_x BETWEEN 10 AND 14
       AND chunk_z BETWEEN 16 AND 20
       AND dimension = 'overworld'
  
  3. 获取特征 ID 列表 → 去重 → map_features 回补元数据
  
  4. 精确距离过滤和排序（欧几里得距离）
```

**查询复杂度**：邻近查询时间复杂度为 O(n)，其中 n 为查询 Chunk 范围内的特征数。由于每个 Chunk 的特征数通常很少（< 10），整体效率较高。

### 11.5 与记忆系统的联动

| 联动方式 | 说明 |
|----------|------|
| **自动创建** | 存入 `map_point`/`map_region`/`map_biome` 类型记忆时，自动创建对应的 MapFeature/MapRegion |
| **同步删除** | 删除关联记忆时，空间索引记录同步删除（`ON DELETE CASCADE/SET NULL`） |
| **语义补充** | map_features 的 `metadata` 字段会写入 Chroma，支持"找钻石""村庄附近"等语义搜索 |
| **概览融合** | `getAreaOverview()` 同时查询空间索引 + 语义记忆，生成综合摘要 |

### 11.7 概览摘要生成策略

`getAreaOverview()` 的 `summary` 字段由以下规则生成：

```typescript
class OverviewBuilder {
  static buildSummary(
    features: MapFeature[],
    regions: MapRegion[],
    bounds: { x1: number; z1: number; x2: number; z2: number },
    dimension: string,
  ): string {
    const parts: string[] = [];

    // 1. 区域范围
    parts.push(`位置: ${dimension} (${bounds.x1},${bounds.z1}) ~ (${bounds.x2},${bounds.z2})`);

    // 2. 命名区域
    if (regions.length > 0) {
      const regionDesc = regions
        .map(r => `${r.name}[${r.regionType}]`)
        .join(', ');
      parts.push(`命名区域: ${regionDesc}`);
    }

    // 3. 特征统计
    const stats = this.countByType(features);
    if (Object.keys(stats).length > 0) {
      parts.push(`特征: ${Object.entries(stats).map(([k, v]) => `${k}×${v}`).join(', ')}`);
    }

    // 4. 关键点（高重要度/高频访问）
    const hotspots = features
      .filter(f => f.tags.includes('important') || f.tags.includes('hot'))
      .slice(0, 3);
    if (hotspots.length > 0) {
      parts.push(`关键点: ${hotspots.map(h => `${h.name||'匿名'}`).join(', ')}`);
    }

    return parts.join('。');
  }

  private static countByType(features: MapFeature[]): Record<string, number> {
    const stats: Record<string, number> = {};
    for (const f of features) {
      stats[f.type] = (stats[f.type] || 0) + 1;
    }
    return stats;
  }
}
```

### 11.8 查询优化

| 场景 | 索引策略 | SQL 示例 |
|------|----------|----------|
| 邻近查询 (半径 R) | Chunk 网格 + 精确距离 | 先查 Chunk 范围，再欧几里得过滤 |
| 区域查询 (矩形) | map_features.coords 索引 | `WHERE x BETWEEN ? AND ? AND z BETWEEN ? AND ?` |
| 区域重叠检测 | map_regions 索引 | `WHERE x1 <= ? AND x2 >= ? AND z1 <= ? AND z2 >= ?` |
| 维度隔离 | dimension 索引 | 所有查询带 `dimension = ?` 条件 |

**内存缓存策略**：MapIndex 在 Agent Core 启动时，将 `map_features` 和 `map_regions` 全量加载到内存（通常 < 1MB，即使 10 万个特征也在 5MB 以内），后续所有查询走内存，写入时同步写 SQLite。
