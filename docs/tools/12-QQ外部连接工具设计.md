# 12-QQ外部连接工具设计

## 1. 模块概述

### 1.1 目标
让 AI 能够：
- 接收 QQ 群/私聊消息
- 发送消息到 QQ 群/私聊
- 桥接 QQ 消息与游戏内消息
- 通过 QQ 接收指令并执行

### 1.2 架构

```
┌─────────────────────────────────────────┐
│              QQ 服务器                   │
│  (go-cqhttp / NapCat / Lagrange)        │
└──────────────┬──────────────────────────┘
               │ WebSocket / HTTP
               ▼
┌─────────────────────────────────────────┐
│           QQ 连接管理器                  │
│  - 消息收发                             │
│  - 事件监听                             │
│  - 消息队列                             │
└──────────────┬──────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│           消息桥接层                     │
│  - QQ <-> 游戏内消息转换                │
```

## 2. 工具列表

| 工具名 | 功能 | 参数 | 返回 |

| `qq_send` | 发送 QQ 消息 | `type: 'group_msg' \| 'private_msg' \| 'image' \| 'file'`, `target: string`, `content?: string`, `file_url?: string` | 操作结果 |

## 3. 工具详细设计

### 3.1 qq_send（发送 QQ 消息）

通过 `type` 参数支持群消息、私聊、图片、文件四种发送方式。

**功能**：发送消息到 QQ 群或私聊，支持文本、图片、文件

**参数**：
```typescript
{
  type: 'group_msg' | 'private_msg' | 'image' | 'file';  // 发送类型
  target: string;         // 目标 ID（群号或 QQ 号）
  content?: string;       // 消息内容（文本消息时必填）
  file_url?: string;      // 文件/图片 URL（图片或文件时必填）
}
```

**返回**：
```typescript
{
  success: boolean;
  message_id?: string;
  error?: string;
}
```

**示例**：
```typescript
// 发送群消息
await qq_send({
  type: 'group_msg',
  target: '123456789',
  content: '大家好，我是 AI 助手'
});

// 发送私聊消息
await qq_send({
  type: 'private_msg',
  target: '987654321',
  content: '你好，需要帮助吗？'
});

// 发送图片
await qq_send({
  type: 'image',
  target: '123456789',
  file_url: 'https://example.com/image.jpg'
});

// 发送文件
await qq_send({
  type: 'file',
  target: '123456789',
  file_url: 'https://example.com/file.zip'
});
```

### 3.2 qq_info（查询 QQ 信息）

通过 `type` 参数支持查询群信息、群成员列表、用户信息三种查询。

**功能**：查询 QQ 群信息、群成员列表或用户信息

**参数**：
```typescript
{
  type: 'group' | 'members' | 'user';  // 查询类型
  target_id: string;       // 目标 ID（群号或 QQ 号）
}
```

**返回**：
```typescript
{
  success: boolean;
  data?: {
    // group 类型
    group_id?: string;
    group_name?: string;
    member_count?: number;
    max_member_count?: number;
    owner_id?: string;
    
    // members 类型
    members?: Array<{
      user_id: string;
      user_name: string;
      role: string;
    }>;
    
    // user 类型
    user_id?: string;
    user_name?: string;
    nickname?: string;
  };
  error?: string;
}
```

**示例**：
```typescript
// 查询群信息
await qq_info({
  type: 'group',
  target_id: '123456789'
});

// 查询群成员列表
await qq_info({
  type: 'members',
  target_id: '123456789'
});

// 查询用户信息
await qq_info({
  type: 'user',
  target_id: '987654321'
});
```

## 4. 消息桥接逻辑

### 4.1 QQ -> 游戏内

```typescript
// QQ 消息监听
qqClient.on('message', (msg: QQMessage) => {
  // 1. 检查是否启用桥接
  const bridge = bridgeManager.getBridge(msg.group_id);
  if (!bridge || bridge.direction === 'game_to_qq') return;
  
  // 2. 检查过滤规则
  if (bridge.filter) {
    if (bridge.filter.keywords && !bridge.filter.keywords.some(kw => msg.content.includes(kw))) {
      return;
    }
    if (bridge.filter.users && !bridge.filter.users.includes(msg.user_id)) {
      return;
    }
  }
  
  // 3. 转换消息格式
  const gameMsg = `[QQ] ${msg.user_name}: ${msg.content}`;
  
  // 4. 发送到游戏内
  gameClient.sendChat(gameMsg);
});
```

### 4.2 游戏内 -> QQ

```typescript
// 游戏内消息监听
gameClient.on('chat', (msg: GameMessage) => {
  // 1. 检查是否启用桥接
  const bridges = bridgeManager.getBridgesByDirection('qq_to_game');
  
  // 2. 发送到所有启用的群
  for (const bridge of bridges) {
    const qqMsg = `${bridge.prefix || ''} ${msg.player_name}: ${msg.content}`;
    qqClient.sendGroupMsg(bridge.group_id, qqMsg);
  }
});
```

### 4.3 指令解析

```typescript
// 解析 QQ 消息中的指令
function parseQQCommand(content: string): {
  isCommand: boolean,
  command?: string,
  args?: string[]
} {
  // 指令格式：#指令名 参数1 参数2
  if (!content.startsWith('#')) {
    return { isCommand: false };
  }
  
  const parts = content.slice(1).split(' ');
  const command = parts[0];
  const args = parts.slice(1);
  
  return {
    isCommand: true,
    command,
    args
  };
}

// 示例指令
// #挖钻石 10 -> AI 去挖 10 个钻石
// #回家 -> AI 返回基地
// #状态 -> AI 报告当前状态
```

## 5. 权限控制

### 5.1 权限等级

```typescript
enum QQPermission {
  NONE = 0,        // 无权限
  BASIC = 1,       // 基础权限（查询状态）
  COMMAND = 2,     // 指令权限（执行简单指令）
  ADMIN = 3        // 管理员权限（所有权限）
}

// 权限配置
const permissionConfig = {
  owner_id: 987654321,      // 群主 QQ 号（自动 ADMIN）
  admins: [111222333],       // 管理员 QQ 号列表
  whitelist: [444555666],    // 白名单用户（COMMAND 权限）
  default_permission: QQPermission.BASIC  // 默认权限
};
```

### 5.2 权限检查

```typescript
function checkPermission(userId: number, requiredPerm: QQPermission): boolean {
  // 群主自动拥有 ADMIN 权限
  if (userId === permissionConfig.owner_id) return true;
  
  // 管理员拥有 ADMIN 权限
  if (permissionConfig.admins.includes(userId)) return true;
  
  // 白名单用户拥有 COMMAND 权限
  if (permissionConfig.whitelist.includes(userId)) {
    return requiredPerm <= QQPermission.COMMAND;
  }
  
  // 其他用户只有默认权限
  return permissionConfig.default_permission >= requiredPerm;
}
```

## 6. 消息队列与时效性

### 6.1 消息队列

```typescript
class QQMessageQueue {
  private messages: QQMessage[] = [];
  private maxSize = 100;
  
  addMessage(msg: QQMessage) {
    this.messages.push(msg);
    
    // 超过最大容量，移除最旧的消息
    if (this.messages.length > this.maxSize) {
      this.messages.shift();
    }
  }
  
  getUnread(filter?: QQMessageFilter): QQMessage[] {
    let unread = this.messages.filter(m => !m.read);
    
    if (filter) {
      if (filter.type) unread = unread.filter(m => m.type === filter.type);
      if (filter.group_id) unread = unread.filter(m => m.group_id === filter.group_id);
      if (filter.user_id) unread = unread.filter(m => m.user_id === filter.user_id);
      if (filter.since) unread = unread.filter(m => m.timestamp >= filter.since);
    }
    
    return unread;
  }
  
  markRead(messageId: number) {
    const msg = this.messages.find(m => m.message_id === messageId);
    if (msg) msg.read = true;
  }
}
```

### 6.2 时效性处理

```typescript
// QQ 消息默认 5 分钟过期
const QQ_MESSAGE_TTL = 5 * 60 * 1000;

function isQQMessageExpired(msg: QQMessage): boolean {
  return Date.now() - msg.timestamp > QQ_MESSAGE_TTL;
}

// 定期清理过期消息
setInterval(() => {
  qqMessageQueue.messages = qqMessageQueue.messages.filter(
    m => !isQQMessageExpired(m)
  );
}, 60 * 1000);  // 每分钟清理一次
```

## 7. 错误处理与重连

### 7.1 连接状态管理

```typescript
enum QQConnectionState {
  DISCONNECTED = 'disconnected',
  CONNECTING = 'connecting',
  CONNECTED = 'connected',
  RECONNECTING = 'reconnecting'
}

class QQConnectionManager {
  private state: QQConnectionState = QQConnectionState.DISCONNECTED;
  private reconnectAttempts = 0;
  private maxReconnectAttempts = 5;
  private reconnectDelay = 5000;  // 5秒
  
  async connect() {
    this.state = QQConnectionState.CONNECTING;
    
    try {
      await qqClient.connect();
      this.state = QQConnectionState.CONNECTED;
      this.reconnectAttempts = 0;
    } catch (error) {
      this.state = QQConnectionState.DISCONNECTED;
      this.scheduleReconnect();
    }
  }
  
  private scheduleReconnect() {
    if (this.reconnectAttempts >= this.maxReconnectAttempts) {
      console.error('QQ 连接失败，已达最大重试次数');
      return;
    }
    
    this.state = QQConnectionState.RECONNECTING;
    this.reconnectAttempts++;
    
    const delay = this.reconnectDelay * Math.pow(2, this.reconnectAttempts - 1);
    setTimeout(() => this.connect(), delay);
  }
}
```

### 7.2 消息发送失败处理

```typescript
async function sendQQMessageWithRetry(
  target: { type: 'group' | 'private', id: number },
  message: string,
  maxRetries = 3
): Promise<boolean> {
  for (let i = 0; i < maxRetries; i++) {
    try {
      if (target.type === 'group') {
        await qqClient.sendGroupMsg(target.id, message);
      } else {
        await qqClient.sendPrivateMsg(target.id, message);
      }
      return true;
    } catch (error) {
      console.warn(`QQ 消息发送失败 (尝试 ${i + 1}/${maxRetries}):`, error);
      if (i < maxRetries - 1) {
        await sleep(1000 * (i + 1));  // 递增延迟
      }
    }
  }
  
  console.error('QQ 消息发送失败，已达最大重试次数');
  return false;
}
```

## 9. 使用场景示例

### 9.1 场景 1：通过 QQ 控制 AI

```
用户在 QQ 群发送: #挖钻石 10
  ↓
AI 解析指令: command="挖钻石", args=["10"]
  ↓
AI 检查权限: 用户有 COMMAND 权限
  ↓
AI 执行任务: 前往矿区，挖掘 10 个钻石
  ↓
AI 回复: "好的，我去挖 10 个钻石"
  ↓
QQ 群收到: "好的，我去挖 10 个钻石"
```

### 9.2 场景 2：QQ 与游戏内消息同步

```
游戏内玩家说: "谁能帮我打一下僵尸？"
  ↓
消息桥接: "[游戏] Steve: 谁能帮我打一下僵尸？"
  ↓
QQ 群收到: "[游戏] Steve: 谁能帮我打一下僵尸？"
  ↓
AI 看到消息，判断相关性: medium
  ↓
AI 决定响应: "我来帮你！"
  ↓
QQ 群收到: "我来帮你！"
  ↓
AI 前往 Steve 位置，击杀僵尸
```

### 9.3 场景 3：AI 主动汇报

```
AI 完成任务: 挖了 10 个钻石
  ↓
AI 决定汇报: "我挖完 10 个钻石了！"
  ↓
QQ 群收到: "我挖完 10 个钻石了！"
  ↓
游戏内也收到: "[QQ] AI: 我挖完 10 个钻石了！"
```
