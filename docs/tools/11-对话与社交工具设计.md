# 对话与社交工具设计（Communication & Social Tools Design）

> 版本：v1.0
> 日期：2026-07-01
> 架构：LLM 调用工具 → 执行AI 执行操作 → 返回结果

---

## 0. 设计原则

### 0.1 核心概念

**聊天操作由执行AI负责，LLM 只调用工具**

```
┌─────────────────────────────────────────────────────┐
│  LLM（大脑）                                         │
│  - 决定要发送消息/私聊/广播                           │
│  - 调用对话工具                                       │
└────────────────┬────────────────────────────────────┘
                 │ 调用工具
                 ▼
        ┌────────────────┐
        │  对话工具        │
        │  - chat         │
        │  - whisper      │
        │  - message      │
        └────────┬───────┘
                 │ 委托执行
                 ▼
        ┌────────────────┐
        │  执行AI（假人）  │
        │  - 发送消息     │
        │  - 监听消息     │
        └────────┬───────┘
                 │ 返回结果
                 ▼
        ┌────────────────┐
        │  工具返回        │
        │  - 成功/失败    │
        └────────────────┘
```

### 0.2 设计原则

| 原则 | 说明 |
|---|---|
| **简单直接** | 工具粒度适中，避免过度设计 |
| **有反馈** | 工具返回操作结果（成功/失败） |
| **可失败** | 玩家不在线、消息过长等 |

## 3. 消息队列系统

### 3.1 消息数据结构

```typescript
interface ChatMessage {
  id: string;
  sender: string;        // 发送者
  content: string;       // 消息内容
  type: 'public' | 'private' | 'system';  // 消息类型
  timestamp: number;     // 时间戳
  read: boolean;         // 是否已读
  replyTo?: string;      // 回复的消息ID
}
```

### 3.2 消息队列实现

```typescript
class MessageQueue {
  private messages: ChatMessage[] = [];
  private maxMessages: number = 100;  // 最多保留100条
  private contextSize: number = 20;   // 上下文窗口20条
  
  // 添加消息
  addMessage(msg: ChatMessage): void {
    this.messages.push(msg);
    // 超过上限时移除最旧的
    if (this.messages.length > this.maxMessages) {
      this.messages.shift();
    }
  }
  
  // 获取未读消息
  getUnread(limit?: number): ChatMessage[] {
    const unread = this.messages.filter(m => !m.read);
    return limit ? unread.slice(-limit) : unread;
  }
  
  // 按条件过滤
  getMessages(filter?: MessageFilter): ChatMessage[] {
    let result = [...this.messages];
    if (filter?.sender) result = result.filter(m => m.sender === filter.sender);
    if (filter?.type) result = result.filter(m => m.type === filter.type);
    if (filter?.keyword) result = result.filter(m => m.content.includes(filter.keyword!));
    if (filter?.since) result = result.filter(m => m.timestamp >= filter.since!);
    if (filter?.limit) result = result.slice(-filter.limit);
    return result;
  }
  
  // 标记已读
  markRead(messageId: string): boolean {
    const msg = this.messages.find(m => m.id === messageId);
    if (msg) {
      msg.read = true;
      return true;
    }
    return false;
  }
  
  // 获取上下文（最近N条）
  getContext(size?: number): ChatMessage[] {
    return this.messages.slice(-(size || this.contextSize));
  }
}
```

### 4.2 执行AI接口

```typescript
// 执行AI接口（对话操作）
interface ExecutionAI {
  chat(message: string, mode: 'chat' | 'broadcast' | 'emote'): Promise<ChatResult>;
  whisper(target: string, message: string): Promise<WhisperResult>;
  message(action: string, message_id?: string, content?: string): Promise<MessageResult>;
}
```

### 4.3 执行AI实现（示例）

```typescript
class ExecutionAIImpl implements ExecutionAI {
  private player: Player;
  
  constructor(player: Player) {
    this.player = player;
  }
  
  async chat(message: string, mode: 'chat' | 'broadcast' | 'emote'): Promise<ChatResult> {
    switch (mode) {
      case 'chat':
        // 检查消息长度
        if (message.length > 256) {
          return { success: false, reason: 'message_too_long' };
        }
        this.player.chat(message);
        return { success: true, reason: 'success' };
        
      case 'broadcast':
        // 检查权限
        if (!this.player.isOp()) {
          return { success: false, reason: 'no_permission' };
        }
        if (message.length > 256) {
          return { success: false, reason: 'message_too_long' };
        }
        mc.broadcast(message);
        return { success: true, reason: 'success' };
        
      case 'emote':
        // 执行表情动作
        const validEmotes = ['wave', 'dance', 'laugh', 'cry', 'angry', 'point'];
        if (!validEmotes.includes(message)) {
          return { success: false, reason: 'emote_not_found' };
        }
        this.player.emote(message);
        return { success: true, reason: 'success' };
    }
  }
  
  async whisper(target: string, message: string): Promise<WhisperResult> {
    // 检查玩家是否在线
    const targetPlayer = mc.getPlayer(target);
    if (!targetPlayer) {
      return { success: false, reason: 'player_not_found' };
    }
    
    // 检查消息长度
    if (message.length > 256) {
      return { success: false, reason: 'message_too_long' };
    }
    
    // 发送私聊
    this.player.whisper(target, message);
    
    return { success: true, reason: 'success', target };
  }
  
  async message(action: string, message_id?: string, content?: string): Promise<MessageResult> {
    switch (action) {
      case 'list':
        const messages = messageQueue.getMessages();
        return { success: true, messages: messages, total: messages.length };
        
      case 'unread':
        const unread = messageQueue.getUnread();
        return { success: true, messages: unread, unreadCount: unread.length };
        
      case 'mark_read':
        if (!message_id) return { success: false, reason: 'error' };
        const ok = messageQueue.markRead(message_id);
        return { success: ok, reason: ok ? 'success' : 'message_not_found' };
        
      case 'reply':
        if (!message_id || !content) return { success: false, reason: 'error' };
        this.player.chat(`/msg ${messageQueue.getMessage(message_id)?.sender} ${content}`);
        return { success: true, reason: 'success', replyTo: message_id };
    }
  }
}
```

## 6. 总结

### 6.1 核心设计

| 组件 | 职责 |
|---|---|
| **LLM（大脑）** | 决定要发送/接收消息，调用对话工具 |
| **对话工具** | 封装聊天操作（发送+接收），返回执行结果 |
| **消息队列** | 缓存消息，支持过滤/上下文/未读管理 |
| **执行AI（假人）** | 监听游戏事件，填充消息队列 |

### 6.2 关键特性

1. **双向通信**：发送（chat/whisper）+ 接收（message）
2. **消息队列**：缓存最近100条消息，支持按发送者/类型/关键词过滤
3. **上下文管理**：默认保留20条对话历史，支持对话连贯性
4. **有反馈**：工具返回操作结果（成功/失败）
5. **可失败**：玩家不在线、消息过长、无权限等

### 6.3 工具列表

| 工具名 | 功能 |
|---|---|
| `chat` | 发送聊天消息（支持普通聊天/广播/表情动作三种模式） |
| `whisper` | 私聊 |
| `message` | 消息管理（支持列表查询/未读消息/标记已读/回复消息四种操作） |

### 6.4 消息相关性判断

AI 也是玩家，不是所有消息都与自己有关。需要判断消息相关性：

```typescript
interface MessageRelevance {
  relevance: 'high' | 'medium' | 'low' | 'none';  // 相关性等级
  reason: string;  // 判断原因
  shouldRespond: boolean;  // 是否应该回复
}

// 相关性判断规则
function judgeRelevance(message: ChatMessage, agentState: AgentState): MessageRelevance {
  // 高相关性：直接提到AI名字、私聊、询问AI
  if (message.content.includes(agentState.name) || message.type === 'private') {
    return { relevance: 'high', reason: '直接提及', shouldRespond: true };
  }
  
  // 中相关性：与当前任务相关、附近玩家聊天
  if (isRelatedToCurrentTask(message.content) || isNearbyPlayer(message.sender)) {
    return { relevance: 'medium', reason: '任务相关', shouldRespond: false };
  }
  
  // 低相关性：一般聊天、与AI无关
  return { relevance: 'low', reason: '与AI无关', shouldRespond: false };
}
```

**判断规则**：

| 相关性 | 条件 | 是否回复 |
|---|---|---|
| **high** | 直接提到AI名字、私聊、询问AI | 立即回复 |
| **medium** | 与当前任务相关、附近玩家聊天 | 延迟回复 |
| **low** | 一般聊天、与AI无关 | 不回复 |

### 6.5 消息时效性

消息有时效性，过期消息不需要处理：

```typescript
interface MessageTimeliness {
  isExpired: boolean;  // 是否过期
  age: number;  // 消息年龄（秒）
  urgency: 'urgent' | 'normal' | 'low';  // 紧急程度
}

// 时效性判断
function judgeTimeliness(message: ChatMessage): MessageTimeliness {
  const age = (Date.now() - message.timestamp) / 1000;  // 秒
  
  // 超过5分钟视为过期
  if (age > 300) {
    return { isExpired: true, age, urgency: 'low' };
  }
  
  // 超过1分钟紧急度降低
  if (age > 60) {
    return { isExpired: false, age, urgency: 'normal' };
  }
  
  return { isExpired: false, age, urgency: 'urgent' };
}
```

**时效性规则**：

| 年龄 | 紧急程度 | 处理方式 |
|---|---|---|
| < 1分钟 | urgent | 立即处理 |
| 1-5分钟 | normal | 正常处理 |
| > 5分钟 | expired | 不处理 |

### 6.6 选择性回复策略

不是所有消息都需要回复，需要决策：

```typescript
interface ResponseStrategy {
  shouldRespond: boolean;
  responsePriority: number;  // 0-100，越高越优先
  responseDelay: number;  // 延迟回复时间（毫秒）
  reason: string;
}

// 回复决策
function decideResponse(message: ChatMessage, agentState: AgentState): ResponseStrategy {
  const relevance = judgeRelevance(message, agentState);
  const timeliness = judgeTimeliness(message);
  
  // 过期消息不回复
  if (timeliness.isExpired) {
    return { shouldRespond: false, responsePriority: 0, responseDelay: 0, reason: '消息已过期' };
  }
  
  // 高相关性立即回复
  if (relevance.relevance === 'high') {
    return { shouldRespond: true, responsePriority: 90, responseDelay: 0, reason: '直接相关' };
  }
  
  // 中相关性延迟回复
  if (relevance.relevance === 'medium') {
    return { shouldRespond: true, responsePriority: 50, responseDelay: 2000, reason: '任务相关' };
  }
  
  // 低相关性不回复
  return { shouldRespond: false, responsePriority: 10, responseDelay: 0, reason: '与AI无关' };
}
```

### 6.7 消息优先级队列

使用优先级队列管理消息：

```typescript
class PriorityMessageQueue {
  private messages: Array<{
    message: ChatMessage;
    priority: number;
    timestamp: number;
  }> = [];
  
  // 添加消息（带优先级）
  addMessage(msg: ChatMessage, priority: number): void {
    this.messages.push({ message: msg, priority, timestamp: Date.now() });
    // 按优先级排序
    this.messages.sort((a, b) => b.priority - a.priority);
  }
  
  // 获取最高优先级消息
  getTopMessage(): ChatMessage | null {
    // 移除过期消息
    this.removeExpired();
    return this.messages.length > 0 ? this.messages[0].message : null;
  }
  
  // 移除过期消息
  private removeExpired(): void {
    const now = Date.now();
    this.messages = this.messages.filter(item => {
      const age = (now - item.message.timestamp) / 1000;
      return age < 300;  // 5分钟内有效
    });
  }
}
```
