# 工具协议规范（Tool Protocol Specification）

> 版本：v1.0
> 日期：2026-07-14
> 状态：已定稿
> 范围：所有工具的统一接口契约

---

## 1. 概述

本文档定义 Alice Mod 所有工具的**统一接口契约**（Protocol），包括参数规范、返回格式、错误码、注册方式等。所有工具设计文档必须遵循本规范。

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| **松耦合** | 协议只定义契约，不绑定实现方式 |
| **自描述** | 参数和返回结构应能自我说明 |
| **可扩展** | 协议版本化，向后兼容 |
| **可测试** | 契约可独立验证，不依赖运行时 |

### 1.2 协议分层

```
┌─────────────────────────────────────────────────────┐
│  LLM 层                                              │
│  - 通过 ToolSchema 感知工具                           │
│  - 调用工具、解析结果                                  │
└──────────────────────┬──────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────┐
│  工具协议层（本文档）                                  │
│  - ToolInterface（契约定义）                          │
│  - ResultEnvelope（返回信封）                         │
│  - ErrorCode（错误码标准）                             │
│  - ToolRegistry（注册机制）                            │
└──────────────────────┬──────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────┐
│  执行层（ExecutionAI）                                │
│  - 实现具体工具逻辑                                    │
│  - 返回符合协议的结果                                   │
└─────────────────────────────────────────────────────┘
```

## 2. 工具接口契约（ToolInterface）

### 2.1 接口定义

```typescript
/**
 * 工具的核心契约。
 * 
 * 一个工具 = { name, description, parameterSchema, invoke }。
 * 协议层不关心工具的实现方式（本地/远程/同步/异步），
 * 只关心契约是否满足。
 */
interface ToolInterface {
  /** 工具名称，snake_case 格式 */
  name: string;

  /** 
   * 工具描述，展示给 LLM 使用。
   * 应包含：用途、何时使用、每个参数的含义、注意事项。
   */
  description: string;

  /**
   * 参数规范，使用 JSON Schema（OpenAI tool-parameter dialect）。
   * 这是 LLM 理解工具参数的唯一来源。
   */
  parameterSchema: ParameterSchema;

  /**
   * 执行工具。
   * 参数已通过 parameterSchema 校验，直接使用。
   * 返回结果必须符合 ResultEnvelope 格式。
   */
  invoke(params: Record<string, any>): Promise<ResultEnvelope>;
}
```

### 2.2 参数规范（ParameterSchema）

使用 JSON Schema 的 OpenAI 工具参数方言：

```typescript
interface ParameterSchema {
  type: 'object';
  properties: Record<string, PropertySchema>;
  required?: string[];
  additionalProperties?: boolean;
}

interface PropertySchema {
  type: 'string' | 'number' | 'boolean' | 'object' | 'array';
  description?: string;           // 参数说明（LLM 可见）
  enum?: string[];                // 枚举值
  default?: any;                  // 默认值
  minimum?: number;               // 数值最小值
  maximum?: number;               // 数值最大值
  properties?: Record<string, PropertySchema>;  // object 类型子属性
  items?: PropertySchema;         // array 类型元素定义
}
```

### 2.3 参数规范约定

| 约定 | 规则 | 示例 |
|------|------|------|
| **命名** | 小驼峰（camelCase） | `targetType`, `blockName` |
| **必填** | 明确标注 `required` | `required: ['x', 'y', 'z']` |
| **可选** | 提供 `default` 值 | `default: 16` |
| **枚举** | 使用 `enum` 限定 | `enum: ['equip', 'unequip']` |
| **描述** | 必须提供 `description` | 对 LLM 有用的说明 |

**参数类型映射**：

| JSON Schema | TypeScript | 说明 |
|-------------|------------|------|
| `string` | `string` | 字符串 |
| `number` | `number` | 数值（整数/浮点数） |
| `boolean` | `boolean` | 布尔值 |
| `object` | `{}` / `interface` | 对象 |
| `array` | `[]` | 数组 |

## 3. 返回信封（ResultEnvelope）

### 3.1 信封结构

所有工具必须返回统一的 `ResultEnvelope` 结构：

```typescript
interface ResultEnvelope<T = any> {
  /** 是否成功 */
  success: boolean;

  /** 错误信息（success=false 时必填） */
  error?: {
    code: ErrorCode;       // 错误码
    message: string;       // 人类可读的错误描述
    details?: any;         // 额外错误详情（可选）
  };

  /** 返回数据（success=true 时包含） */
  data?: T;

  /** 执行元数据 */
  meta?: {
    duration: number;      // 执行耗时（毫秒）
    cost?: Record<string, number>;  // 资源消耗（如饥饿、耐久）
  };
}
```

### 3.2 返回格式约定

```typescript
// 成功返回
{
  "success": true,
  "data": {
    // 具体返回数据
  },
  "meta": {
    "duration": 1500,
    "cost": { "hunger": 0.5 }
  }
}

// 失败返回
{
  "success": false,
  "error": {
    "code": "NOT_FOUND",
    "message": "实体不存在: entity_123",
    "details": { "entityId": "entity_123" }
  },
  "meta": {
    "duration": 200
  }
}
```

### 3.3 自然语言格式化

工具返回的 JSON 结果会在注入回 LLM 上下文前格式化为自然语言：

```typescript
/**
 * 将 ResultEnvelope 格式化为给 LLM 易读的自然语言文本。
 * 每种工具可自定义格式化逻辑。
 */
function formatResult(envelope: ResultEnvelope, toolName: string): string {
  if (!envelope.success) {
    return `[${toolName}] 失败: ${envelope.error.code} - ${envelope.error.message}`;
  }
  
  // 由各工具自己实现格式化
  return customFormat(envelope.data, toolName);
}
```

## 4. 错误码标准（ErrorCode）

### 4.1 通用错误码

| 错误码 | 含义 | 适用场景 |
|--------|------|----------|
| `NOT_FOUND` | 目标不存在 | 实体、方块、物品不存在 |
| `TOO_FAR` | 距离太远 | 目标超出交互距离 |
| `TIMEOUT` | 操作超时 | 寻路超时、任务执行超时 |
| `NO_PERMISSION` | 无权限 | 广播权限、OP 权限 |
| `INVALID_PARAMS` | 参数无效 | 参数校验失败 |
| `INTERNAL_ERROR` | 内部错误 | 未预期的运行时错误 |
| `CANCELLED` | 操作被取消 | 用户中断、事件中断 |

### 4.2 领域错误码

| 领域 | 错误码 | 含义 |
|------|--------|------|
| **背包** | `INVENTORY_FULL` | 背包已满 |
|  | `ITEM_NOT_FOUND` | 背包中找不到指定物品 |
|  | `CONTAINER_FULL` | 容器已满 |
|  | `CONTAINER_NOT_FOUND` | 容器不存在 |
| **战斗** | `TARGET_NOT_FOUND` | 目标实体不存在 |
|  | `WEAPON_BROKEN` | 武器损坏 |
|  | `NO_AMMO` | 弹药不足 |
| **生存** | `NOT_HUNGRY` | 不饿，无需进食 |
|  | `NO_FOOD` | 背包中没有食物 |
|  | `NOT_NIGHT` | 还不是夜晚，无法睡觉 |
|  | `MONSTERS_NEARBY` | 附近有怪物，无法睡觉 |
|  | `BED_OCCUPIED` | 床被占用 |
| **移动** | `NO_PATH` | 无法找到路径 |
|  | `BLOCKED` | 路径被阻挡 |
|  | `ENTITY_MOVED` | 目标实体已移动 |
|  | `NOT_RIDEABLE` | 该实体不可骑乘 |
|  | `NOT_RIDING` | 当前未在骑乘状态 |
| **方块** | `BLOCK_NOT_FOUND` | 方块不存在 |
|  | `NOT_BREAKABLE` | 方块不可破坏 |
|  | `POSITION_OCCUPIED` | 位置已被占用 |
|  | `CANNOT_PLACE` | 无法放置方块 |
|  | `TOO_LARGE` | 区域操作体积过大 |
|  | `NO_ORE` | 未找到矿脉 |
| **对话** | `MESSAGE_TOO_LONG` | 消息过长 |
|  | `PLAYER_NOT_FOUND` | 玩家不在线 |
|  | `MUTED` | 被禁言 |
|  | `EMOTE_NOT_FOUND` | 表情动作不存在 |
|  | `MESSAGE_NOT_FOUND` | 消息不存在 |
| **QQ** | `CONNECTION_FAILED` | QQ 连接失败 |
|  | `SEND_FAILED` | 消息发送失败 |
| **记忆** | `MEMORY_NOT_FOUND` | 记忆不存在 |
|  | `STORAGE_FULL` | 存储空间已满 |
| **任务** | `TASK_NOT_FOUND` | 任务不存在 |
|  | `TASK_ALREADY_RUNNING` | 任务已在运行 |
|  | `DEPENDENCY_FAILED` | 依赖任务失败 |
|  | `MAX_RETRIES_EXCEEDED` | 超过最大重试次数 |

## 5. 工具注册（ToolRegistry）

### 5.1 注册接口

```typescript
/**
 * 工具注册中心。
 * 所有工具必须在启动时注册，注册后只读。
 */
class ToolRegistry {
  /**
   * 注册一个工具。
   * 不允许重复注册相同名称的工具。
   * 应在应用初始化阶段调用。
   */
  static register(tool: ToolInterface): void;

  /**
   * 通过名称获取工具（精确匹配）。
   */
  static get(name: string): ToolInterface | undefined;

  /**
   * 模糊匹配工具名（先精确匹配，再大小写不敏感匹配）。
   * 用于容错 LLM 生成的工具名大小写错误。
   */
  static resolve(name: string): ToolInterface | undefined;

  /**
   * 获取所有已注册的工具。
   * 返回按注册顺序排列的列表。
   */
  static all(): ToolInterface[];

  /**
   * 获取所有工具的 Schema 列表（用于 LLM 工具调用）。
   */
  static allSchemas(): ToolSchema[];

  /**
   * 获取已注册工具数量。
   */
  static size(): number;
}
```

### 5.2 工具 Schema 格式（LLM 可见）

```typescript
/**
 * 工具 Schema，直接传递给 LLM API。
 * 格式兼容 OpenAI function calling / tool_use。
 */
interface ToolSchema {
  type: 'function';
  function: {
    name: string;
    description: string;
    parameters: ParameterSchema;
  };
}
```

### 5.3 注册示例

```typescript
// 注册工具
ToolRegistry.register({
  name: 'move_to',
  description: '移动到目标位置，支持坐标、实体、方块三种目标类型',
  parameterSchema: {
    type: 'object',
    properties: {
      target: {
        type: 'object',
        description: '目标位置或实体引用',
        properties: {
          x: { type: 'number', description: 'X 坐标' },
          y: { type: 'number', description: 'Y 坐标' },
          z: { type: 'number', description: 'Z 坐标' }
        }
      },
      targetType: {
        type: 'string',
        enum: ['coordinate', 'entity', 'block'],
        description: '目标类型'
      },
      distance: {
        type: 'number',
        description: '停止距离（格）',
        default: 2
      }
    },
    required: ['target', 'targetType']
  },
  async invoke(params) {
    // 执行逻辑...
    return {
      success: true,
      data: { finalPosition: { x: 128, y: 64, z: -256 } },
      meta: { duration: 4500, cost: { hunger: 0.5 } }
    };
  }
});
```

## 7. 协议变更流程

### 7.1 向后兼容规则

1. **新增参数**：必须提供 `default` 值，确保旧调用不报错
2. **新增返回字段**：兼容，旧调用忽略新字段
3. **废弃参数**：标记为 `deprecated`，保留至少一个版本
4. **移除参数**：必须大版本升级

### 7.2 版本号规则

```
协议版本 = 主版本.次版本
- 主版本：不兼容的协议变更（参数/返回格式破坏性改变）
- 次版本：向后兼容的变更（新增字段/参数）
```

## 8. 附录

### 8.1 与 Numen 协议对比

| 特性 | Numen | Alice Mod |
|------|-------|-----------|
| 工具接口 | `NumenTool` (Java) | `ToolInterface` (TypeScript) |
| 参数格式 | `Map<String, Object>` (JSON Schema) | `ParameterSchema` (JSON Schema) |
| 返回格式 | 纯 JSON String | `ResultEnvelope` (结构化信封) |
| 错误码 | 内嵌在 JSON 中 | 标准 `ErrorCode` 枚举 |
| 注册方式 | `ToolRegistry.register()` | `ToolRegistry.register()` |
| 调用方式 | `invoke(ToolCall)` → `complete(resultJson)` | `invoke(params)` → `Promise<ResultEnvelope>` |

### 8.2 协议速查表

```typescript
// ─── 快速参考 ───

// 1. 定义工具
const myTool: ToolInterface = {
  name: 'example_tool',
  description: '一个示例工具',
  parameterSchema: {
    type: 'object',
    properties: {
      param1: { type: 'string', description: '参数1' }
    },
    required: ['param1']
  },
  async invoke(params) {
    try {
      const result = await doSomething(params.param1);
      return { success: true, data: result, meta: { duration: 100 } };
    } catch (e) {
      return {
        success: false,
        error: { code: 'INTERNAL_ERROR', message: e.message },
        meta: { duration: 50 }
      };
    }
  }
};

// 2. 注册工具
ToolRegistry.register(myTool);

// 3. 获取 LLM Schema
const schemas = ToolRegistry.allSchemas();
// → [{ type: 'function', function: { name, description, parameters } }]

// 4. 调用工具
const tool = ToolRegistry.resolve('example_tool');
const result = await tool.invoke({ param1: 'value' });
// → { success: true, data: {...}, meta: { duration: 100 } }
```
