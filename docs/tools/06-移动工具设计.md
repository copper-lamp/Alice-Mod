# 移动工具设计（Movement Tools Design）

> 版本：v3.0
> 日期：2026-07-14
> 架构：LLM 调用工具 → 执行AI（假人）执行移动 → 返回结果
> 协议：遵循 `00-工具协议规范.md` 统一接口契约
> 变更：move_to 改用参数组合推断目标类型（参考 Numen 设计），移除 target/target_type/sprint/options 等冗余参数

***

## 0. 设计原则

### 0.1 核心概念

**移动由执行AI负责，LLM 只调用基础移动工具**

```
┌─────────────────────────────────────────────────────┐
│  LLM（大脑）                                         │
│  - 决定要去哪里                                       │
│  - 调用基础移动工具                                   │
└────────────────┬────────────────────────────────────┘
                 │ 调用工具
                 ▼
        ┌────────────────┐
        │  移动工具        │
        │  - move_to      │
        │  - ride/dismount│
        └────────┬───────┘
                 │ 委托执行
                 ▼
        ┌────────────────┐
        │  执行AI（假人）  │
        │  - 寻路算法     │
        │  - 避障         │
        │  - 跳跃/攀爬    │
        │  - 游泳         │
        │  - 视角控制     │
        │  （自动处理）    │
        └────────┬───────┘
                 │ 返回结果
                 ▼
        ┌────────────────┐
        │  工具返回        │
        │  - 成功/失败    │
        │  - 最终位置     │
        │  - 自然语言反馈  │
        └────────────────┘
```

### 0.2 设计原则

| 原则       | 说明                   |
| -------- | -------------------- |
| **粒度适中** | 一个工具 = 一个"动作"，不是"脚本" |
| **有反馈**  | 工具返回执行结果（成功/失败/中断原因） |
| **可失败**  | 寻路失败、被阻挡、体力不足等       |
| **有成本**  | 时间、饥饿、耐久             |
| **可中断**  | 移动过程中可被事件中断（遇敌、受伤）   |

## 2. 工具定义

### 2.1 move_to（移动到目标位置）

**设计思路**：参考 Numen 的隐式推断，通过参数组合决定目标类型，不再需要显式的 `target_type` 枚举。LLM 只需要填它知道的参数即可。

**调用方式**：

| 填充的参数 | 含义 | 说明 |
|-----------|------|------|
| `x` + `z` | 去位置 | Y 自动解析到地表，最常用的"去那里" |
| `x` + `y` + `z` | 去精确方块 | 已知精确坐标时使用 |
| `entity` | 追踪/接近实体 | 填此参数时不填 xyz，跟随移动实体 |
| `y` 单独 | 升降高度 | 在当前柱子上爬到目标 y |

**ParameterSchema**：

| 参数名 | 类型 | 必填 | 默认值 | 描述 |
|------|------|------|--------|------|
| `x` | `number?` | 否 | — | 目标 X。和 z 一起填 → 去位置；和 y+z 一起填 → 精确方块 |
| `y` | `number?` | 否 | — | 目标 Y。单独填 → 升降高度；和 x+z 填 → 精确方块；不填 → 自动找地面 |
| `z` | `number?` | 否 | — | 目标 Z。和 x 一起填 → 去位置 |
| `entity` | `string?` | 否 | — | 实体 ID/名称。填此项时不填 xyz，自动追踪实体位置 |
| `break` | `boolean` | 否 | `false` | 是否允许挖方块开路 |
| `distance` | `number` | 否 | `2` | 停止距离（格），实体目标时有用 |

**返回**：

```typescript
interface MoveToResult {
  x: number;         // 最终位置 X
  y: number;         // 最终位置 Y（地面高度）
  z: number;         // 最终位置 Z
  message: string;   // 自然语言反馈（成功/失败原因 + 教学信息）
}
```

**错误码**：

| 错误码 | 含义 | 触发条件 |
|-------|------|---------|
| `TIMEOUT` | 操作超时 | 寻路超时 |
| `NO_PATH` | 无法找到路径 | 无可达路径 |
| `NOT_FOUND` | 目标不存在 | 实体目标不存在 |
| `CANCELLED` | 操作被取消 | 被事件中断 |
| `INTERNAL_ERROR` | 内部错误 | 未预期异常 |

**示例调用**（去坐标位置 - 最常见）：

```
LLM 调用: move_to(x=128, z=-256)
返回:
{
  "x": 128.5, "y": 64, "z": -256.2,
  "message": "到达位置 (128, 64, -256)"
}
```

**示例调用**（追踪实体）：

```
LLM 调用: move_to(entity="zombie_123", distance=3)
返回:
{
  "x": 143.2, "y": 64, "z": -271.5,
  "message": "接近僵尸，距离 2.8 格"
}
```

**示例调用**（精确方块位置）：

```
LLM 调用: move_to(x=100, y=20, z=-300, break=true)
返回:
{
  "x": 100.5, "y": 20, "z": -300.2,
  "message": "到达精确位置 (100, 20, -300)。途中挖了 3 个方块开路"
}
```

**失败示例**：

```
LLM 调用: move_to(x=1000, z=-1000)
返回:
{
  "x": 150.2, "y": 64, "z": -280.1,
  "message": "超时: 距离目标还有 850 格，已在 30 秒内移动了 30 格。目的地太远，建议分段移动"
}
```

### 2.2 ride（骑乘）

**ParameterSchema**：

| 参数名 | 类型 | 必填 | 描述 |
|------|------|------|------|
| `entity_id` | `string` | 是 | 实体 ID（马、猪、船等） |

**返回**：

```typescript
interface RideResult {
  isRiding: boolean;
  mountType?: string;  // 骑乘类型
  message: string;
}
```

**错误码**：

| 错误码 | 含义 | 触发条件 |
|-------|------|---------|
| `NOT_RIDEABLE` | 实体不可骑乘 | 该实体不支持骑乘 |
| `TOO_FAR` | 距离太远 | 实体超出交互距离 |
| `NOT_FOUND` | 目标不存在 | 实体不存在 |

**示例调用**：

```
LLM 调用: ride(entity_id="horse_456")
返回:
{
  "isRiding": true,
  "mountType": "horse",
  "message": "骑上马"
}
```

### 2.3 dismount（下马/下船）

**参数**：无参数。

**返回**：

```typescript
interface DismountResult {
  isRiding: boolean;
  message: string;
}
```

**错误码**：

| 错误码 | 含义 | 触发条件 |
|-------|------|---------|
| `NOT_RIDING` | 当前未在骑乘状态 | 调用 dismount 时未骑乘 |

**示例调用**：

```
LLM 调用: dismount()
返回:
{
  "isRiding": false,
  "message": "下马成功"
}
```

## 3. 工具实现

### 3.1 工具注册

```typescript
// 注册移动工具
function registerMovementTools() {
  // move_to
  ToolRegistry.register({
    name: 'move_to',
    description: '移动到目标位置。x+z=去位置/Y自动找地面，x+y+z=精确方块，entity=追踪实体，y单独=升降高度',
    parameterSchema: {
      type: 'object',
      properties: {
        x: { type: 'number', description: '目标 X。和 z 一起填 → 去位置' },
        y: { type: 'number', description: '目标 Y。不填→自动找地面；单独填→升降高度' },
        z: { type: 'number', description: '目标 Z。和 x 一起填 → 去位置' },
        entity: { type: 'string', description: '实体 ID/名称。填此项时不填 xyz' },
        break: { type: 'boolean', description: '是否允许挖方块开路', default: false },
        distance: { type: 'number', description: '停止距离（格）', default: 2 }
      }
    },
    async invoke(params) {
      const { x, y, z, entity, break: allowBreak, distance = 2 } = params;
      
      // 解析目标类型
      const hasX = x != null, hasY = y != null, hasZ = z != null;
      
      if (entity) {
        // 实体目标 → 追踪
        return await executionAI.followEntity(entity, distance, allowBreak);
      } else if (hasX && hasZ) {
        // 坐标目标
        return await executionAI.moveTo(x, hasY ? y : null, z, allowBreak);
      } else if (hasY && !hasX && !hasZ) {
        // 升降高度
        return await executionAI.moveToHeight(y, allowBreak);
      } else {
        return { error: '需要提供 x+z（去位置）、x+y+z（精确方块）、entity（实体）或 y（升降高度）' };
      }
    }
  });
  
  // ride
  ToolRegistry.register({
    name: 'ride',
    description: '骑乘实体（马、猪、船等）',
    parameterSchema: {
      type: 'object',
      properties: {
        entity_id: { type: 'string', description: '实体 ID' }
      },
      required: ['entity_id']
    },
    async invoke(params) {
      return await executionAI.ride(params.entity_id);
    }
  });
  
  // dismount
  ToolRegistry.register({
    name: 'dismount',
    description: '下马/下船',
    parameterSchema: {
      type: 'object',
      properties: {},
      required: []
    },
    async invoke() {
      return await executionAI.dismount();
    }
  });
}
```

### 3.2 执行AI接口

```typescript
// 执行AI接口（假人控制）
interface ExecutionAI {
  // 移动到一个坐标位置
  moveTo(x: number, z: number, y?: number | null, allowBreak?: boolean): Promise<MoveToResult>;
  
  // 追踪实体
  followEntity(entityId: string, distance: number, allowBreak?: boolean): Promise<MoveToResult>;
  
  // 升降高度
  moveToHeight(y: number, allowBreak?: boolean): Promise<MoveToResult>;
  
  // 骑乘
  ride(entity_id: string): Promise<RideResult>;
  dismount(): Promise<DismountResult>;
}
```

### 3.3 执行AI实现（示例）

```typescript
class ExecutionAIImpl implements ExecutionAI {
  private player: Player;
  private pathfinder: Pathfinder;
  
  constructor(player: Player) {
    this.player = player;
    this.pathfinder = new Pathfinder();
  }
  
  async moveTo(x: number, z: number, y?: number | null, allowBreak?: boolean): Promise<MoveToResult> {
    const startPos = { x: this.player.pos.x, y: this.player.pos.y, z: this.player.pos.z };
    
    // 解析目标 Y
    let targetY = y;
    if (targetY == null) {
      targetY = await this.resolveGroundY(x, z);  // 自动找地表高度
    }
    
    // 计算路径
    const path = await this.pathfinder.findPath(startPos, { x, y: targetY, z }, { allowBreak });
    
    if (!path) {
      return {
        x: startPos.x, y: startPos.y, z: startPos.z,
        message: `无法找到通往 (${x}, ${z}) 的路径`
      };
    }
    
    // 执行移动
    const result = await this.executePath(path);
    
    return {
      x: this.player.pos.x,
      y: this.player.pos.y,
      z: this.player.pos.z,
      message: result.success
        ? `到达位置 (${Math.round(x)}, ${Math.round(targetY!)}, ${Math.round(z)})`
        : `移动失败: ${result.reason}，当前位置 (${Math.round(this.player.pos.x)}, ${Math.round(this.player.pos.y)}, ${Math.round(this.player.pos.z)})`
    };
  }
  
  async followEntity(entityId: string, distance: number, allowBreak?: boolean): Promise<MoveToResult> {
    const entity = getEntityById(entityId);
    if (!entity) {
      return {
        x: this.player.pos.x, y: this.player.pos.y, z: this.player.pos.z,
        message: `实体不存在: ${entityId}`
      };
    }
    
    // 持续追踪实体位置
    const targetPos = { x: entity.pos.x, y: entity.pos.y, z: entity.pos.z };
    const path = await this.pathfinder.findPath(
      { x: this.player.pos.x, y: this.player.pos.y, z: this.player.pos.z },
      targetPos,
      { allowBreak, targetDistance: distance }
    );
    
    if (!path) {
      return {
        x: entity.pos.x, y: entity.pos.y, z: entity.pos.z,
        message: `无法接近实体 ${entityId}`
      };
    }
    
    const result = await this.executePath(path);
    
    return {
      x: this.player.pos.x, y: this.player.pos.y, z: this.player.pos.z,
      message: result.success
        ? `接近 ${entity.name}，距离 ${distance} 格`
        : `追踪失败: ${result.reason}`
    };
```

## 5. 总结

### 5.1 核心设计

| 组件 | 职责 |
|------|------|
| **LLM（大脑）** | 决定要去哪里，调用基础移动工具 |
| **移动工具** | 封装移动能力，返回执行结果 |
| **执行AI（假人）** | 负责具体移动操作（寻路、避障、跳跃、攀爬、游泳、视角等） |

### 5.2 关键特性

1. **参数简洁**：通过参数组合推断目标类型，无需 target_type 枚举
2. **Y 自动解析**：不传 Y 时自动找地表高度，消除常见失败源
3. **教学反馈**：返回自然语言消息帮助 LLM 学习正确用法
4. **可失败**：寻路失败、超时等有清晰返回
5. **执行AI自动处理**：跳跃、攀爬、游泳、视角控制等由执行AI自动处理

### 5.3 工具列表

| 类别 | 工具 |
|------|------|
| 基础移动 | move_to（通过参数组合区分坐标/实体/升降目标） |
| 骑乘 | ride, dismount |
