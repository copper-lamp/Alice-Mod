# 学习与记忆工具设计（Memory & Learning Tools Design）

> 版本：v2.0
> 日期：2026-07-17
> 架构：LLM 调用工具 → 执行AI 执行操作 → 返回结果

---

## 1. 模块概述

### 1.1 目标

将 AI 的"学习与记忆"能力重构为 5 个独立分类，各司其职：

| 分类 | 工具数 | 用途 |
|------|:------:|------|
| **skill** | 1 | 加载技能文档，注入到 LLM 上下文中辅助决策 |
| **knowledge** | 1 | 查询知识库，获取领域知识 |
| **memory** | 4 | 管理经验记忆（事件/人物/经验），包含列表/查询/创建/修改/删除 |
| **maps** | 2 | 管理地图路径点，包含查询（关键词/半径）和编辑（CRUD） |
| **aim** | 3 | 管理主线/支线任务，AI 空闲时自动推进，可更新进度 |

**总计：11 个工具**

### 1.2 架构

```
┌──────────────────────────────────────────────────┐
│                  LLM 大脑                        │
│  - 工具提示词末尾附有已启用 skill 描述            │
│  - 无用户指令时，自动推进 aim 任务                │
│  - 主动调用 knowledge/memory/maps 查询信息        │
└──────┬──────┬──────┬──────┬──────┬───────────────┘
       │      │      │      │      │
       ▼      ▼      ▼      ▼      ▼
   skill  knowledge  memory  maps  aim
   (1个)   (1个)    (4个)   (2个)  (3个)
```

### 1.3 数据存储

所有数据统一存储在 SQLite 数据库中，通过 MemoryManager 统一管理。每种分类在 SQLite 中使用独立的表或 type 前缀隔离。

## 2. 工具分类详细设计

### 2.1 Skill — 技能加载

**设计思路**：Skill 不是传统意义上的"工具调用"，而是一个**加载机制**。系统启动时扫描 skills 目录下的 .md 文件，将已启用的技能描述放在所有工具提示词的末尾，让 LLM 在决策时始终知道有哪些技能可用。

**工具**：无独立工具调用入口。技能通过 SkillInjector 自动注入到 LLM system prompt 中。

**工作方式**：
1. skills 目录下的 .md 文件按阶段分类（plan/execute/transfer/summarize）
2. 启动时扫描加载，按 token 预算裁剪
3. 已启用技能的描述自动追加到工具提示词末尾
4. 无需 LLM 主动调用，始终在上下文中

**SkillInjector 接口**：

```typescript
interface Skill {
  name: string;           // 技能名称
  phase: SkillPhase;      // 适用阶段：plan/execute/transfer/summarize
  content: string;        // Markdown 内容
  enabledByDefault: boolean;
  estimatedTokens: number;
}

class SkillInjector {
  list(): Skill[];                              // 列出所有技能
  pick(phase, enabledSkills?, disabledSkills?): Skill[];  // 按阶段选取
  render(skills): string;                       // 渲染为 prompt 文本
}
```

### 2.2 Knowledge — 知识查询

**设计思路**：Knowledge 是**外部知识库**的查询入口，用于获取领域知识、合成配方、生物特性、方块属性等结构化信息。知识库内容由预设数据填充，AI 不可修改。

**工具**：1 个

| 工具名 | 功能 | 参数 | 返回 |
|--------|------|------|------|
| `knowledge_query` | 查询知识库 | `query: string` — 自然语言查询 | `ResultEnvelope<{ results: KnowledgeEntry[] }>` |

#### 2.2.1 knowledge_query

**功能**：查询知识库，获取与查询相关的知识点

**参数**（ParameterSchema）：

| 参数 | 类型 | 必填 | 描述 |
|------|------|:----:|------|
| `query` | `string` | 是 | 自然语言查询，例如"如何合成钻石剑" |
| `limit` | `number` | 否 | 返回结果上限，默认 5 |

**返回**：`ResultEnvelope<{ results: KnowledgeEntry[] }>`

```typescript
interface KnowledgeEntry {
  id: string;
  title: string;        // 知识点标题
  content: string;      // 知识点内容（纯文本）
  category: string;     // 分类：crafting/combat/survival/redstone/...
  tags: string[];       // 标签
  source?: string;      // 来源
}
```

**错误码**：`INTERNAL_ERROR`

**示例**：
```typescript
const result = await knowledge_query({
  query: '如何合成钻石剑',
  limit: 3
});
// 返回
// {
//   success: true,
//   data: {
//     results: [
//       {
//         id: 'kn_001',
//         title: '钻石剑合成配方',
//         content: '在工作台中，将 2 颗钻石放在第 1 列中间和下方，1 根木棍放在第 2 列中间',
//         category: 'crafting',
//         tags: ['武器', '钻石', '合成']
//       }
//     ]
//   }
// }
```

### 2.3 Memory — 经验记忆

**设计思路**：Memory 是 AI 的**经验系统**，记录 AI 在游戏过程中经历的事件、遇到的人物、积累的经验。记忆格式统一为 `{type, name, content}`，其中 content 为一段纯文本描述。

**记忆格式**：

```typescript
interface Memory {
  id: string;
  type: 'event' | 'character' | 'experience';  // 事件/人物/经验
  name: string;                                  // 记忆名称
  content: string;                               // 一段纯文本描述
  tags: string[];                                // 标签
  importance: number;                            // 重要度 1-10
  createdAt: number;
  updatedAt: number;
}
```

#### 2.3.1 memory_list

**功能**：列出所有记忆，支持按类型过滤和分页

**参数**（ParameterSchema）：

| 参数 | 类型 | 必填 | 描述 | 默认值 |
|------|------|:----:|------|:------:|
| `type` | `string` | 否 | 过滤类型：`event` \| `character` \| `experience` | - |
| `tags` | `array[string]` | 否 | 标签过滤（AND） | - |
| `limit` | `number` | 否 | 返回数量上限 | `20` |
| `offset` | `number` | 否 | 分页偏移量 | `0` |

**返回**：`ResultEnvelope<{ memories: Memory[], total: number }>`

#### 2.3.2 memory_query

**功能**：按关键词或语义搜索记忆

**参数**（ParameterSchema）：

| 参数 | 类型 | 必填 | 描述 | 默认值 |
|------|------|:----:|------|:------:|
| `keywords` | `array[string]` | 否 | 关键词匹配（在 content 中模糊搜索） | - |
| `query` | `string` | 否 | 语义查询文本（自然语言，向量检索） | - |
| `type` | `string` | 否 | 按类型过滤 | - |
| `limit` | `number` | 否 | 返回数量上限 | `10` |

**返回**：`ResultEnvelope<{ memories: Memory[], total: number }>`

#### 2.3.3 memory_edit

**功能**：创建、修改或删除记忆。通过 `action` 参数区分操作模式

**参数**（ParameterSchema）：

| 参数 | 类型 | 必填 | 描述 |
|------|------|:----:|------|
| `action` | `string` | 是 | 操作类型：`create` \| `update` \| `delete` |
| `id` | `string` | 否 | 记忆 ID（`update`/`delete` 时必填） |
| `type` | `string` | 否 | 记忆类型（`create` 时必填）：`event` \| `character` \| `experience` |
| `name` | `string` | 否 | 记忆名称（`create` 时必填，`update` 时可选） |
| `content` | `string` | 否 | 记忆内容，一段纯文本（`create` 时必填，`update` 时可选） |
| `tags` | `array[string]` | 否 | 标签列表 |
| `importance` | `number` | 否 | 重要度 1-10 |

**返回**：`ResultEnvelope<{ id?: string }>`

- `action=create` 时返回新记忆的 `id`
- `action=update`/`delete` 时返回 `{ id: string }`

**错误码**：
- `INTERNAL_ERROR`：内部错误
- `MEMORY_NOT_FOUND`：指定的记忆 ID 不存在
- `INVALID_PARAMS`：参数校验失败

**示例**：
```typescript
// 创建记忆
const result = await memory_edit({
  action: 'create',
  type: 'event',
  name: '发现钻石矿脉',
  content: '在坐标 (128, 20, -256) 发现了一个钻石矿脉，约 10 个矿脉，附近有岩浆湖',
  tags: ['钻石', '矿区', 'Y=20'],
  importance: 8
});

// 修改记忆
const result = await memory_edit({
  action: 'update',
  id: 'mem_001',
  content: '在坐标 (128, 20, -256) 发现钻石矿脉，已开采完毕',
  importance: 5
});

// 删除记忆
const result = await memory_edit({
  action: 'delete',
  id: 'mem_001'
});
```

### 2.4 Maps — 地图路径点

**设计思路**：Maps 管理 AI 探索过的地图路径点。路径点格式统一为 `{dimension, coordinates, name, description}`。支持按关键词搜索和按半径搜索。

**路径点格式**：

```typescript
interface Waypoint {
  id: string;
  dimension: 'overworld' | 'nether' | 'the_end';
  x: number;
  y: number;
  z: number;
  name: string;              // 路径点名称
  description?: string;      // 可选描述
  tags: string[];
  createdAt: number;
  updatedAt: number;
}
```

#### 2.4.1 maps_query

**功能**：搜索路径点，支持按关键词搜索和按半径搜索

**参数**（ParameterSchema）：

| 参数 | 类型 | 必填 | 描述 | 默认值 |
|------|------|:----:|------|:------:|
| `keywords` | `array[string]` | 否 | 按关键词搜索（匹配 name/description/tags） | - |
| `x` | `number` | 否 | 半径搜索中心 X（与 `radius` 配合使用） | - |
| `z` | `number` | 否 | 半径搜索中心 Z | - |
| `radius` | `number` | 否 | 搜索半径（格） | - |
| `dimension` | `string` | 否 | 过滤维度 | - |
| `limit` | `number` | 否 | 返回上限 | `20` |

**返回**：`ResultEnvelope<{ waypoints: Waypoint[], total: number }>`

**注意**：`keywords` 和 `(x, z, radius)` 是两种搜索模式，可组合使用。若同时提供 `keywords` 和 `radius`，则取交集。

#### 2.4.2 maps_edit

**功能**：创建、修改或删除路径点。通过 `action` 参数区分操作模式

**参数**（ParameterSchema）：

| 参数 | 类型 | 必填 | 描述 |
|------|------|:----:|------|
| `action` | `string` | 是 | 操作类型：`create` \| `update` \| `delete` |
| `id` | `string` | 否 | 路径点 ID（`update`/`delete` 时必填） |
| `dimension` | `string` | 否 | 维度（`create` 时必填） |
| `x` | `number` | 否 | X 坐标（`create` 时必填） |
| `y` | `number` | 否 | Y 坐标（`create` 时必填） |
| `z` | `number` | 否 | Z 坐标（`create` 时必填） |
| `name` | `string` | 否 | 路径点名称（`create` 时必填） |
| `description` | `string` | 否 | 可选描述 |
| `tags` | `array[string]` | 否 | 标签列表 |

**返回**：`ResultEnvelope<{ id?: string }>`

**错误码**：
- `INTERNAL_ERROR`：内部错误
- `WAYPOINT_NOT_FOUND`：指定的路径点 ID 不存在
- `INVALID_PARAMS`：参数校验失败

**示例**：
```typescript
// 创建路径点
const result = await maps_edit({
  action: 'create',
  dimension: 'overworld',
  x: 128,
  y: 20,
  z: -256,
  name: '钻石矿脉',
  description: '发现大量钻石矿石',
  tags: ['钻石', '矿物']
});

// 修改路径点
const result = await maps_edit({
  action: 'update',
  id: 'wp_001',
  name: '钻石矿脉（已开采）',
  description: '已开采完毕的钻石矿脉'
});

// 删除路径点
const result = await maps_edit({
  action: 'delete',
  id: 'wp_001'
});
```

### 2.5 Aim — 目标任务

**设计思路**：Aim 是**用户定义的主线/支线任务系统**。AI 在没有接收到用户指令时，会自动查看并推进 aim 任务。任务采用表格结构，AI 可更新进度（勾选已完成子项）。

**任务结构**：

```typescript
interface AimTask {
  id: string;
  type: 'main' | 'side';           // 主线/支线
  title: string;                    // 任务标题
  description: string;              // 任务描述
  items: AimItem[];                 // 子任务列表（表格行）
  progress: number;                 // 进度百分比 0-100
  status: 'active' | 'completed' | 'abandoned';
  createdAt: number;
  updatedAt: number;
}

interface AimItem {
  id: string;
  content: string;                  // 子任务内容
  done: boolean;                    // 是否已完成
}
```

#### 2.5.1 aim_list

**功能**：列出所有目标任务，按类型过滤

**参数**（ParameterSchema）：

| 参数 | 类型 | 必填 | 描述 | 默认值 |
|------|------|:----:|------|:------:|
| `type` | `string` | 否 | 过滤类型：`main` \| `side` | - |
| `status` | `string` | 否 | 过滤状态：`active` \| `completed` \| `abandoned` | - |

**返回**：`ResultEnvelope<{ tasks: AimTask[] }>`

#### 2.5.2 aim_query

**功能**：查询指定任务的详细内容和进度

**参数**（ParameterSchema）：

| 参数 | 类型 | 必填 | 描述 |
|------|------|:----:|------|
| `id` | `string` | 是 | 任务 ID |

**返回**：`ResultEnvelope<{ task: AimTask }>`

#### 2.5.3 aim_update

**功能**：更新任务进度，支持勾选/取消勾选子任务

**参数**（ParameterSchema）：

| 参数 | 类型 | 必填 | 描述 |
|------|------|:----:|------|
| `id` | `string` | 是 | 任务 ID |
| `item_id` | `string` | 是 | 子任务项 ID |
| `done` | `boolean` | 是 | 是否已完成 |

**返回**：`ResultEnvelope<{ task: AimTask }>`

**行为**：当 `item_id` 对应的子任务被勾选为 `done=true` 时，系统自动重新计算 `progress`（已完成项数 / 总项数 × 100）。当所有子任务完成时，自动将 `status` 设为 `completed`。

**示例**：
```typescript
// 列出所有活跃的主线任务
const result = await aim_list({
  type: 'main',
  status: 'active'
});

// 查询任务详情
const result = await aim_query({
  id: 'aim_001'
});

// 勾选子任务完成
const result = await aim_update({
  id: 'aim_001',
  item_id: 'item_003',
  done: true
});
```

## 4. 交互流程

### 4.1 技能注入流程

```
系统启动
  ↓
扫描 skills/ 目录，加载 .md 技能文件
  ↓
按阶段和 token 预算筛选已启用技能
  ↓
将所有工具 Schema + 已启用技能描述 → 组装为 LLM system prompt
  ↓
LLM 在决策时始终能看到技能描述
```

### 4.2 记忆存储流程

```
AI 经历了重要事件（如发现钻石矿脉）
  ↓
AI 调用 memory_edit({ action: 'create', type: 'event', ... })
  ↓
执行AI 存储到 SQLite（type='event', name='发现钻石矿脉', content='...'）
  ↓
若 Chroma 可用，自动生成向量嵌入
  ↓
返回 { id: 'mem_xxx' }
```

### 4.3 地图路径点流程

```
AI 到达一个值得记录的位置
  ↓
AI 调用 maps_edit({ action: 'create', dimension, x, y, z, name, ... })
  ↓
执行AI 存储到 SQLite
  ↓
自动同步到空间索引（MapIndex）
  ↓
返回 { id: 'wp_xxx' }
```

### 4.4 任务推进流程

```
用户定义 aim 任务（通过配置文件或聊天界面）
  ↓
AI 空闲时调用 aim_list() 查看活跃任务
  ↓
AI 选择任务开始执行
  ↓
完成一个子任务后，调用 aim_update({ id, item_id, done: true })
  ↓
系统自动计算进度百分比
  ↓
所有子任务完成 → 自动标记 status='completed'
```

## 5. 与现有系统的关系

### 5.1 复用 MemoryManager

v2.0 工具设计复用了现有的 `MemoryManager` 作为底层存储引擎：

```
memory_edit  → MemoryManager.store() / update() / forget()
memory_list  → MemoryManager.list()
memory_query → MemoryManager.recall() / getSimilar()
maps_query   → MemoryManager.mapIndex.query()
maps_edit    → MemoryManager.mapIndex.saveWaypoint() / updateWaypoint() / deleteWaypoint()
```

### 5.2 复用 MapIndex

现有的 `MapIndex` 空间索引系统继续用于 `maps_query` 的半径搜索，无需重新实现。

### 5.3 复用 SkillInjector

现有的 `SkillInjector` 继续用于技能注入，无需修改。

### 5.4 新增 Aim 存储

Aim 任务数据存储在 SQLite 中，可复用现有数据库连接，使用独立的表 `aim_tasks` 和 `aim_items`。

```typescript
/** IPC 调用封装 — 记忆（v2.0 重构） */
export const memoryApi = {
  // 记忆列表（按类型过滤）
  memoryList: (params: { type?: string; tags?: string[]; limit?: number; offset?: number }) =>
    window.electronAPI.invoke('memory:list', params) as Promise<{ memories: MemoryItem[]; total: number }>,

  // 记忆搜索（关键词/语义）
  memoryQuery: (params: { keywords?: string[]; query?: string; type?: string; limit?: number }) =>
    window.electronAPI.invoke('memory:query', params) as Promise<{ memories: MemoryItem[]; total: number }>,

  // 记忆编辑（create/update/delete 三合一）
  memoryEdit: (params: { action: 'create' | 'update' | 'delete'; id?: string; type?: string; name?: string; content?: string; tags?: string[]; importance?: number }) =>
    window.electronAPI.invoke('memory:edit', params) as Promise<{ success: boolean; id?: string; error?: string }>,

  // 保留旧接口兼容（调用新的 memory:... channel）
  list: (params: any) => window.electronAPI.invoke('memory:list', params) as Promise<any>,
  store: (params: any) => window.electronAPI.invoke('memory:store', params) as Promise<any>,
  update: (id: string, updates: Record<string, unknown>) => window.electronAPI.invoke('memory:update', { id, updates }) as Promise<any>,
  forget: (id: string) => window.electronAPI.invoke('memory:forget', { id }) as Promise<any>,
  similar: (query: string, type?: string, limit?: number) => window.electronAPI.invoke('memory:similar', { query, type, limit }) as Promise<any>,
}

/** IPC 调用封装 — 地图路径点（v2.0 新增） */
export const mapsApi = {
  list: (params: { keywords?: string[]; x?: number; z?: number; radius?: number; dimension?: string; limit?: number; offset?: number }) =>

  create: (params: { dimension: string; x: number; y: number; z: number; name: string; description?: string; tags?: string[] }) =>
    window.electronAPI.invoke('maps:create', params) as Promise<{ id: string }>,

  update: (params: { id: string; name?: string; description?: string; tags?: string[] }) =>
    window.electronAPI.invoke('maps:update', params) as Promise<{ success: boolean }>,

  delete: (id: string) =>
    window.electronAPI.invoke('maps:delete', { id }) as Promise<{ success: boolean }>,
}

/** IPC 调用封装 — 目标任务（v2.0 新增） */
```
