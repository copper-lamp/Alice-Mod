# 自定义资料库设计

> 版本：v1.0
> 日期：2026-07-11
> 状态：待实现

---

## 1. 概述

允许用户在资料库中添加自定义知识条目，支持 3 种来源类型：

| 类型 | 说明 | 存储内容 |
|------|------|----------|
| URL | 网页链接 | URL + 标题 + 描述 |
| 本地文档 | 本地文件 | 文件路径 + 文件名 + 文件内容（文本） |
| 手动输入 | 自由文本 | 标题 + Markdown 正文 |

所有条目统一存储在记忆系统的 `memory_meta` 表中，通过 `type='custom_knowledge'` 区分。

---

## 2. 数据模型

### 2.1 记忆内容结构

```typescript
// URL 类型
interface UrlKnowledge {
  sourceType: 'url'
  url: string
  title: string
  description: string
}

// 本地文档类型
interface DocumentKnowledge {
  sourceType: 'document'
  filePath: string
  fileName: string
  content: string   // 文件文本内容（截断至 10000 字符）
}

// 手动输入类型
interface ManualKnowledge {
  sourceType: 'manual'
  title: string
  content: string   // Markdown 格式
}
```

### 2.2 记忆元数据

| 字段 | 值 |
|------|-----|
| `type` | `'custom_knowledge'` |
| `branch` | `'knowledge'` |
| `tags` | `[sourceType, ...userTags]` |
| `importance` | `5`（默认，用户可调） |

## 3. 后端 IPC

### 3.1 新增 `dialog:open-file`

打开系统文件选择对话框，返回选中文件的路径和文本内容。

```
渲染进程 → invoke('dialog:open-file') → 主进程
  → 打开文件对话框（过滤 .txt/.md/.json/.csv 等）
  → 读取文件内容（UTF-8）
  → 返回 { filePath, fileName, content }
```

### 3.2 复用 `memory:store`

自定义资料条目通过已有的 `memory:store` 存储：

```typescript
await memoryApi.store({
  type: 'custom_knowledge',
  branch: 'knowledge',
  content: { sourceType: 'url', url: '...', title: '...', description: '...' },
  tags: ['url', 'minecraft'],
  importance: 5,
})
```

### 3.3 复用 `memory:forget` / `memory:update`

删除和编辑通过已有的 `memory:forget` 和 `memory:update` 实现。

## 4. 前端 UI

### 4.1 添加按钮

在搜索框右侧添加 "+" 圆形按钮：

```
[搜索框...] [搜索] [+]
```

### 4.2 添加弹窗

点击 "+" 弹出模态框，内含 3 个 Tab：

```
┌─────────────────────────────────────┐
│  📝 添加自定义知识                    │
│  ┌──────┬──────────┬──────────┐     │
│  │ URL  │ 本地文档  │ 手动输入  │     │
│  ├──────┴──────────┴──────────┤     │
│  │ URL 类型表单:               │     │
│  │   URL: [________________]  │     │
│  │   标题: [________________]  │     │
│  │   描述: [________________]  │     │
│  └────────────────────────────┘     │
│           [取消] [保存]             │
└─────────────────────────────────────┘
```

**URL Tab**:
- URL 输入框（必填）
- 标题输入框（可选，自动从页面抓取）
- 描述文本域（可选）

**本地文档 Tab**:
- 「选择文件」按钮 → 触发 `dialog:open-file`
- 显示选中文件路径
- 支持格式：.txt, .md, .json, .csv, .log, .yml, .yaml, .toml, .ini, .cfg

**手动输入 Tab**:
- 标题输入框（必填）
- Markdown 内容文本域（必填）
